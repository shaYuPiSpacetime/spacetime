这是一套独立静态 Demo，用一眼能懂的方式展示 PRD-01 的移动端准入链路、资料认证、核心拦截，以及管理后台的用户管理、审核和配置。

16移动端页面/状态
7后台工作台
0真实接口依赖
100%写死脱敏数据

移动端

登录、性别、出生日期、身份、学历、现居地、基本资料、三重认证和核心准入拦截。

进入

管理后台

App 用户卡片宫格、画像抽屉、审核队列、准入配置和导入导出弹窗。

进入

页面范围

轻量引导

授权、性别、生日、身份、学历、城市，保存后反显到基本资料。

资料认证

头像、实名、学历、三重认证状态和资料完整度。

后台运营

用户筛选、画像、冻结、导入、导出、审核与准入配置。

端到端闭环

移动端提交资料和认证，核心功能按缺口拦截；管理后台查看画像、处理审核、保存准入配置；Mock 后端用同语义数据表达接口形态。

1. 首登

协议、验证码、轻量资料。

2. 认证

头像、实名、学历状态流。

3. 拦截

喜欢、私信、曝光统一提示缺口。

4. 审核

后台审核通过后准入开放。

架构说明

前端静态页可直接双击打开。后端 mock server 可选启动，用于展示接口契约和数据流，和前端默认写死数据互不依赖。
