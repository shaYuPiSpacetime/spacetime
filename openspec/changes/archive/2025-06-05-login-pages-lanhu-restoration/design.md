## Context

当前登录资料四页（`miniapp/src/pages/login/gender.tsx`、`age.tsx`、`education.tsx`、`address.tsx`）已实现基础交互逻辑，并有一个共用壳组件 `LoginProfileShell`。但各页面视觉样式与蓝湖设计稿仍有偏差——背景色、卡片尺寸、圆角、字体、间距、按钮状态等未做到像素级还原。

蓝湖设计稿以 750×1624 尺寸提供，本项目直接按 `rpx`（750rpx = 屏幕宽度）做一一映射，无需额外换算。

**技术栈：** Taro 3.x + React 18 + TypeScript，小程序原生组件（`@tarojs/components`）内联样式。

## Goals / Non-Goals

**Goals:**
- 四个登录资料页视觉严格按照蓝湖设计稿 1:1 还原
- 共用壳 `LoginProfileShell` 统一管理背景、导航、标题、下一步按钮
- 各页面差异部分通过 children 插槽和组件 props 配置
- 保留现有交互逻辑（性别点选、年龄滚动、学历点选、地址选择/定位）
- 不修改页面路由和登录流程

**Non-Goals:**
- 不修改后端接口或数据模型
- 不新增登录步骤或变更流程
- 不引入新的第三方 UI 库
- 不涉及性能优化或 bundle 体积优化

## Decisions

### 1. 组件拆分策略：壳组件 + 页面内联子组件

```
LoginProfileShell (共用壳)
├── gender.tsx
│   └── GenderCard × 2 (页面内联)
├── age.tsx
│   └── PickerView + 可见层 (页面内联)
├── education.tsx
│   └── EducationCard × 4 (页面内联)
└── address.tsx
    └── 选择行 + 定位行 (页面内联)
```

**选择理由：** 四页差异较大，抽象成通用组件反而增加复杂度。壳组件负责固定布局（背景、导航、标题、按钮），各页面通过 `children` 填充内容区域。页面内的卡片/选择器等子组件如果简单（如 education 的选项卡片），直接在页面中通过 `map` 渲染即可；如果需要复用（如 gender 的 GenderCard），定义为页面内联函数组件。

**备选方案：** 将 GenderCard、EducationCard 等都抽成独立文件 → 过于碎片化，四页组件没有跨页面复用需求。

### 2. 样式方案：内联 style 对象

**选择理由：** 沿用项目现有风格——所有页面和组件都使用内联 `style` 对象 + rpx 单位，不引入 SCSS/CSS Module。这在 Taro 小程序场景下足够灵活，且不会产生样式冲突。

**备选方案：** SCSS Module → 需要额外配置，且与项目现有风格不一致。

### 3. 年龄页视觉方案：透明 PickerView + 自定义可见层

蓝湖要求五行三列视觉效果（选中行浅蓝背景 + 蓝色描边），但小程序原生 `PickerView` 不能自定义指示器外观。采用「透明 PickerView 覆盖 + 自定义可见渲染层」方案：
- 底层：自定义 View 渲染五行（基于当前 value 计算 ±2 偏移的可见值）
- 顶层：`PickerView` 设置 `opacity: 0` 覆盖在自定义视图上，接收触摸事件
- 选中行高亮：蓝色背景 + 蓝色描边固定在第 3 行位置

**选择理由：** 这是唯一能在保持原生滚轮体验的同时实现自定义外观的方案。

### 4. 地址页简化

删除现有城市快捷标签区域，简化为一行「选择城市」（带 ◎ 图标）+「获取定位」按钮。点击「选择城市」用 `Taro.showActionSheet` 两级联动选择省/市；点击「获取定位」调用 `Taro.chooseLocation` / `Taro.getLocation`。

### 5. 下一步按钮状态管理

| 页面 | 默认态 | 交互后 |
|------|--------|--------|
| 性别 | 高亮（#2876FF） | 始终高亮（有默认值） |
| 年龄 | 浅蓝（#E3F1FE） | 用户滚动后高亮 |
| 学历 | 浅蓝（#E3F1FE） | 用户点击后高亮 |
| 地址 | 浅蓝（#E3F1FE） | 选择城市/定位后高亮 |

性别页有默认选中值，所以按钮始终可用；其余三页需要用户主动交互后才激活。

## Risks / Trade-offs

- **[PickerView 兼容性]** 不同机型/微信版本的 PickerView 触摸响应区域可能有差异 → 已在现有 age.tsx 中验证可行
- **[蓝湖资源依赖]** 如设计稿中有背景图片或图标资源，需要先导出切图 → 当前使用纯 CSS 渐变和 Unicode 字符代替，如后期需要切图，替换即可
- **[地址数据 Mock]** 省份城市数据硬编码在 `useLogin` hook 中 → 后续接入真实接口时替换数据源即可，不影响 UI
