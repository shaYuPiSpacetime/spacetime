统一分层规范（每个业务包必须遵守）

controller 接收请求、参数校验、调 service、返回 R<T>
service 接口定义，只声明方法签名和 Javadoc
service.impl 实现类，加 @Service，注入 Mapper，写业务逻辑。禁止直接调 controller
mapper MyBatis-Plus BaseMapper 接口，复杂 SQL 写 XML
dto request = 入参对象（加 @Valid 校验注解），response = 返回给前端的 VO

技术要点

  • Java 21 — 虚拟线程（Virtual Threads）、Record、Pattern Matching
  • MyBatis-Plus 3.5+ — BaseMapper 自动 CRUD，LambdaQueryWrapper，分页插件
  • Entity 统一放 common/entity/，所有 mapper 都可以引用同一份实体
  • DTO 不要跨包用 — admin 的 dto 只在 admin 包内流转，miniapp 同理
