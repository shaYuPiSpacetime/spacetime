4. 前端编码规范 + 目录结构

admin-web 目录结构
src/
├── components/       ← 公共组件
│   ├── ui/              ← shadcn/ui 组件
│   ├── layout/         ← AppLayout, Sidebar, Header
│   └── common/         ← PageContainer, DataTable, ConfirmDialog
├── pages/            ← 页面（按菜单模块分）
│   ├── user/          ← 用户管理
│   ├── auth/          ← 认证审核
│   ├── scale/         ← 量表管理
│   ├── finance/       ← 财务中心
│   ├── operation/     ← 运营中心
│   ├── system/        ← 系统管理
│   ├── promotion/     ← 推广管理
│   └── login/         ← 登录页
├── hooks/           ← 自定义 hooks
│   ├── useAuth.ts
│   ├── useTable.ts      ← 分页+搜索+排序
│   └── usePermission.ts
├── stores/          ← Zustand stores
│   ├── authStore.ts
│   └── appStore.ts
├── api/             ← axios 请求
│   ├── request.ts     ← axios 实例，拦截器，统一错误
│   ├── user.ts
│   └── finance.ts
├── router/          ← react-router-dom 路由配置
│   ├── index.tsx
│   └── guard.tsx      ← 路由守卫（登录/权限）
├── types/           ← TypeScript 类型
├── utils/           ← 工具函数
├── App.tsx
└── main.tsx
命名规范
  • 组件文件：PascalCase — UserList.tsx
  • 页面目录：kebab-case — user-mgr/
  • Hook：useXxx.ts
  • Store：xxxStore.ts
  • API 文件：按模块 — user.ts, finance.ts
  • Type：XxxVO, XxxReq（和后端对齐）
关键约束
  • 页面组件只负责拼 UI + 调 Hook，不写业务逻辑
  • API 调用统一走 api/ 目录，不在页面里直接 axios
  • 列表页统一用 useTable hook 管理分页/搜索/排序
  • 路由守卫在 router/guard.tsx 统一处理，不在每个页面判断
  • 敏感字段脱敏在工具函数做，不在 JSX 里直接 slice
