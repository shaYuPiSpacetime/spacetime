3. 后端编码规范

Java 类命名规范
层级 命名 示例
ControllerXxxControllerUserMgrController
Service 接口XxxServiceUserMgrService
Service 实现XxxServiceImplUserMgrServiceImpl
MapperXxxMapperUserMapper
Entity表名AppUser
Request DTOXxxReqUserPageReq
Response VOXxxVOUserDetailVO
枚举XxxEnumUserStatusEnum
关键约束
  • Controller 不写业务逻辑，只做参数校验 + 调 Service + 返回
  • Controller 统一返回 R<T>（code + msg + data）
  • Service 接口必须写，不能直接在 Controller 调 ServiceImpl
  • Mapper 不能跨包调用：admin 的 Service 不能调 miniapp 的 Mapper
  • 分页查询统一用 MyBatis-Plus Page + LambdaQueryWrapper
  • 复杂 SQL 写 XML，简单查询用 LambdaQueryWrapper
  • 异常统一抛 BusinessException，GlobalExceptionHandler 拦截
  • 日期字段用 LocalDateTime，别用 Date
  • 手机号/身份证 脱敏用工具类 DesensitizeUtil.mask()
SQL 规范
  • 关键字大写，表名字段名小写下划线（SELECT ... FROM ... WHERE ...）
  • 禁止 SELECT *，必须列出字段
  • WHERE 条件里的列必须走索引，新表建表时同步评估索引
  • 批量操作用 MyBatis-Plus saveBatch / updateBatch，禁止循环单条
  • 模糊查询用 LIKE CONCAT('%', #{val}, '%')，不用 ${} 拼接
  • 分页查询必须有 ORDER BY，否则每次翻页结果可能不同
