分层架构 — 加入 DAO 层

Controller → Service → ServiceImpl → DAO → DAOImpl → Mapper

完整调用链
Controller → Service
接口 → ServiceImpl
实现类 → DAO
接口 → DAOImpl
实现类 → Mapper
SQL在这里
各层职责
层 职责 不能做什么
Controller 接收请求、参数校验(@Valid)、调 Service、返回 R<T> 不能写业务逻辑，不能直接调 DAO
Service 接口定义 + Javadoc 没有任何实现代码
ServiceImpl 业务逻辑、事务控制(@Transactional)、组装数据 不能写 SQL，不能直接调 Mapper
DAO 数据访问接口定义 没有任何实现代码
DAOImpl 数据访问实现，注入 Mapper，调 MyBatis-Plus 不能写业务逻辑（纯粹的数据存取）
Mapper 继承 BaseMapper<Entity>，写 XML SQL 不写业务逻辑
