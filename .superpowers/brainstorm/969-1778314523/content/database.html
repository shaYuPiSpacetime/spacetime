数据库选型

影响到 JSON 字段、全文搜索、字符集、运维成本

A

MySQL 8.0+

最主流。国内云厂商默认支持，运维资料丰富。

Pros

  • 团队最熟悉，学习成本为零
  • 云厂商 RDS 一键创建
  • MyBatis-Plus 兼容性最好
  • 8.0 支持 JSON 字段、窗口函数、CTE

Cons

  • JSON 查询不如 PostgreSQL 灵活
  • 全文搜索弱（中文需额外 ES）
B

PostgreSQL 15+

功能最强。JSONB、全文搜索、数组类型原生支持。

Pros

  • JSONB 字段灵活（量表计分规则等配置类数据很适合）
  • 原生全文搜索可省 ES
  • 数组、枚举类型丰富

Cons

  • 国内云厂商 RDS 稍贵
  • 团队需要学习曲线

推荐

MySQL 8.0。理由：

  • 你们项目没到需要 PG 高级特性的规模
  • MyBatis-Plus 对 MySQL 适配最成熟
  • 微信小程序生态几乎清一色 MySQL，招人也方便
  • 后期真有搜索需求可以补 ES，没必要为这个换 PG
