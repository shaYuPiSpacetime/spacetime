方案 A — 包路径分区

一个 Spring Boot 项目，通过 package 划清模块边界

项目包结构
com.chengjialiye
├── common/ ← 公共：实体、工具、配置、异常
│   ├── entity/   User, Order, Scale...
│   ├── util/     JwtUtil, DesensitizeUtil...
│   ├── config/   RedisConfig, SecurityConfig...
│   └── exception/
├── admin/ ← 管理后台
│   ├── controller/   UserMgrController, AuditController...
│   ├── service/
│   └── dto/
└── miniapp/ ← 小程序端
    ├── controller/   MateController, VipController...
    ├── service/
    └── dto/

调用规则（团队约束）

admin/
→ 可以调 →
common/
← 可以调 ←
miniapp/
admin/ 和 miniapp/ 不能互相调用

和方案 B 相比

  • 更简单，一个项目启动就能跑全部
  • 不需要 Maven 多模块配置
  • 包名就是边界，约定大于配置

要注意

  • admin/ 和 miniapp/ 不能互相 import（靠 Code Review 守住）
  • 以后拆微服务需要搬代码
