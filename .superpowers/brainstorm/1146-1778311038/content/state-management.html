React 状态管理

管理后台的状态量：登录态、用户信息、路由权限、列表查询条件、UI 状态

A

Zustand

轻量、无模板代码、TypeScript 友好、按需订阅渲染。React 社区当前最主流选择。

Pros

  • API 极简，5 分钟上手
  • 无 Provider 包裹地狱
  • ts 类型推断自然
  • 体积小（~1KB）

Cons

  • 没有 Redux DevTools 那么成熟的调试工具链
B

React Context + useReducer

React 内置方案。不需要额外依赖。

Pros

  • 零依赖
  • 足够覆盖后台常见场景

Cons

  • Context 值变化时所有消费者重渲染
  • 多 Context 嵌套恶心
  • 没有中间件（如持久化、日志）
C

Redux Toolkit

老牌方案，生态最全。但管理后台场景偏重。

Pros

  • RTK Query 可接管 API 请求
  • DevTools 调试最强

Cons

  • 模板代码多（slice/createAsyncThunk）
  • 对管理后台来说太重了

推荐

A — Zustand。管理后台的状态模式很固定（登录 → 拿菜单 → CRUD），不需要 Redux 的复杂度。Zustand 足够用，新人看一眼就明白。
