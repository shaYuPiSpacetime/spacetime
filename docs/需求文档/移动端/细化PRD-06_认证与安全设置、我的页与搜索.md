# 成家立业小程序 细化PRD-06 认证与安全设置、我的页与搜索

## 1. 文档定位

- 文档名称：细化PRD-06 认证与安全设置、我的页与搜索
- 适用端：移动端（微信小程序）
- 文档目标：在推荐算法、测评体系暂未确定前，先补齐不依赖算法结果的用户自助管理能力，包括认证中心聚合展示、安全中心、隐私与通知设置、我的页入口体系、帮助与客服、全局搜索。
- 关联文档：
  - `细化PRD-01_用户准入与资料认证初始化`
  - `细化PRD-03_消息、私信与通知中心`
  - `细化PRD-04_商业化（VIP、成家币、解锁与资产中心）`
  - `细化PRD-05_推荐模块（朋友、社区与内容互动）`

## 2. 范围说明

### 2.1 本期纳入

- 认证中心入口聚合、认证结果查看、认证状态引导
- 安全中心
- 隐私设置
- 通知设置
- 黑名单/不看TA动态/关键词屏蔽
- 账号安全
- 我的页
- 设置页
- 客服、帮助、公告、协议、关于我们等静态服务入口
- 全局搜索首页与搜索结果页

### 2.2 本期不纳入

- 推荐算法本身
- 精选、理想型、觅缘主推荐位的策略规则
- 测评体系与量表
- 已判定为旧遗留的历史页面口径，不再单独立项

### 2.3 与已出PRD的边界

- `PRD-01` 负责三项认证的首次提交、审核流、状态机和准入门槛。
- 本文档只补充认证中心在“用户已注册后”的查看、重提、引导、服务承接，不重复展开初始化采集流程。
- `PRD-03` 已定义消息/通知中心规则，本文只定义通知开关入口与用户侧设置。
- `PRD-04` 已定义 VIP、成家币、隐藏访问记录等商业规则，本文只定义对应设置入口与展示口径。

## 3. 产品目标

### 3.1 用户目标

- 用户可以快速查看自己的认证状态、资料安全状态和账号风险状态。
- 用户可以自行完成隐私偏好、通知偏好、屏蔽关系和账号管理。
- 用户可以从“我的”统一进入资产、认证、客服、帮助、协议等服务入口。
- 用户可以通过搜索快速找到用户、动态、话题等目标内容。

### 3.2 业务目标

- 降低客服承接基础咨询的压力。
- 提升认证补齐率、资料完善率、设置项使用率。
- 为后续推荐算法、商业化、社区治理提供稳定的用户设置底座。

## 5. 我的页

### 5.1 页面目标

- 作为用户个人服务中心，承接认证、资产、客服、帮助、设置、协议等入口。
- 不承担推荐结果展示，不与推荐算法模块耦合。

### 5.2 页面结构

#### 5.2.1 顶部用户卡片

- 展示头像、昵称、年龄、学校、城市、资料完成度。
- 展示三项认证状态徽章：实名认证、头像认证、学历认证。
- 若三项未完成，卡片区展示引导文案：
  - 文案示例：`完成实名认证、头像认证、学历认证后，才能进入成家核心互动功能`
- 点击用户卡片进入个人资料编辑页。

#### 5.2.2 核心功能入口

- 认证中心
- 会员中心
- 我的成家币
- 喜欢我的
- 最近看过我的
- 相互喜欢
- 消息通知

说明：

- 其中 `喜欢我的 / 最近看过我的 / 相互喜欢 / 消息通知 / 会员中心 / 我的成家币` 的具体规则以已输出 PRD 为准。
- 我的页只负责入口聚合和状态提示。

#### 5.2.3 服务与帮助区

- 客服中心
- 帮助文档
- 平台公告
- 关于我们
- 用户协议
- 隐私政策
- 第三方信息共享清单
- 个人信息收集清单

说明：

- 首版只保留已经进入正式 PRD 体系的核心入口。
- `邀请朋友`、`签到`、`活动` 等历史入口不纳入本期正式范围。

#### 5.2.4 底部设置入口

- 设置

### 5.3 配置要求

- 我的页功能入口支持后台排序、启停、文案修改。
- 支持对不同用户态展示不同入口角标：
  - 未认证用户：显示 `去认证`
  - 已认证失败用户：显示 `审核失败`
  - VIP用户：显示 `会员有效期至`
  - 成家币余额不足用户：显示 `去充值`

### 5.4 状态说明


| 状态        | 页面表现                         |
| --------- | ---------------------------- |
| 未登录       | 允许进入我的页，但核心能力点击后先登录          |
| 已登录未完成三认证 | 展示认证引导卡；允许查看设置、帮助、客服、协议等基础服务 |
| 已完成三认证    | 正常展示全部入口                     |
| 账号异常/封禁   | 我的页顶部展示限制提示，部分互动能力置灰         |


## 6. 认证中心

### 6.1 页面目标

- 聚合展示三项认证状态。
- 提供失败重提、审核中说明、认证结果说明。
- 作为 `PRD-01` 的后续使用承接页。

### 6.2 页面结构

- 顶部说明区：认证价值、平台对真实性要求说明、认证权益说明。
- 三项认证卡片：
  - 实名认证
  - 头像认证
  - 学历认证
- 底部帮助区：认证常见问题、联系客服。

认证权益说明需明确告知：

- 三项认证全部通过后，才开放成家核心互动能力。
- 三项认证全部通过后，才开放社区发布、评论、点赞、关注、@ 等真实互动能力。
- 任一认证失效后，相关社交能力即时回退受限。

### 6.3 认证状态


| 状态   | 页面文案        | 按钮   |
| ---- | ----------- | ---- |
| 未提交  | 未完成认证       | 去认证  |
| 审核中  | 预计24小时内完成   | 查看说明 |
| 已通过  | 已认证         | 已完成  |
| 审核失败 | 认证未通过，可重新提交 | 重新认证 |


### 6.4 与其他模块的联动

- 三项全部通过后：
  - 成家相关真实互动能力开放
  - 社区发布/评论/点赞/关注/@ 等能力开放
- 任一认证失效或被撤销后：
  - 立即回退到受限态
  - 已有历史资产和已购权益保留，但受前置门槛约束的社交能力暂停

## 8. 隐私设置

### 8.1 页面目标

- 允许用户控制展示范围、屏蔽关系和隐私偏好。
- 仅保留与当前已确认业务一致的设置项，不沿用总 PRD 中已经过时的旧私信规则。

### 8.2 设置分组

#### 8.2.1 展示与可见范围

- 允许展示距离
- 隐藏活跃时间
- 婚况对外展示
- 资料更新动态开关
- 动态在特定频道展示开关

规则说明：

- 关闭某类资料展示时，用户自己也不能再查看同类对方信息时，需显式提示对等规则。

#### 8.2.2 访问与互动隐私

- 隐藏访问记录
  - VIP 权益
  - 开启后，用户访问他人主页不产生对外可见访客记录
  - 到期后自动关闭
- 只接受异性互动
  - 默认开启
  - 用于前台关系链展示口径一致

#### 8.2.3 屏蔽设置

- 黑名单列表
- 不看TA的动态
- 关键词屏蔽

#### 8.2.4 个性化与推荐提示

- 接收个性化推送
- 个性化配对聊天提示
- 智能灵感回复

说明：

- 由于推荐算法与测评未确定，本期这些开关只作为前台设置项和后端预留，不展开策略细节。
- `只接受异性互动` 首版只约束成家关系链与推荐展示相关场景，不影响公开社区内容浏览。

### 8.3 不纳入本期的旧口径

- `谁可以给我发私信（互关后/VIP可发/所有人）`

说明：

- 该旧规则已与 `PRD-03` 的“普通私信仅在匹配成功后开放”冲突，本期不再保留。

## 9. 黑名单、动态屏蔽与关键词屏蔽

### 9.1 黑名单

#### 9.1.1 功能定义

- 用户可查看自己已拉黑对象列表。
- 支持解除拉黑。

#### 9.1.2 生效规则

- 拉黑后：
  - 双方不可再发送新消息
  - 对方不可继续互动、查看部分资料或继续形成推荐承接
  - 历史聊天记录保留但会话失效，沿用 `PRD-03`
- 解除拉黑后：
  - 不自动恢复历史关系状态
  - 后续是否重新可见由系统正常规则判断

### 9.2 关键词屏蔽

#### 9.2.1 功能定义

- 用户可配置个人不希望在社区动态/评论中看到的关键词。

#### 9.2.2 生效规则

- 命中关键词的动态或评论，对该用户不展示或降权隐藏。
- 不影响原内容真实发布状态。
- 首版仅作用于社区动态与评论，不作用于私信、系统通知与搜索结果。

### 9.3 不看TA动态

- 用户可将指定对象加入“不看TA动态”列表。
- 生效后：
  - 对方动态不再出现在社区信息流中
  - 不影响私信、主页访问、黑名单等其他关系

## 11. 账号安全与设置页

### 11.1 设置页结构

- 手机号绑定
- 微信号绑定
- 隐私设置
- 消息通知
- 软件更新
- 第三方信息共享清单
- 个人信息收集清单
- 关于我们
- 退出登录

### 11.2 账号安全能力

#### 11.2.1 手机号绑定/换绑

- 已绑定手机号展示脱敏号码。
- 支持换绑手机号。
- 触发敏感操作时需短信验证或微信侧安全校验。

#### 11.2.2 微信绑定

- 展示当前绑定状态。
- 已绑定支持更换或重新授权。

#### 11.2.3 退出登录

- 点击后弹二次确认框。
- 退出后清空本地登录态，回到登录前页或首页。

#### 11.2.4 注销账号

- 放在设置页或账号安全二级页中。
- 流程：
  - 用户阅读注销说明
  - 勾选确认
  - 校验是否存在进行中权益或风险状态
  - 提交注销申请
- 注销规则：
  - 进入 30 天后悔期
  - 后悔期内可恢复
  - 到期后执行物理清除且不可恢复

注销前需至少校验以下限制项：

- 存在封禁处罚中的账号不可直接注销
- 存在未完成退款中的账号不可直接注销
- 存在进行中的付费争议不可直接注销
- 存在未到期 VIP 权益的账号需先完成风险提示或按规则处理后再允许注销

### 11.3 软件更新

- 小程序场景不做“应用商店升级”式复杂表达。
- 可保留为：
  - 当前版本号展示
  - 若有强更或能力升级提示，由后台文案配置或版本策略承接

## 13. 搜索模块

### 13.1 页面目标

- 提供统一搜索入口。
- 支持用户、动态、话题的模糊搜索。
- 不依赖推荐算法结果即可独立运行。

### 13.2 入口

- 顶部导航搜索图标
- 社区页搜索入口
- 相关空状态页推荐入口

### 13.3 搜索首页

#### 13.3.1 组成

- 搜索框
- 热门搜索词
- 历史搜索记录
- 清空历史按钮

#### 13.3.2 交互规则

- 点击历史词或热门词可直接发起搜索。
- 输入关键词后点击搜索，进入结果页。
- 历史搜索记录按本地账号维度保存，保留最近 10 条。
- 热门搜索词由后台配置与更新。

### 13.4 搜索结果页

#### 13.4.1 结果 Tab

- 用户
- 动态
- 话题

#### 13.4.2 用户结果

- 展示字段：
  - 头像
  - 昵称
  - 年龄
  - 学校
  - 城市
  - 认证标签
- 点击进入用户主页。
- 若用户资料受隐私限制，则按隐私规则降级展示。

#### 13.4.3 动态结果

- 结果卡片与社区动态卡片保持一致。
- 点击进入动态详情页。

#### 13.4.4 话题结果

- 展示话题名、参与动态数、简述。
- 点击进入话题聚合页。

### 13.5 搜索规则

- 支持模糊匹配。
- 默认按综合相关度排序。
- 本期不展开推荐算法排序，不承诺复杂推荐权重。
- 首版仅保留 `综合相关度 + 后台基础排序` 能力。
- 命中黑名单、屏蔽关系、违规下架内容时，不展示对应结果。

### 13.6 空状态与异常态


| 场景    | 页面表现          |
| ----- | ------------- |
| 无搜索词  | 展示热门词和历史词     |
| 搜索无结果 | 展示空状态文案和返回建议  |
| 搜索词违规 | 提示“搜索内容不支持展示” |
| 网络异常  | 支持重试          |


### 13.7 与现有代码的处理建议

- 当前 `Search.vue` 混入了“最近看过我的”和“理想型搜索”旧逻辑。
- 本期正式 PRD 中，搜索模块应回归“全局搜索”定位，不再承接关系反馈或理想型解锁逻辑。
- 旧混合页面逻辑应视为历史实现参考，不作为正式产品结构继续沿用。

## 15. 已确认结论

本期原待确认项，甲方已全部采纳本文建议，现统一固化如下：

| 已确认事项 | 最终确定 |
| --- | --- |
| 我的页核心入口范围 | 首版只保留已进入正式 PRD 体系的核心入口，`邀请朋友/签到/活动` 等历史入口不纳入本期 |
| 认证中心展示内容 | 首版展示简版权益说明，明确三项认证与成家互动、社区互动的解锁关系 |
| 安全中心治理数据模块 | 首版不做治理公示数据模块 |
| 熟人屏蔽 | 首版不纳入正式范围 |
| 关键词屏蔽作用范围 | 首版仅作用于社区动态与评论，不作用于私信、系统通知和搜索结果 |
| `只接受异性互动` 生效范围 | 首版只约束成家关系链与推荐展示相关场景，不影响公开社区浏览 |
| 注销前校验范围 | 至少校验封禁处罚、未完成退款、进行中的付费争议、未到期 VIP |
| 公告/规则/协议页承接方式 | 首版优先支持后台配置 H5 地址，强合规页面保留原生页兜底 |
| 搜索热门词与历史词 | 热门词后台配置；历史词按账号本地保存最近 10 条 |
| 搜索排序口径 | 首版只保留 `综合相关度 + 后台基础排序`，复杂排序待推荐算法阶段补充 |
