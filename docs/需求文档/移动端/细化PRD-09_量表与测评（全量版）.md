# 成家立业移动端细化 PRD-09

## 1. 文档信息

- 文档主题：量表与测评移动端全量需求
- 关联后台文档：`管理后台细化PRD-09_量表与测评配置（全量版）`
- 参考 demo：`/Users/外快/homeproject/homeformobile`
- 参考页面：`AssessmentList.vue`、`AssessmentDetail.vue`、`ReportList.vue`、`HollandReport.vue`、`JungReport.vue`、`MbtiReport.vue`、`MarriageReports.vue`、`Compatibility.vue`、`MatchUsers.vue`
- 目标：定义用户从看到测评、完成作答、查看报告、复用结果到职业/婚恋推荐的完整链路。

## 2. 模块定位

量表与测评是“成家立业”的用户画像生成模块，承担三类职责：

1. 帮助用户认识自己：人格、认知功能、恋爱风格、心理状态、职业兴趣。
2. 帮助平台生成结构化画像：结果类型、维度分、标签、风险等级、匹配特征。
3. 支撑业务场景复用：心印合拍、觅缘推荐、合拍度分析、个人主页展示、职业推荐、报告分享。

## 3. 场景范围

### 3.1 成家/心印测评

用于恋爱交友和婚恋匹配，结果可进入推荐算法。

首批全量测评包括：

| 测评 | 主要用途 | 是否影响交友推荐 |
| --- | --- | --- |
| 荣格认知功能八维测评 | 生成认知功能分、MBTI/人格倾向、心印标签 | 是 |
| 成人依恋类型测评 | 生成依恋类型、关系安全感、亲密距离特征 | 是 |
| 沟通模式问卷 | 生成冲突沟通模式、积极沟通程度、追逃特征 | 是 |
| 爱情价值观问卷 | 生成爱情态度维度、关系偏好标签 | 是 |

说明：

1. 首批量表名称以 `docs/相关资料/测评题目相关资料/` 现有题库资料为准。
2. `爱情价值观问卷` 的题库正文标题为 `LAS-R 爱情态度量表修订版（23题）`，当前 PRD 先按“资料目录名 + 题库正文标题”并存记录。

### 3.2 立业/职业测评

用于职业探索和个人主页补充展示。

| 测评 | 主要用途 | 是否影响交友推荐 |
| --- | --- | --- |
| 霍兰德职业兴趣测评 | 生成 RIASEC 代码、职业兴趣图谱、推荐岗位 | 否 |
| Super职业价值观测评 | 生成工作价值偏好、核心职业价值排序 | 否 |

### 3.3 心理健康测评

当前提供的测评题库资料中未包含可直接开发的心理健康量表，因此心理健康测评不纳入当前首批量表范围。

若后续甲方补齐心理健康量表题库、计分规则、解释文案、免责声明和高风险处理规则，再单独纳入量表 PRD。

## 5. 测评列表页

### 5.1 页面目标

展示当前场景可用测评，并引导用户完成必要前置测评。

### 5.2 信息结构

每个测评卡片展示：

| 字段 | 说明 |
| --- | --- |
| 测评名称 | 如荣格认知功能八维测评、霍兰德职业兴趣测评 |
| 简介 | 一句话说明测评价值 |
| 预计耗时 | 后台配置，单位分钟 |
| 题目数量 | 后台返回 |
| 状态 | 未开始、进行中、已完成、已过期、未解锁、已下架 |
| 操作按钮 | 开始测评、继续测评、查看报告、重新测评、未解锁 |
| 依赖提示 | 当前测评是否需要先完成其他测评 |

### 5.3 分组规则

1. 心印测评列表优先展示推荐测评，再展示未解锁测评。
2. 职业测评列表展示“事业有成测评”主入口和其他职业测评卡片。
3. 当前首批报告列表按性格、婚恋、职业分类；心理健康分类暂不进入首批。
4. 搜索报告时支持按报告名称、测评名称、结果标签模糊匹配。

### 5.4 状态规则

| 状态 | 展示 | 点击行为 |
| --- | --- | --- |
| 未开始 | 开始测评 | 进入说明页 |
| 进行中 | 继续测评 | 从断点进入作答页 |
| 已完成 | 查看报告 | 进入最新有效报告 |
| 已过期 | 重新测评 | 提示旧报告仍可查看，重测生成新版本 |
| 未解锁 | 未解锁按钮置灰 | 点击可展示解锁条件 |
| 已下架 | 不展示或只在历史报告中展示 | 不允许重新作答 |

## 6. 测评说明页

用户开始正式作答前，展示：

1. 测评名称、简介、预计耗时、题目数量。
2. 答题规则：是否必答、是否可返回上一题、是否逐题解锁、是否允许中断。
3. 结果用途：报告展示、主页展示、推荐算法使用范围。
4. 隐私提示：哪些结果会展示给他人，哪些仅自己可见。
5. 练习题：不计入正式分数，用于熟悉题型。

用户必须完成练习题或勾选知情提示后，才能点击“开始测评”。

## 7. 作答页

### 7.1 支持题型

| 题型 | 交互 |
| --- | --- |
| 单选题 | 点击选项后记录答案，可自动进入下一题或保持当前页 |
| 多选题 | 支持多选，点击下一题前校验最少/最多选择数 |
| 量尺题 | 支持 1-7 分或后台配置范围滑动 |
| 排序题 | 拖拽排序或点击排序 |
| 填空题 | 输入文本，支持字数限制 |
| 说明题 | 仅展示，不计分 |

### 7.2 分页与进度

1. 后台可配置每页题数。
2. 顶部展示页码和总进度。
3. 页面内可逐题解锁，也可一次展示整页题目，由后台配置。
4. 点击下一页时校验当前页必答题。
5. 漏答时提示题号，并滚动到第一个未完成题目。

### 7.3 断点续答

1. 每次答题后自动保存答案、当前页、已解锁题目、反应时。
2. 用户退出后再次进入，展示“继续测评”。
3. 服务端保存进度，客户端本地可做临时缓存。
4. 测评下架后，进行中的答题不可继续，提示“测评已更新，请重新开始”。

### 7.4 反应时

记录每题从展示/解锁到首次作答的时间，单位毫秒。反应时用于：

1. 结果置信度计算。
2. 异常作答识别。
3. 运营分析和信效度优化。

## 8. 提交与结果生成

### 8.1 提交流程

```text
用户点击提交
  -> 客户端校验必答/题型规则
  -> 提交答案、反应时、量表版本
  -> 服务端计算维度分、总分、类型、标签、置信度
  -> 生成报告
  -> 回写用户画像和推荐特征
  -> 返回报告 ID
  -> 客户端进入报告页
```

### 8.2 异常处理

| 场景 | 处理 |
| --- | --- |
| 网络失败 | 保留本地答案，提示重试 |
| 公式计算失败 | 提示“报告生成中”，进入报告列表显示处理中 |
| 测评版本失效 | 提示重新开始或查看旧报告 |
| 重复提交 | 服务端幂等处理，返回同一报告 |
| 答案不完整 | 客户端阻断提交，服务端二次校验 |

## 9. 报告页

### 9.1 通用报告结构

| 模块 | 说明 |
| --- | --- |
| 报告头部 | 测评名称、完成时间、结果类型、核心标签 |
| 维度图表 | 雷达图、条形图、分布图等 |
| 结果解释 | 结果类型描述、优势、可能挑战 |
| 细分维度 | 每个维度分数、等级、解释 |
| 发展建议 | 行动建议、关系建议、职业建议 |
| 推荐模块 | 合拍的人、职业机会、相关课程或服务 |
| 隐私与操作 | 隐藏/展示、重新测试、分享、保存海报 |

### 9.2 性格/心印报告

包含：

1. MBTI 或人格类型。
2. 荣格八维分数：Se、Si、Te、Ti、Fe、Fi、Ne、Ni。
3. 主导/辅助/第三/劣势功能解释。
4. 性格标签。
5. 恋爱中的优势与风险。
6. “寻找合拍的人”入口。
7. “合拍度分析”入口。

### 9.3 婚恋报告

包含：

1. 依恋类型。
2. 沟通模式六维结果。
3. 爱情价值观问卷结果，含爱情态度维度或主导类型。
4. 沟通风格和冲突处理建议。
5. 与候选用户的匹配理由。

### 9.4 职业报告

包含：

1. 霍兰德 RIASEC 六维分数和三字码。
2. 职业兴趣解释。
3. Super职业价值观 15 维结果与排序靠前维度。
4. 推荐岗位/行业。
5. 职业优势、风险和发展建议。
6. 职业机会入口。

## 10. 合拍度分析

### 10.1 入口

1. 用户主页 MBTI 卡片。
2. 觅缘卡片“揭秘你们的合拍度”。
3. 报告页“寻找合拍的人”。
4. 合拍的人列表。

### 10.2 前置条件

1. 当前用户和对方都完成必要心印测评时，直接进入合拍度分析页。
2. 任一方未完成时，引导当前用户先完成测评。
3. 对方隐藏结果时，只展示综合分和可公开理由，不展示详细维度。

### 10.3 页面内容

1. 双方头像和综合合拍度百分比。
2. MBTI/荣格适配度。
3. 依恋风格适配度。
4. 沟通模式适配度、爱情价值观/爱情态度一致性。
5. 共同点、互补点、潜在摩擦点。
6. 互动建议，例如如何开场、如何沟通。

## 11. 合拍的人

### 11.1 列表规则

展示基于测评结果和基础资料生成的候选用户，默认按合拍度排序。

卡片展示：

1. 头像、昵称、年龄、城市、学校/职业。
2. 合拍度分数。
3. MBTI 或人格标签。
4. 匹配理由标签。
5. 查看主页入口。

### 11.2 筛选与排序

支持：

1. MBTI 类型筛选。
2. 合拍度最高。
3. 年龄相近。
4. 距离最近。
5. 后续可扩展为依恋类型、价值观、兴趣标签筛选。

### 11.3 解锁与付费

若合拍的人属于付费场景：

1. 普通用户头像或主页入口可模糊展示。
2. 点击触发成家币/VIP 解锁。
3. 解锁成功后可查看主页和详细合拍分析。
4. 已解锁记录进入历史，可重复查看。

## 12. 个人主页复用

测评结果可在个人主页展示：

1. MBTI 类型。
2. 性格标签。
3. 职业兴趣标签。
4. 报告卡片。
5. 与访问者的合拍度入口。

用户可在报告页或隐私设置中控制展示状态。展示关闭只影响他人可见性，是否影响推荐算法需由后台策略明确。

## 13. 推荐算法联动

### 13.1 输入字段

| 字段 | 来源 | 用途 |
| --- | --- | --- |
| `mbtiType` | 荣格/MBTI 结果 | 内容过滤、互惠排序、推荐解释 |
| `jungFunctionScores` | 荣格八维 | 认知功能相似/互补计算 |
| `attachmentType` | 成人依恋类型测评 | 依恋适配 |
| `communicationPatternScores` | 沟通模式问卷 | 冲突沟通模式适配 |
| `loveValueTags` | 爱情价值观问卷 | 爱情态度/关系偏好一致性 |
| `hollandCode` | 霍兰德职业测评 | 职业推荐，不默认进入交友推荐 |
| `workValueTopDimensions` | Super职业价值观测评 | 职业推荐和主页补充展示，不默认进入交友推荐 |

### 13.2 输出字段

推荐结果可回传：

1. 综合合拍度。
2. 各维度适配分。
3. 推荐理由标签。
4. 使用的匹配方案代码。
5. 是否需要补测才能提升推荐质量。

## 16. 权限与隐私

1. 用户必须登录后才能保存进度、提交答案和查看报告。
2. 报告默认仅自己可见，是否默认展示到主页由后台配置。
3. 用户隐藏报告后，主页不展示报告卡片和详细标签。
4. 推荐算法使用测评结果时，应在隐私政策或测评说明中明示。
