# 成家立业移动端细化 PRD-04

## 1. 文档信息

- 文档主题：商业化（VIP、成家币、解锁与资产中心）
- 对应原始关联模块：
  - `VIP 会员中心`
  - `成家币充值`
  - `资产明细`
  - `订单记录`
  - 各业务场景中的付费触点
- 关联前置文档：
  - `细化PRD-01_用户准入与资料认证初始化`
  - `细化PRD-02_关系反馈与互动链路`
  - `细化PRD-03_消息、私信与通知中心`
- 目标：输出一份可直接支撑产品设计、前后端开发、联调、测试的细化 PRD

## 2. 模块定位

本模块负责承接移动端所有核心商业化能力，把分散在 `成家`、`消息`、`我的` 中的付费行为统一收口为：

1. `VIP`：
   承接高频权益、全量解锁、社交提效能力。
2. `成家币`：
   承接按次消耗型能力和低门槛付费动作。
3. `资产中心`：
   承接余额、权益状态、订单记录、消费流水的统一查看。

它承担三类职责：

1. 转化职责：
   在配额不足、功能受限、信息模糊时承接付费转化。
2. 权益职责：
   统一管理 VIP 和成家币在各业务场景中的生效规则。
3. 资产职责：
   让用户可查询、可追溯自己的充值、消费和权益状态。

## 3. 设计原则

1. 低门槛和高权益分层：按次型能力优先走 `成家币`，持续型能力优先走 `VIP`。
2. 规则统一：所有单条解锁、全量解锁、悄悄话、配额扩容等规则在本模块统一定义。
3. 前置场景触发，中心页承接：业务场景内负责触发，具体购买与资产查看由本模块统一承接。
4. 实时生效：充值成功、开通成功后，页面权益状态立即刷新。
5. 到期可回退：VIP 到期后回退普通态；单条或按次购买结果按各业务场景的保留规则继续生效。
6. 与推荐解耦：本模块不展开推荐算法，只定义其商业化触发点与结算规则。
7. 展示包装参考成熟竞品：页面结构、档位分层、标签体系可参考青藤之恋，但价格与最终权益以本项目配置为准。

## 4. 页面范围

本模块涉及以下页面：

1. `VIP 会员中心页`
2. `成家币充值页`
3. `资产中心页`
4. `成家币流水页`
5. `VIP 订单记录页`
6. `支付结果承接页/弹窗`
7. `业务场景付费引导弹窗`
8. `连续订阅说明区`

## 5. 与 PRD-01、PRD-02、PRD-03 的关系

本模块严格继承以下已确认规则：

1. `实名认证 + 头像认证 + 学历认证` 三项全部通过后，才开放真实曝光、匹配、关系反馈和真实聊天链路。
2. `喜欢我的` 与 `最近看过我的`：
   - 单条解锁走 `成家币`
   - 解锁全部走 `VIP`
   - 不做 `成家币解锁全部`
3. VIP 生效期间：
   - 查看全部清晰记录
   - 到期后回退普通态
   - 已单条购买的记录永久可见
4. `悄悄话`：
   - 普通用户可直接使用成家币发送
   - VIP 默认每日享 `1` 次免费悄悄话，具体次数后台可配
   - 免费次数用完后也可继续使用成家币发送
5. 通知中心已确认纳入：
   - 充值成功
   - 成家币变动
   - VIP 到期提醒

补充明确：

1. 商业化模块对所有已登录用户可见。
2. 未完成三项认证的用户可查看和购买资产类商品，但相关社交权益仅在满足前置条件后实际可用。
3. 本模块只定义微信小程序内的商业化规则，不展开 Web、App、线下礼包等其他渠道。

参考补充：

1. 参考 `青藤之恋` 调研结论，本模块建议吸收其成熟的商业化页面结构：
   - 成家币页采用 `四档套餐卡片 + 标签/优惠说明`
   - VIP 页采用 `普通套餐 / 连续订阅套餐` 分层展示
   - 套餐卡支持 `尝新首选 / 最多人选 / 节省最多 / 热销 / 推荐` 等标签位
2. 上述仅作为产品结构参考，不直接继承竞品价格。

## 6. 用户流程总览

## 6.1 VIP 开通链路

1. 用户在任一业务场景命中 `VIP` 触点
2. 弹出 `VIP 引导弹窗`
3. 点击 `开通 VIP`
4. 进入 `VIP 会员中心页`
5. 选择套餐并发起支付
6. 支付成功后：
   - VIP 状态立即生效
   - 回跳来源页或权益中心
   - 相关页面立即刷新可用权益

## 6.2 成家币充值链路

1. 用户在任一按次消费场景命中 `成家币` 触点
2. 弹出 `成家币充值引导`
3. 点击充值或直接选择套餐
4. 进入 `成家币充值页`
5. 完成支付
6. 支付成功后：
   - 余额立即刷新
   - 返回原业务场景或保留当前上下文，支持用户继续当前动作

## 6.3 单条解锁链路

1. 用户点击某条模糊记录
2. 弹出 `单条解锁弹窗`
3. 若余额充足：
   - 确认支付成家币
4. 若余额不足：
   - 跳转充值承接
5. 支付成功后：
   - 当前记录变清晰
   - 永久保留该单条解锁结果

## 6.4 资产查询链路

1. 用户进入 `我的`
2. 点击：
   - `会员中心`
   - `我的成家币`
   - `资产明细`
3. 查看当前 VIP 状态、成家币余额、订单记录、流水明细

## 9. 核心状态与规则

## 9.1 商业化入口规则

首版主要商业化入口包括：

1. `喜欢我的`：
   - 单条解锁
   - 解锁全部
2. `最近看过我的`：
   - 单条解锁
   - 解锁全部
3. `悄悄话`
4. `觅缘配额不足`
5. `高级筛选`
6. `理想型用户解锁`
7. `精选主页解锁`
8. `我的 -> 会员中心 / 我的成家币 / 资产明细`

## 9.2 VIP 权益规则

首版 VIP 权益统一承接以下能力：

1. 每日额外查看配额
2. 查看 `喜欢我的` 全部高清信息
3. 查看 `最近看过我的` 全部高清信息
4. 每日免费悄悄话次数
5. 高级筛选解锁
6. 隐藏访问记录
7. 其他后台可配置权益

说明：

1. VIP 具体权益展示顺序、开关、文案、是否生效均由后台配置。
2. 首版默认推荐展示以上权益，但不把单一权益列表写死为不可调整的交付边界。
3. 查看配额默认口径为：普通用户每日 `10` 次，VIP 每日额外增加 `10` 次，总计 `20` 次，具体数值后台可配。
4. VIP 免费悄悄话默认口径为每日 `1` 次，具体次数后台可配。

## 9.3 成家币规则

首版成家币承接以下按次消费场景：

1. 发悄悄话
2. 解锁 `喜欢我的` 单条记录
3. 解锁 `最近看过我的` 单条记录
4. 解锁理想型单个用户
5. 批量解锁理想型用户
6. 解锁精选用户主页

规则说明：

1. 成家币余额不足时，不允许透支
2. 余额不足时弹出 `成家币套餐购买弹窗`，默认直接展示常用套餐
3. 需先充值后消费
4. 消耗单价由后台配置
5. 所有成家币，包括充值所得、赠送所得、退款退回，均不过期
6. 用户侧不拆分充值币与赠送币双余额池，只展示一个总余额；后台流水区分来源即可

## 9.4 解锁规则

已确认规则如下：

1. `喜欢我的`：
   - 单条解锁走成家币
   - 解锁全部走 VIP
2. `最近看过我的`：
   - 单条解锁走成家币
   - 解锁全部走 VIP
3. 不做 `成家币解锁全部`
4. `喜欢我的` 与 `最近看过我的` 解锁状态独立
5. 单条解锁不自动影响另一模块
6. 已单条购买的记录永久可见
7. 用户开通 VIP 后：
   - VIP 期间全部清晰可见
8. VIP 到期后：
   - `喜欢我的` 与 `最近看过我的` 相关全量权益即时回退普通态
   - 高级筛选、隐藏访问记录、额外配额即时失效
   - 但已单条购买的记录继续保持清晰
9. `理想型`：
   - 走成家币解锁
   - 支持单个解锁，也支持一次勾选最多 `5` 个批量解锁
   - VIP 用户解锁理想型也照常消耗成家币，不额外免单
   - 解锁后保留 `90` 天，后台可配置
10. `精选`：
   - 解锁主页走成家币
   - 解锁后可回看 `3` 天，后台可配置

## 9.5 悄悄话付费规则

已确认规则如下：

1. 普通用户允许直接使用成家币发送悄悄话
2. VIP 用户默认每日享 `1` 次免费悄悄话，具体次数后台可配
3. 免费次数用完后，VIP 也可继续使用成家币发送
4. 同一目标用户在上一条悄悄话未被回复或未失效前，不允许重复发送新的悄悄话
5. 对方忽略后，发送方 `7 天内` 不可再次向同一用户发送悄悄话

## 9.6 生效与到期规则

1. 支付成功后，权益立即生效
2. VIP 到期后，回退为普通用户态
3. VIP 到期不影响：
   - 已单条解锁记录
   - 已充值到账成家币余额
4. VIP 到期后即时失效的能力包括：
   - 额外查看配额
   - 查看喜欢我的全部高清信息
   - 查看最近看过我的全部高清信息
   - 高级筛选
   - 隐藏访问记录
5. 所有成家币到账后不过期
6. 通知信息一直保留，资产流水长期保留

## 9.7 竞品参考吸收原则

基于 `青藤之恋` 调研结论，首版建议吸收以下成熟表达方式：

1. 成家币页：
   - 默认 `4` 档套餐卡片
   - 每张卡片支持 `标签 + 折扣/权益说明 + 金额`
   - 默认高亮一个推荐档位
2. VIP 页：
   - 支持 `普通套餐` 与 `连续订阅套餐` 分层展示
   - 每个套餐同时展示 `总价` 与 `折算月价`
   - 连续订阅需额外展示自动续费说明
3. 更多页入口：
   - `成家会员` 与 `成家币` 作为独立入口长期保留
4. 人气曝光包：
   - 需要纳入整体商业化规划
   - 当前先只保留产品预留口径，不展开详细规则
   - 需待推荐规则完善后，再细化 `觅缘` 与 `社区动态` 两个场景的曝光售卖与生效逻辑

## 9.8 支付结果规则

1. 支付成功：
   - 展示成功反馈
   - 刷新权益或余额
   - 允许回跳来源页继续动作
2. 支付失败：
   - 保留当前页
   - 提示失败原因或通用失败文案
3. 支付取消：
   - 不变更资产
   - 订单在倒计时后关闭，状态记为 `closed`
   - 可停留当前页，允许再次发起支付
4. 特批退款：
   - 前台不开放主动退款申请入口
   - 订单记录中支持展示 `refunding` / `refunded` 状态

## 10. 页面级详细需求

## 10.1 VIP 会员中心页

### 10.1.1 页面目标

集中展示 VIP 套餐、权益说明、当前开通状态，并承接会员购买。

### 10.1.2 页面入口

1. `我的 -> 会员中心`
2. 业务场景内 `开通 VIP` 弹窗
3. 配额耗尽引导
4. 解锁全部引导
5. 高级筛选引导

### 10.1.3 页面结构

1. 顶部状态区：
   - 当前 VIP 状态
   - 到期时间
2. 权益列表区
3. 套餐选择区
4. 支付按钮区
5. 套餐类型切换区

1. 未开通：
   - 展示 `立即开通 VIP`
2. 已开通：
   - 展示 `VIP 有效至 XXXX-XX-XX`
3. 已过期：
   - 展示 `已过期，续费可继续享受权益`

### 10.1.5 权益展示

首版默认展示，且后台可配置启停、顺序和文案：

1. 每日额外查看配额
2. 查看喜欢我的全部高清头像
3. 查看所有访客记录高清头像
4. 每日免费悄悄话次数
5. 高级筛选全部解锁
6. 隐藏访问记录

补充口径：

1. 默认查看配额：普通用户每日 `10` 次，VIP 每日 `20` 次，后台可配置
2. 默认免费悄悄话：每日 `1` 次，后台可配置

### 10.1.6 套餐区

1. 月卡
2. 季卡
3. 年卡
4. 支持 `推荐`、`热销` 标签

1. `普通套餐`：
   - 月卡 / 季卡 / 年卡
2. `连续订阅套餐`：
   - 连续包月 / 连续包季 / 连续包年
3. 每个套餐展示：
   - 总价
   - 折算月价
   - 标签
   - 相比原价的优惠说明

### 10.1.7 支付按钮

1. 按当前选中套餐展示价格
2. 支持展示协议勾选提示
3. 点击后拉起微信支付

协议提示：

1. 普通套餐需勾选 `会员协议`
2. 连续订阅套餐需勾选：
   - `会员协议`
   - `连续订阅服务协议`

## 14. 异常与边界场景

1. 用户支付成功后立即返回来源页：
   权益和余额需实时刷新
2. 用户支付取消：
   订单倒计时后关闭为 `closed`，资产不变
3. 用户 VIP 到期：
   回退普通态，但已单条购买记录继续按各场景保留规则生效
4. 用户先单条解锁后又开通 VIP：
   VIP 期间全部可见，到期后已单条购买记录继续可见
5. 用户成家币不足：
   不允许透支，弹出成家币套餐购买弹窗
6. 用户未完成三项认证但已购买 VIP：
   资产状态正常生效，但涉及真实曝光、匹配、聊天等权益需等满足前置条件后实际可用；会员页购买前和购买成功后均需明确提示该限制
7. 悄悄话免费次数用完：
   VIP 用户继续按成家币规则发送
8. 通知信息一直保留：
   资产、充值、到期提醒可长期回看
9. 连续订阅套餐：
   需支持展示续费说明与取消续费提示，但具体订阅扣费细则以后端支付接入规范为准
10. 理想型批量解锁：
   单次最多支持勾选 `5` 个用户一起扣币解锁
11. 精选主页解锁：
   解锁后回看 `3` 天，超时后需重新解锁
12. 人气曝光包：
   已纳入后续商业化规划，但当前 PRD-04 不展开推荐分发、曝光次数和结算细则
