# 成家立业移动端细化 PRD-05

## 1. 文档信息

- 文档主题：推荐模块（朋友、社区与内容互动）
- 对应原始关联模块：
  - `推荐（朋友）`
  - `推荐（社区）`
  - `社区动态`
  - `关注/粉丝`
- 关联前置文档：
  - `细化PRD-01_用户准入与资料认证初始化`
  - `细化PRD-03_消息、私信与通知中心`
  - `细化PRD-04_商业化（VIP、成家币、解锁与资产中心）`
- 目标：输出一份可直接支撑产品设计、前后端开发、联调、测试的细化 PRD

## 2. 模块定位

本模块负责承接 `推荐` 一级导航下的非算法社交与内容场景，把分散在 `朋友`、`社区`、`用户主页动态` 中的内容消费和轻互动统一收口为：

1. `朋友场景`：
   承接弱关系兴趣社交，包括 `觅知音`、`悦目`、`诚意贴`。
2. `社区场景`：
   承接公开图文内容、话题参与、点赞评论、举报治理。
3. `关系延伸场景`：
   承接关注/粉丝、个人动态展示、内容驱动的主页访问。

它承担三类职责：

1. 内容消费职责：
   提供公开图文与兴趣内容浏览，增强留存与活跃。
2. 轻社交职责：
   通过关注、点赞、评论、话题参与建立弱关系连接。
3. 风险隔离职责：
   在不突破前序 PRD 约束的前提下，隔离内容互动与恋爱匹配、私信链路。

## 3. 设计原则

1. 与匹配解耦：本模块不展开推荐算法排序逻辑，只定义前台内容形态、互动规则和审核链路。
2. 浏览优先、重互动慎开放：优先保证公开内容浏览，发布和高风险互动遵循认证门槛。
3. 公开内容与私域聊天隔离：关注、点赞、评论不直接等价于私信开口条件。
4. 审核前置：发布、评论、举报、处罚都必须纳入审核或风控链路。
5. 配置优先：话题、举报原因、空态文案、发帖限制、联系方式开关等均支持后台配置。
6. 机审优先：所有 UGC 文本与图片内容，首版先对接微信小程序官方内容安全能力，再进入人工抽检或人工审核链路。

## 5. 与 PRD-01、PRD-03、PRD-04 的关系

本模块严格继承以下已确认规则：

1. 未完成三项认证前：
   - 不开放社区动态发布
   - 不进入恋爱匹配与真实聊天链路
2. 普通私信仅在 `匹配成功后` 开放，本模块中的 `关注` 不直接解锁私信。
3. `悄悄话`、`VIP`、`成家币` 规则仍以后续消息和商业化模块为准，本模块不新增平行私信规则。
4. `充值成功 / 成家币变动 / VIP 到期提醒` 已在通知模块确认，本模块补充承接 `点赞 / 评论 / 关注 / @ / 审核结果` 等内容互动类通知。

补充明确：

1. 推荐与社区内容浏览对已登录用户开放。
2. 发布动态、发布诚意贴、发表评论、@用户、关注他人、点赞等互动动作，首版按 `三项认证全部通过后开放` 设计。
3. 举报动作允许已登录用户发起，用于提前形成安全治理闭环。

## 6. 用户流程总览

## 6.1 推荐首页浏览链路

1. 用户点击底部 `推荐` Tab
2. 进入推荐首页容器
3. 顶部切换：
   - `朋友`
   - `社区`
4. 用户按卡片流或列表流浏览内容
5. 点击头像或内容卡片进入详情页或用户主页

## 6.2 内容发布链路

1. 用户点击 `发布`
2. 系统校验登录状态与三项认证状态
3. 进入发布页填写文本、图片、话题、@用户
4. 提交后进入审核判断：
   - 机审拦截直接驳回
   - 机审通过后进入人工审核
   - 人工审核通过后才公开
5. 发布结果通过站内通知反馈

## 6.3 内容互动链路

1. 用户在列表页或详情页执行：
   - 点赞
   - 评论
   - 关注
   - 举报
2. 系统校验互动资格
3. 成功后更新前台状态
4. 对被互动方生成对应通知

## 7. 功能边界

### 7.1 本模块产出

1. 朋友内容浏览
2. 社区动态浏览
3. 动态与诚意贴发布
4. 点赞、评论、关注、举报
5. 话题参与

## 9. 核心状态与规则

## 9.1 准入与互动规则

1. 已登录用户可浏览推荐与社区公开内容。
2. `发布动态`、`发布诚意贴`、`评论`、`关注`、`@用户`、`点赞` 首版统一要求：
   `实名认证 + 头像认证 + 学历认证` 三项全部通过。
3. 举报动作对已登录用户开放。
4. 本模块不提供“因为互相关注而直接开私信”的新规则。
5. 从推荐/社区进入用户主页，只承接主页浏览，不改变恋爱匹配或私信规则。

## 9.2 内容状态规则

1. 动态 / 诚意贴状态包括：
   - `pending`
   - `published`
   - `rejected`
   - `deleted`
   - `blocked`
2. 评论状态包括：
   - `pending`
   - `published`
   - `rejected`
   - `deleted`
   - `blocked`
3. 用户主动删除后：
   - 前台立即不可见
   - 后台保留记录供审核与申诉
4. 被举报并核实违规后：
   - 内容可转为 `blocked`
   - 同步触发违规通知或处罚链路

## 9.3 审核规则

### 9.3.1 微信小程序官方机审能力接入

首版内容机审需优先对接微信小程序官方服务端内容安全能力：

1. 文本内容：
   使用官方 `文本内容安全识别 /wxa/msg_sec_check`
2. 图片内容：
   使用官方 `多媒体内容安全识别 /wxa/media_check_async`
3. 接入方式：
   - 由服务端统一调用
   - 前端只提交内容，不直接调用微信安全接口
4. 适用范围：
   - 社区动态正文
   - 诚意贴标题
   - 诚意贴正文
   - 评论内容
   - 动态图片
   - 诚意贴图片

说明：

1. 微信官方机审只作为首层安全校验，不替代平台自身的人工审核、人工抽检与处罚流程。
2. 如微信官方接口异常、超时或返回不确定状态，内容应进入 `待人工复核`，不直接公开。

### 9.3.2 按内容类型的审核流程

1. 发布动态：
   - 先走微信官方机审
   - 文本先校验 `msg_sec_check`
   - 图片异步校验 `media_check_async`
   - 任一校验不通过则直接驳回
   - 全部通过后进入人工审核队列
   - 仅人工审核通过后公开
2. 诚意贴：
   - 先走微信官方机审
   - 文本与图片均通过后，进入人工审核队列
   - 仅人工审核通过后公开
3. 评论：
   - 先走微信官方文本机审
   - 通过后前台展示
   - 不通过则直接驳回
4. 审核不通过时：
   - 通过通知中心反馈驳回结果
   - 支持用户修改后重新提交
5. 举报后的复核：
   - 已公开内容被举报后，可再次进入人工复核
   - 必要时触发下架、封禁、通知与处罚链路

## 9.4 互动规则

1. 点赞：
   - 需完成三项认证后才可操作
   - 单用户对单条内容仅可点赞一次
   - 再次点击取消点赞
2. 评论：
   - 支持多级评论
   - 支持删除自己的评论
   - 评论内容需通过机审
3. 关注：
   - 单向关注
   - 互相关注只生成关系标识，不自动开放私信
4. 举报：
   - 举报后提示 `已提交审核`
   - 审核结果通过通知中心返回

## 9.5 内容类型规则

### 9.5.1 觅知音

1. 基于兴趣标签、学校、资料标签展示用户列表
2. 展示内容以人卡+摘要为主
3. 点击头像或卡片进入用户主页

### 9.5.2 悦目

1. 双列瀑布流展示公开动态首图
2. 卡片展示：
   - 图片
   - 作者头像
   - 点赞数
3. 点击进入动态详情页

### 9.5.3 诚意贴

1. 必选话题
2. 必填标题
3. 正文不少于 `20` 字
4. 图片最多 `9` 张
5. 是否允许留联系方式由后台开关控制
6. 首版默认不允许留联系方式，后台可开启
7. 首版公开前需先完成微信官方机审
8. 微信机审通过后，再进入人工审核并通过后公开

### 9.5.4 社区动态

1. 文本最多 `500` 字
2. 图片最多 `9` 张
3. 必选 `1` 个话题
4. 支持 `@用户`
5. 发布后先走微信官方机审
6. 机审通过后进入人工审核，审核通过后公开
7. `@用户` 仅支持在发布动态时使用，每条最多 `5` 人，并向被 @ 用户发送通知

## 9.6 通知规则

本模块新增承接以下通知类型：

1. 收到点赞
2. 收到评论
3. 收到评论回复
4. 收到新关注
5. 被 @ 提及
6. 动态审核结果
7. 诚意贴审核结果
8. 举报处理结果

说明：

1. 以上通知类型正式纳入 `消息、私信与通知中心`。
2. 是否展示为独立分组、是否计入总红点，由通知中心统一承接。

## 13. 异常与边界场景

1. 未完成三项认证用户点击发布：
   提示先完成认证，不进入发布链路
2. 用户内容微信官方机审未通过：
   内容不公开，通知中心反馈驳回原因
3. 用户内容机审接口异常：
   内容进入待人工复核，不直接公开
4. 用户删除动态：
   前台立即不可见，后台保留审计记录
5. 被举报内容核实违规：
   内容下架，并按处罚规则处理作者
6. 用户互相关注：
   只更新关系状态，不自动开放私信
7. 评论作者被封禁：
   历史评论按后台规则处理，新增评论不可再发送
8. 话题被下线：
   历史内容保留，但话题入口不再允许新增参与
9. 已发布动态尝试编辑：
   前台不提供编辑入口，仅支持删除

## 14. 机审能力调研结论

结合微信小程序官方文档，首版可明确写入的官方平台能力如下：

1. 微信小程序服务端提供 `文本内容安全识别 /wxa/msg_sec_check`
2. 微信小程序服务端提供 `多媒体内容安全识别 /wxa/media_check_async`
3. 因此本项目中的动态、诚意贴、评论审核，应统一采用：
   - 文本先走官方文本安全识别
   - 图片先走官方多媒体安全识别
   - 再进入业务侧人工抽检 / 人工审核 / 举报复核流程

## 17. 已确认结论

| 已确认事项 | 最终确定 |
| --- | --- |
| 推荐与社区模块浏览准入 | 用户登录后即可浏览推荐和社区公开内容；未完成三项认证不可发布、点赞、评论 |
| 发布、评论、关注、@用户门槛 | 必须完成 `实名认证 + 头像认证 + 学历认证` 三项认证后才可操作 |
| 点赞门槛 | 点赞同样必须完成三项认证 |
| 互相关注与私信关系 | 互相关注不开放私信；所有私聊仅在匹配成功后开放 |
| 社区首页菜单结构 | 社区页面采用 `关注 / 同城 / 发现` 菜单结构，并支持后台配置 |
| 内容公开审核门槛 | 动态和诚意贴都需先通过机审，再经人工审核后才公开 |
| 动态编辑规则 | 动态发布后不可编辑，只可删除；发布前可编辑文字、图片和话题 |
| 评论能力范围 | 支持多级评论，支持删除自己的评论 |
| @用户能力边界 | 仅发布动态时支持 @ 用户；每条最多 @5 人；被 @ 用户收到通知 |
| 话题体系来源 | 话题统一由后台配置，引导用户使用话题 |
| 诚意贴联系方式 | 默认不允许留联系方式 |
| 互动通知承接 | 点赞、评论、评论回复、关注、@、动态审核结果、诚意贴审核结果、举报处理结果统一纳入消息中心 |
