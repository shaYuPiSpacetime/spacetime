# 成家立业移动端细化 PRD-02

## 1. 文档信息

- 文档主题：关系反馈与互动链路
- 合并范围：
  - 喜欢我的
  - 最近看过我的
  - 相互喜欢
  - 匹配成功反馈
  - 单条解锁与全量解锁
  - 悄悄话入口边界
- 对应原始关联模块：
  - 成家模块 `喜欢我的`
  - 成家模块 `最近看过我的`
  - 成家模块隐含 `相互喜欢/匹配成功`
  - 消息模块中的前置社交反馈入口
- 目标：输出一份可直接支撑产品设计、前后端开发、联调、测试的细化 PRD

## 2. 模块定位

该模块是 `用户准入与资料认证初始化` 之后的第一条核心反馈链路，用于让用户感知：

1. 谁对我产生了兴趣
2. 谁看过我
3. 我是否已经形成双向喜欢
4. 我是否可以进入进一步互动

它承担两类职责：

1. 社交反馈职责：
   让用户明确知道自己的资料已被看见、被喜欢、被回应。
2. 商业化承接职责：
   通过模糊展示、单条解锁、VIP 解锁等方式承接付费转化。

## 3. 设计原则

1. 先反馈，后互动：先让用户感知“有人喜欢我/看过我”，再承接主页查看、悄悄话、聊天等动作。
2. 认证先行：未完成实名认证的用户，不进入真实反馈链路。
3. 模糊分层：普通用户可以感知有反馈，但完整身份信息和高频使用能力需通过 VIP 或成家币解锁。
4. 链路闭环：`喜欢 -> 喜欢我的 -> 相互喜欢 -> 聊天`，形成完整路径。
5. 与推荐解耦：本模块只定义反馈展示、解锁、互动规则，不展开推荐算法。
6. 与消息解耦：本模块只定义进入消息前的社交反馈边界，完整消息会话规则放在下一份 PRD 中展开。

## 4. 页面范围

本模块涉及以下页面：

1. `喜欢我的列表页`
2. `最近看过我的列表页`
3. `相互喜欢列表页`
4. `匹配成功弹窗/页面`
5. `单条解锁弹窗`
6. `VIP 引导弹窗`
7. `成家币充值弹窗`
8. `用户主页（作为跳转承接页）`

## 5. 与 PRD-01 的关系

本模块严格继承 PRD-01 已确认规则：

1. 未完成 `实名认证 + 头像认证 + 学历认证` 任一项的用户，不进入本模块反馈链路。
2. 未完成三项认证任一项的用户，不进入 `喜欢我的`、`最近看过我的`、`相互喜欢/匹配成功`。
3. 未完成三项认证任一项的用户，不形成真实喜欢关系，不开放聊天链路。
4. 仅当 `实名认证 + 头像认证 + 学历认证` 全部通过后，用户的喜欢行为才真实生效，并可能进入反馈链路。

补充说明：

1. 当前模块已确认：`实名、头像、学历三项必须认证` 才可进入关系反馈链路。
2. 账号状态异常（冻结、注销中、已注销）时，本模块不可用。

## 6. 用户流程总览

## 6.1 喜欢反馈链路

1. 用户 A 在成家模块中对用户 B 执行 `喜欢`
2. 若 A、B 均满足准入条件，则该喜欢关系真实写入
3. 用户 B 的 `喜欢我的` 列表新增一条记录
4. 若 B 也喜欢 A，或双方满足其他匹配成功条件，则双方进入 `相互喜欢/匹配成功`
5. 双方收到匹配成功反馈，并可进一步进入聊天链路

## 6.2 来访反馈链路

1. 用户 A 进入用户 B 主页
2. 若满足访客记录规则，则为 B 生成一条来访记录
3. B 在 `最近看过我的` 中看到该记录
4. 普通用户看到模糊信息，VIP 或付费后看到完整信息

## 6.3 解锁链路

1. 用户进入 `喜欢我的` 或 `最近看过我的`
2. 点击单个模糊用户
3. 弹出单条解锁弹窗
4. 用户选择：
   - 只解锁 ta
   - 解锁全部
   - 开通 VIP
5. 支付成功后更新对应记录可见状态

## 6.4 相互喜欢链路

1. 双方真实喜欢成立
2. 系统生成 `相互喜欢/匹配成功` 记录
3. 前台当次喜欢操作触发双向喜欢时，立即弹一次匹配成功弹窗；同一对用户仅首次匹配成功时弹出。
4. 用户可选择：
   - 去看对方主页
   - 进入聊天
5. 完整聊天规则在下一份 `消息、私信与通知中心` 文档定义

## 9. 核心状态与规则

## 9.1 喜欢记录生效规则

喜欢行为真实生效前提：

1. 发起方已完成实名认证、头像认证、学历认证
2. 接收方已完成实名认证、头像认证、学历认证
3. 双方账号状态正常
4. 双方不在彼此黑名单中

不生效场景：

1. 任一方未完成三项认证中的任一项
2. 任一方账号冻结/注销中/已注销
3. 已被拉黑/屏蔽
4. 重复喜欢幂等去重后不新增记录

## 9.2 来访记录生成规则

生成前提：

1. 访问的是真实用户主页
2. 访问方和被访问方都为正常可用账号
3. 访问行为未被配置排除

首版建议规则：

1. 进入对方主页即生成来访记录
2. 同一用户短时间连续进入同一主页，不重复生成多条记录
3. 同一日可按后台规则合并展示
4. 若访问者开启 `隐藏我的访问记录` 且具备该权益，则不进入对方访客列表
5. 首版默认：凡进入真实用户主页且未命中隐藏规则的行为，都计入来访

## 9.3 相互喜欢/匹配成功生成规则

首版匹配成功定义如下，满足任一条件即可视为 `匹配成功`：

1. `双方互送爱心`：
   - A 喜欢 B
   - B 后续喜欢 A
   - 两条喜欢记录都真实生效
   - 系统生成匹配成功记录
2. `精选页面送出“对 ta 心动”后，对方回送爱心`：
   - 用户 A 在精选页对用户 B 送出 `对 ta 心动`
   - 用户 B 观看 A 主页后回送 `爱心`
   - 系统生成匹配成功记录
3. `悄悄话后对方回复`：
   - 用户 A 向用户 B 发出悄悄话
   - 用户 B 对该悄悄话作出回复
   - 系统判定双方匹配成功

补充：

1. 匹配成功后，双方进入 `相互喜欢/匹配成功` 可见范围
2. 双方主页中的 `Yo` 按钮改为 `聊天 icon`
3. 底部主按钮改为 `聊天 icon + 聊天`

## 9.4 解锁生效规则

1. `单条解锁`：
   仅对当前记录生效
2. `全量解锁`：
   对当前模块对应的全部记录生效
3. `VIP 解锁`：
   按 VIP 权益规则展示清晰信息

补充：

1. `喜欢我的` 与 `最近看过我的` 解锁状态独立
2. 单条解锁不自动解锁另一模块

## 10. 页面级详细需求

## 10.1 喜欢我的列表页

### 10.1.1 页面目标

向用户展示所有对自己点过 `喜欢` 的异性用户，并承接付费解锁与匹配反馈。

### 10.1.2 页面入口

1. 成家模块内 Tab 入口
2. 消息页中的快捷入口
3. 匹配成功后回流入口

### 10.1.3 页面结构

1. 顶部统计区：
   - 喜欢我的人数
   - 副标题文案
2. 模糊头像/卡片列表
3. 底部 `解锁全部` 按钮

### 10.1.4 展示规则

VIP 用户：

1. 全部清晰展示头像
2. 展示昵称、年龄、学校/标签等核心信息
3. 可直接点击进入对方主页

普通用户：

1. 列表可展示，但头像默认模糊
2. 页面可展示：
   - `1小时前在线`
   - `985/211`
   - `同乡`
   - `同城`
   - `校友`
   - `同专业`
   - `距离小于 5 公里`
3. 头像上方可展示相同标签，如：
   - 恋爱观
   - 运动
4. 可展示理想型相关标签，来源于理想型页面已选择标签
5. 昵称、年龄、学校、清晰头像默认不展示
6. 点击单个用户弹出 `单条解锁弹窗`
7. 页面底部固定 `解锁全部喜欢我的人`

### 10.1.5 交互规则

1. 点击单个卡片：
   - 未解锁：弹单条解锁弹窗
   - 已解锁：进入对方主页
2. 点击底部 `解锁全部`：
   - 跳 VIP/充值承接
3. 若该用户已与自己形成双向喜欢：
   - 卡片展示 `相互喜欢` 标识
   - 可优先进入主页或消息链路

### 10.1.6 页面状态

1. 加载中
2. 空状态：
   - 暂无人喜欢你
3. 模糊列表态
4. 全量清晰态
5. 部分单条解锁态

### 10.1.7 空状态建议

1. 文案示例：
   `还没有人喜欢你，去完善资料和上传更吸引人的照片吧`
2. 按钮建议：
   - 去完善资料
   - 去成家看看

## 10.2 最近看过我的列表页

### 10.2.1 页面目标

向用户展示最近一段时间访问过自己主页的异性用户，并承接付费解锁。

### 10.2.2 页面入口

1. 成家模块内 Tab 入口
2. 消息页中的快捷入口

### 10.2.3 页面结构

1. 顶部统计区：
   - 总浏览量
   - 今日访客
   - 今日浏览量
   今日访客（同一个用户一天内多次浏览同一内容，只计 1 UV）=UV，今日浏览量（极短时间重复例如 <1 秒的快速刷新不作为累计）=PV，总浏览量=累计PV；
2. 分组展示区：
   - 今日来访
   - 昨日来访
   - 近 7 天内历史来访
3. 底部 `解锁全部访客` 按钮

### 10.2.4 展示规则

VIP 用户：

1. 全部清晰展示
2. 可直接进入对方主页

普通用户：

1. 默认模糊展示
2. 可展示：
   - `1小时前在线`
   - `访问了你2次`
   - `同乡`
   - `同城`
   - `校友`
   - `同专业`
   - `专业`
   - `985/211`
   - `星座`
3. 点击单个用户弹 `单条解锁弹窗`

### 10.2.5 特殊标识

首版支持以下状态标识：

1. `NEW`：新来访
2. `💌`：对方向你送过悄悄话
3. `✉️`：你向对方送过悄悄话
4. `💕`：已相互喜欢

### 10.2.6 时间规则

1. 访客底层记录可长期保留
2. 页面查询与展示范围为最近 `7 天`
3. 超出 `7 天` 的来访记录不在页面展示
4. 记录保留天数后台可配，首版默认查询范围为 `7 天`
5. 分组口径按自然日切分

### 10.2.7 页面状态

1. 加载中
2. 空状态：
   - 最近还没有人来看过你
3. 模糊列表态
4. 全量清晰态
5. 部分单条解锁态

## 10.3 相互喜欢列表页

### 10.3.1 页面目标

集中展示所有已经匹配成功的用户，作为聊天链路前的重要过渡页。

### 10.3.2 页面入口

1. 匹配成功弹窗点击进入
2. 成家模块内二级入口
3. 消息页快捷入口

### 10.3.3 页面结构

1. 顶部标题：
   - 相互喜欢（人数）
2. 双列或瀑布流卡片列表
3. 卡片底部快捷动作

### 10.3.4 展示规则

1. 相互喜欢列表默认清晰展示
2. 展示头像、昵称、年龄、身高、星座等基础资料
3. 可展示匹配成功时间或最近互动时间

### 10.3.5 快捷动作

1. `聊天`
2. `查看主页`

说明：

1. 匹配成功后，主页中的 `Yo` 改为 `聊天 icon`
2. 底部主按钮改为 `聊天 icon + 聊天`
3. 首版不再把 `Yo` 作为匹配成功后的主动作

### 10.3.6 页面状态

1. 空状态：
   - 还没有相互喜欢的人
2. 正常列表态

## 10.4 匹配成功弹窗/页面

### 10.4.1 触发时机

1. 当前用户对某人点喜欢后，系统发现对方也喜欢过自己
2. 或命中其他已确认的匹配成功条件
3. 当前确认：`立即弹`

### 10.4.2 弹窗目标

1. 强化正向反馈
2. 引导进入主页或消息链路

### 10.4.3 展示内容

1. 双方头像
2. 匹配成功标题
3. 一句鼓励性文案
4. 操作按钮

### 10.4.4 按钮建议

1. 去看看 ta
2. 去聊天
3. 稍后再说

## 10.5 单条解锁弹窗

### 10.5.1 适用场景

1. 喜欢我的未解锁单条记录
2. 最近看过我的未解锁单条记录

### 10.5.2 弹窗结构

1. 标题：
   - 查看对方资料
2. 副标题：
   - 解锁喜欢你的人 / 揭秘是谁来看过你
3. 模糊头像
4. 按钮区：
   - 只看 ta
   - 解锁全部

### 10.5.3 支付承接

1. `只看 ta`：
   - 优先走成家币单次解锁
2. `解锁全部`：
   - 优先走 VIP 权益承接

### 10.5.4 文案规则

1. 单条价格、全量价格、权益说明后台可配
2. 弹窗文案支持按模块区分

## 11. 准入与权限规则

## 11.1 未完成三项认证任一项用户

1. 不可进入 `喜欢我的`
2. 不可进入 `最近看过我的`
3. 不可进入 `相互喜欢`
4. 不可形成真实匹配
5. 不可进入聊天链路

前端拦截建议：

1. 统一跳转认证引导
2. 提示文案：
   `完成实名、头像、学历三项认证后，才可查看谁喜欢你、谁看过你，并开启真实互动`

## 11.2 三项认证全部通过用户

1. 开放 `喜欢我的`
2. 开放 `最近看过我的`
3. 开放 `相互喜欢`
4. 开放真实匹配链路

## 11.3 黑名单与隐私影响

1. 已被当前用户拉黑的用户，不再展示在反馈列表中
2. 已隐藏访问记录的用户，不进入对方访客列表
3. 若对方账号被封禁或注销，对应记录需隐藏或失效

## 12. 商业化边界

本模块只定义触发点和产品规则，不展开完整支付页。

### 12.1 喜欢我的

1. 普通用户：
   - 默认模糊
   - 支持单条解锁
   - 支持 VIP 解锁全部
2. VIP 用户：
   - 默认高清查看全部

### 12.2 最近看过我的

1. 普通用户：
   - 默认模糊
   - 支持单条解锁
   - 支持 VIP 解锁全部
2. VIP 用户：
   - 默认高清查看全部

### 12.3 悄悄话

## 15. 异常与边界场景

1. 喜欢记录被撤回：
   若业务支持撤回，需同步移出 `喜欢我的` 和 `相互喜欢`
2. 对方账号被冻结：
   记录失效，不再展示
3. 对方开启隐藏访问记录：
   不进入访客列表
4. 用户单条解锁后又开通 VIP：
   已解锁记录继续可见，VIP 期间全部可见
5. VIP 到期后：
   回退为普通用户展示规则，但已购买的单条永久解锁记录继续生效
6. `最近看过我的`：
   底层记录可长期保留，但页面仅展示最近 `7 天`
7. 同一用户多次访问：
   访客统计按已确认的 UV/PV 口径与去重规则处理，不无限堆叠
8. 匹配成功后对方又拉黑：
   从 `相互喜欢` 列表失效，后续不可聊天
