## 1. 产品概述
### 1.1 产品定位
“成家立业”是一款聚焦大学生群体的社交与成长小程序。通过 真实认证体系、人格测评、多场景匹配推荐，帮助用户建立从恋爱交友到兴趣社交、职业探索的完整关系链。产品以“成家”模块为核心，驱动深度匹配与安全社交；以“立业”“推荐”“消息”“我的”等模块满足成长、内容消费与个人管理需求，并通过会员与虚拟币体系实现商业闭环。
### 1.2 目标用户
- 在校大学生及毕业3年内的年轻人群
- 有真实恋爱需求，注重信息真实性与匹配效率
- 愿意通过测评、标签展示自我，寻求高质量关系
- 对职业方向、兴趣社交有附加需求
### 1.3 核心功能
- 三重认证：实名、学历、头像认证，构建信任基础
- 心印测评：性格/恋爱观测评，驱动精准匹配
- 觅缘每日推荐：卡片式高匹配用户推荐，限量互动
- 理想型精准搜索：自定义条件+付费解锁
- 精选/喜欢我的/访客：多维度社交反馈与付费触点
- VIP会员与成家币：可配置的会员权益与虚拟币消耗体系
- 立业测评：霍兰德等职业测评，辅助成长
- 推荐模块：聚合朋友与社区内容，满足非恋爱社交与内容发现
- 社区动态：作为推荐模块下的重要内容场景，承载图文内容生态与互动
- 消息与通知：私信、系统通知实时触达

## 3. 详细功能需求
以下按模块逐一描述，所有涉及次数、价格、权益、文案的内容均支持后台动态配置，无需发版调整。配置化参数在对应功能点中以“（后台可配）”标记。

### 3.1 成家模块
成家模块是产品核心，承载全部恋爱交友场景。

#### 3.1.1 觅缘 - 每日推荐流

| 文档信息 | 内容 |
| --- | --- |
| 需求名称 | 觅缘 - 每日推荐流与卡片互动 |
| 优先级 | P0（核心差异化功能） |
| 涉及端 | 移动端（微信小程序） |
| 关联后台 | 用户管理、量表测评、运营中心、系统配置 |
功能概述
“觅缘”是成家模块下的核心匹配场景，以 Tinder式卡片流 呈现每日 AI 推荐的高匹配异性用户。用户可通过卡片快速浏览多维信息，执行“喜欢/跳过/打招呼”等互动，并在每天限量额度内决策。普通用户消耗完当日配额后，通过成家币或 VIP 会员实现即时的社交扩容。
核心目标：
- 提升用户决策效率（信息丰富度 + 快速滑动操作）
- 保障真实性与安全感（三重认证 + 举报/屏蔽）
- 构建清晰的商业转化闭环（配额不足 → VIP / 成家币）

F1 每日推荐流
触发时机
- 用户进入「成家」容器 → 切换至「觅缘」Tab。
- 满足前置条件（见 F1.2）后，客户端请求 /api/mate/recommend 获取推荐列表。
前置条件
- 用户已完成 实名认证、学历认证、头像认证，且头像审核通过。
- 若任一未完成，不展示推荐流，转而显示 F7 认证引导横幅，并锁定此页面。
- 用户已登录且处于正常状态（未被冻结/注销）。
推荐规则
- 算法接口接收用户基础参数，返回以 推荐分 降序排列的用户列表。
- 基础维度：城市、年龄、学历、身高、择偶偏好（由用户偏好设置提供权重）。
- 算法加权因子（可后台配置权重）：
- 个人资料完整度
- 互动活跃度（登录、滑动、被喜欢数）
- 心印测试结果匹配度
- 会员状态（VIP 享有基础加权，具体数值后台配置）
- 硬性过滤：
- 已被当前用户屏蔽/拉黑的用户永久剔除。
- 已被当前用户举报且后台确认违规的用户不再出现。
- 性别过滤：仅展示异性（用户性别由注册时确定）。
- 算法排序细节不在本 PRD 中约束，由后端算法接口统一支持。
配额与限制
所有配额由后台系统参数控制，可随时调整：

| 配置项 | 默认值 | 说明 |
| --- | --- | --- |
| daily_free_limit | 10 | 普通用户每日可喜欢/跳过总次数 |
| vip_extra_quota | 10 | VIP 每日额外增加的配额（即 VIP 每日总计 20 次） |
| quota_reset_time | 每日 00:00 | 配额重置时间点 |
| unlike_continue_limit | 连续左滑 30 次 | 连续不喜欢阈值，触发“休息一下”提示（仅提示，不阻断） |
前端行为：
- 每执行一次“喜欢”或“跳过”（包括滑动/按钮），累计当日使用次数。
- 剩余配额展示：页面顶部轻提示“今日还可查看 X 位”（VIP 版本文案可差异）。
- 配额耗尽时：
- 当前卡片滑完后，弹出《配额用尽》挽留弹窗。
- 弹窗内容：文案“今日精选已看光”，按钮【明早再来】（关闭弹窗）和【开通VIP·无限畅享】（跳转 F8 会员页）。
- 用户关闭弹窗后，页面可保留空白状态，提示“明天零点更新”。

F3 底部互动按钮
固定于屏幕底部，始终悬浮。

| 按钮 | 语义 | 操作逻辑 |
| --- | --- | --- |
| 关闭（✕） | 不喜欢 / 跳过 | 点击扣减当日配额一次，当前卡片向左滑出消失，自动切换至下一人。 |
| 悄悄话 | 付费发招呼语 | 非 VIP 用户：点击弹出底部弹窗，提示“发悄悄话需要 10 币”，引导充值。VIP 用户：每日首次（1次）免费直接发送预设招呼语；超出后消耗成家币。被对方拉黑时，按钮灰色，点击 toast 提示“对方已关闭私信”。 |
| 爱心 | 喜欢 | 点击扣减当日配额一次，当前卡片右滑消失，调用喜欢接口。若对方也喜欢此用户，进入匹配成功流程（不在本模块展开）。 |
| YO icon | 快捷招呼 | 点击后弹出预设招呼语列表（后台文案配置），选中后以 悄悄话形式 发出，消耗规则同“悄悄话”。被拉黑时同样不可用。 |
异常处理：
- 接口超时/失败时，toast 提示“网络异常，请重试”，卡片不消失，允许重试。
- 重复操作（快速连点）需做防抖，1 秒内只生效一次。

F4 右上角更多操作
卡片页右上角“···”图标，点击弹窗 ActionSheet，选项：

| 操作 | 逻辑 |
| --- | --- |
| 微信分享 | 生成当前用户卡片海报（含二维码），调起微信分享。海报上模糊处理部分敏感信息。 |
| 举报用户 | 弹出举报原因选择（后台字典配置：涉黄、骚扰、虚假资料等），可补充文字。提交后：反馈 toast “已提交审核”。后台审核完成后，通过 站内信 通知举报者结果。被举报者根据后台处罚规则自动处理（如警告、禁言、封号，规则后台可配）。 |
| 屏蔽用户 | 二次确认后，将此用户加入个人黑名单。屏蔽后：该用户不再出现在推荐流。双方互相不可见对方主页及动态。已存的聊天记录仅保留，不可再发新消息。 |
| 取消 | 关闭弹窗。 |

F5 觅缘历史页
入口：顶部菜单右侧「历史」图标。
功能描述：
- 展示 最近 3 天 推荐过的用户列表（含当天、昨天、前天）。
- 数据展示规则：
- 当天推荐用户：显示清晰头像、完整标签。
- 非当天（昨天/前天）：头像和标签加高斯模糊（模糊程度后台可配），昵称仍可见。
- 点击任意用户触发判断：
- 若当天且未操作 → 进入该用户卡片页（可继续互动）。
- 若非当天或已模糊 → 弹出提示“该用户仅保留最近 3 天印象，开通 VIP 可回看全部分配”，引导至会员页。
- 若用户购买“解锁历史”功能（用成家币），则去除该条记录的模糊效果，并允许进入主页。具体解锁费用由后台配置。
后台可配项：
- 历史保留天数（默认 3）
- 开始模糊的天数（默认 >= 1天前）
- 解锁单条历史所需成家币数量。

F6 偏好设置
入口：顶部菜单右侧「偏好设置」图标。
分为 基础筛选 和 高级筛选 两档。
基础筛选（免费）
- 城市（多选，最多 3 个）
- 年龄范围（滑块，16-35 岁默认）
- 认证要求（仅显示已通过头像/学历认证的用户，默认勾选且不可取消）
这些设置将直接影响 F1 推荐接口的过滤参数。
高级筛选（引导 VIP）
提供更多选项，但处于锁定状态：
- 身高范围
- 对方家乡
- 谁可以看见我（如“仅我喜欢的”）
- 更多标签（MBTI 类型、兴趣爱好等）
每一高级选项旁显示「VIP」锁图标。点击任意锁定选项，弹出 VIP 开通弹窗，文案“解锁高级筛选，精准遇见对的人”，按钮【开通 VIP】。
成功开通 VIP 后，所有高级选项解锁，且修改后实时刷新推荐池。

F7 认证引导横幅
- 展现条件：当前用户未完成“实名+学历+头像”任一认证，且位于觅缘 Tab。
- 位置：卡片流顶部，固定横幅，不随卡片滚动消失。
- 文案：“立即认证，解锁 3 倍推荐并获得‘真实’标签”，右侧按钮【去认证】。
- 交互：点击跳转至认证中心对应步骤。
- 消失逻辑：三项认证全部通过后，此横幅自动移除，且该 session 内不再出现。

F8 商业化触点整合
觅缘模块内涉及的所有付费/转化节点，统一接入商业化组件：

| 节点 | 触发场景 | 付费内容 | 弹窗内容 | 后续动作 |

异常与边界条件
- 推荐池为空：若算法未返回任何用户，页面显示空状态插画“今日没有新的缘分了”，并提供【修改偏好】按钮。
- 网络异常：在卡片区域展示错误提示“加载失败，点击重试”。
- 用户被注销/冻结：若推荐列表中出现已被冻结的用户，直接过滤不展示。
- 重复推荐：同一用户在同一天内不可重复出现（服务端保证去重）。
- 敏感操作防抖：喜欢/跳过按钮 1s 内只响应一次。

非功能性要求
- 卡片加载：首次进入时，卡片数据预加载，切换卡片滑动手感需丝滑（保持 60fps），图片使用渐进式加载 + 懒加载。
- 安全性：所有接口调用服务端权限校验，禁止前端伪造 VIP 状态或绕过配额。

### 3.10 搜索模块
功能目标
提供全局搜索能力，支持用户、动态、话题的模糊搜索。
功能点
- 入口：全局搜索图标（如导航栏放大镜）。
- 搜索首页：展示热门搜索词、历史搜索记录。
- 搜索结果分Tab：用户、动态、话题。
- 用户结果：头像+昵称+学校+认证标签，点击进主页。
- 动态结果：与社区信息流卡片一致。
- 话题结果：话题名+动态数，点击进话题聚合页。
- 搜索逻辑：支持模糊匹配，优先展示互关好友、同校用户。
数据埋点
- search_start, search_result_show, search_click_result

## 4. 非功能性需求
- 性能：卡片滑动保持60fps，图片懒加载+渐进式展示；首屏加载时间≤2秒。
- 安全：所有接口由服务端校验权限与配额，禁止前端伪造VIP状态或绕过次数限制。敏感字段脱敏展示。
- 可用性：所有弹窗、异常状态、空状态、加载态均有清晰文案与操作引导；表单错误实时校验提示。
- 兼容性：适配微信小程序基础库≥2.10，兼容iOS与Android主流机型。
- 配置化：所有权益明细、价格、配额次数、文案、卡片顺序等均通过后台配置系统实时生效，无需发版。
- 防作弊：快速连点防抖（1秒内只生效1次），接口幂等性校验。
