# 成家立业移动端细化 PRD-09

## 1. 文档信息

- 文档主题：量表与测评移动端需求（固定测评版）
- 对应后台文档：`管理后台细化PRD-09_量表与测评管理（固定测评版）`
- 参考 demo：`/Users/外快/homeproject/homeformobile`
- 参考页面：`AssessmentList.vue`、`AssessmentDetail.vue`、`HollandReport.vue`、`JungReport.vue`、`ReportList.vue`、`Compatibility.vue`、`MatchUsers.vue`
- 版本口径：本文件按“固定几套测评”收敛，不建设通用量表配置平台。

## 2. 产品定位

量表与测评模块用于快速跑通“用户作答 -> 生成报告 -> 结果展示 -> 主页标签 -> 推荐算法预留”的业务闭环。本版本不追求任意量表新增、任意公式配置、报告拖拽搭建等平台化能力。

核心目标：

1. 固定上线三套测评，满足成家、立业、心理健康三个核心场景。
2. 给用户提供可读、可分享、可复用的测评报告。
3. 将测评结果沉淀为结构化字段，支撑个人主页展示和后续推荐算法。
4. 控制开发风险，避免首期投入过大的配置平台能力。

## 3. 固定测评范围

| 测评             | 场景              | 主要结果                                       | 是否进入交友推荐           |
| ---------------- | ----------------- | ---------------------------------------------- | -------------------------- |
| 霍兰德职业测评   | 立业/职业探索     | RIASEC 六维分、前三职业代码、职业建议          | 是，作为轻量加权和解释字段 |
| 荣格认知八维测评 | 成家/心印测试     | 八维功能分、人格类型、性格标签、合拍度基础字段 | 是，作为轻量加权和解释字段 |
| 心理健康测评量表 | 自我了解/风险提示 | 总分、风险等级、维度提示、自助建议             | 否                         |

说明：

1. 霍兰德职业测评进入交友推荐，作为轻量加权和解释字段（如职业兴趣相似度），同时用于职业报告和个人主页展示。
2. 荣格认知八维进入交友推荐，作为轻量加权和解释字段（如人格类型合拍度），只做固定字段预留，不做后台公式化匹配方案。
3. 心理健康测评只用于自我报告和必要的风险提示，不进入交友推荐，避免敏感标签参与社交分发。

## 4. 本版本不做范围

以下能力明确不纳入固定测评版：

1. 后台新增任意量表。
2. 后台编辑题目和选项。
3. 后台配置计分公式。
4. 后台配置常模。
5. 可视化报告拖拽编辑器。
6. 多版本灰度发布和回滚。
7. 婚恋匹配方案自由配置。
8. 任意测评字段映射推荐算法。
9. 复杂心理健康干预流程。

这些能力作为后续平台化版本评估。

## 5. 入口设计

| 入口                   | 展示测评         | 页面行为                                       |
| ---------------------- | ---------------- | ---------------------------------------------- |
| 成家/心印入口          | 荣格认知八维测评 | 完成后查看人格报告和合拍度入口                 |
| 立业/择业入口          | 霍兰德职业测评   | 完成后查看职业报告和职业建议                   |
| 我的/资料编辑入口      | 三套测评列表     | 查看完成状态、进入报告或继续作答               |
| 报告列表入口           | 已完成报告       | 按性格、职业、心理健康分组展示                 |
| 用户主页 MBTI/性格卡片 | 荣格结果         | 已完成则展示类型和合拍度入口，未完成则引导测评 |

## 6. 测评列表页

### 6.1 页面目标

展示三套固定测评的状态，并引导用户开始、继续或查看报告。

### 6.2 卡片字段

| 字段     | 说明                                   |
| -------- | -------------------------------------- |
| 测评名称 | 固定三套测评名称                       |
| 测评简介 | 固定文案，后台可轻量修改               |
| 预计耗时 | 固定配置，后台可改                     |
| 题目数   | 固定配置                               |
| 完成状态 | 未开始、进行中、已完成、需重测         |
| 操作按钮 | 开始测评、继续测评、查看报告、重新测评 |

### 6.3 状态规则

| 状态   | 判断                       | 点击行为         |
| ------ | -------------------------- | ---------------- |
| 未开始 | 无作答记录                 | 进入测评说明页   |
| 进行中 | 有未提交进度               | 进入断点题目     |
| 已完成 | 已生成报告                 | 进入报告页       |
| 需重测 | 后台标记题库更新或结果过期 | 提示后进入新测评 |

## 7. 测评说明页

三套测评均需展示说明页。

通用内容：

1. 测评名称。
2. 测评用途。
3. 预计耗时。
4. 题目数量。
5. 答题规则。
6. 结果展示范围。
7. 隐私提示。

差异内容：

| 测评     | 特殊说明                                       |
| -------- | ---------------------------------------------- |
| 霍兰德   | 用于职业兴趣探索，不作为职业选择唯一依据       |
| 荣格八维 | 用于性格理解和合拍度分析，可能影响推荐解释     |
| 心理健康 | 不构成医疗诊断，如感到持续不适建议寻求专业帮助 |

## 8. 作答页

### 8.1 通用交互

1. 支持分页作答。
2. 支持进度条。
3. 每题必答。
4. 漏答时提示并定位到题目。
5. 每次作答自动保存。
6. 中断后可继续。
7. 记录每题反应时。

### 8.2 题型收敛

固定测评版只支持以下题型：

| 题型   | 使用场景         |
| ------ | ---------------- |
| 单选题 | 三套测评主题型   |
| 量尺题 | 心理健康测评可用 |

多选、排序、填空暂不作为本版本必需能力。

### 8.3 断点续答

保存内容：

1. 用户 ID。
2. 测评编码。
3. 当前题号/页码。
4. 已作答答案。
5. 反应时。
6. 最近更新时间。

用户再次进入同一测评时，若存在未提交进度，展示“继续测评”。

## 9. 提交与报告生成

### 9.1 提交流程

```text
用户提交
  -> 客户端校验完整性
  -> 服务端保存答案和反应时
  -> 服务端按固定代码计算结果
  -> 生成固定报告
  -> 回写用户画像字段
  -> 返回报告 ID
  -> 客户端进入报告页
```

### 9.2 计算方式

计算逻辑固定在服务端代码中，不由后台配置。

| 测评     | 固定计算输出                                       |
| -------- | -------------------------------------------------- |
| 霍兰德   | R/I/A/S/E/C 六维分、Top3 代码、职业推荐标签        |
| 荣格八维 | Se/Si/Te/Ti/Fe/Fi/Ne/Ni 八维分、人格类型、性格标签 |
| 心理健康 | 总分、维度分、风险等级、建议等级                   |

## 10. 报告页

### 10.1 通用报告结构

1. 报告标题。
2. 完成时间。
3. 核心结果。
4. 结果标签。
5. 维度图表。
6. 结果解释。
7. 建议内容。
8. 底部操作：分享、保存、重新测试、展示/隐藏。

### 10.2 霍兰德职业报告

展示：

1. RIASEC 六维条形图或雷达图。
2. Top3 职业代码，例如 `AES`。
3. 六维解释。
4. 推荐职业方向。
5. 职业发展建议。
6. 入口：职业机会/岗位推荐。

### 10.3 荣格认知八维报告

展示：

1. 八维认知功能分。
2. 主导/辅助/第三/劣势功能。
3. 人格类型。
4. 性格标签。
5. 恋爱/相处建议。
6. 入口：寻找合拍的人、合拍度分析。

### 10.4 心理健康报告

展示：

1. 总体状态。
2. 风险等级。
3. 维度提示。
4. 自助建议。
5. 必要时提示联系专业机构或客服。
6. 医疗免责声明。

心理健康报告默认仅自己可见，不展示到他人主页。

## 11. 个人主页复用

### 11.1 展示内容

| 测评     | 主页展示                     |
| -------- | ---------------------------- |
| 霍兰德   | 可展示职业兴趣代码和职业标签 |
| 荣格八维 | 可展示人格类型和性格标签     |
| 心理健康 | 不展示                       |

### 11.2 展示开关

用户可在报告页控制是否展示到个人主页。

规则：

1. 默认不展示，用户主动开启。
2. 隐藏后他人不可见。
3. 隐藏不删除报告。
4. 隐藏对推荐算法是否生效需以后端策略为准；固定版建议隐藏只影响展示。

## 12. 合拍度与推荐联动

### 12.1 测评结果参与推荐方式

固定版输出以下推荐字段：

**荣格认知八维：**

| 字段               | 说明                  |
| ------------------ | --------------------- |
| `jungType`         | 人格类型              |
| `jungTopFunctions` | 最高的 2-3 个认知功能 |
| `personalityTags`  | 性格标签              |
| `jungCompleted`    | 是否完成荣格测评      |

**霍兰德职业测评：**

| 字段               | 说明                        |
| ------------------ | --------------------------- |
| `hollandCode`      | 前三职业代码（如 AES）      |
| `hollandScores`    | RIASEC 六维分               |
| `hollandCompleted` | 是否完成霍兰德测评          |

推荐算法可用这些字段做轻量加权、冷启动补充和推荐理由（如"你们都是探索型人格""职业兴趣相近"）。

### 12.2 不做复杂算法配置

固定版不支持：

1. 后台配置 MBTI 配对矩阵。
2. 后台配置不同测评权重。
3. 后台配置婚恋匹配公式。
4. 按会员等级切换测评匹配方案。

### 12.3 合拍度分析

若双方都有荣格结果：

1. 展示综合合拍度。
2. 展示双方人格类型。
3. 展示共同点、互补点、相处建议。

若当前用户未完成荣格测评：

1. 引导先完成荣格认知八维测评。
2. 完成后返回合拍度页面。

## 13. 报告列表

报告列表展示用户已完成的三类报告：

1. 性格报告：荣格认知八维。
2. 职业报告：霍兰德职业测评。
3. 心理健康报告：心理健康测评量表。

支持：

1. 搜索报告名称。
2. 查看报告。
3. 重新测评。
4. 展示状态标识。

## 14. 异常处理

| 场景           | 处理                                     |
| -------------- | ---------------------------------------- |
| 网络失败       | 保留当前答案，提示重试                   |
| 提交失败       | 停留当前页，允许重新提交                 |
| 报告生成失败   | 展示生成中或失败状态，允许刷新           |
| 用户重复提交   | 返回同一报告或覆盖最新报告，需服务端幂等 |
| 测评被后台停用 | 已完成报告可查看，未完成进度不可继续     |
