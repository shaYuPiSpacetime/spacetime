# 成家立业管理后台细化 PRD-05

## 1. 文档信息

- 文档主题：推荐模块（朋友、社区与内容互动）后台细化需求
- 对应移动端文档：`细化PRD-05_推荐模块（朋友、社区与内容互动）`
- 对齐基准文档：`成家立业 · 管理后台产品需求文档 V1.0`
- 使用目标：在不突破现有报价文档模块边界的前提下，细化移动端 PRD-05 对后台的落地要求

## 2. 对齐原则

本文件继续以总体后台需求文档为基准，不重新定义一级模块，不默认新增独立后台系统。仅当移动端 PRD-05 已确认需求无法自然落入既有后台模块时，才补充 `必要细化` 的二级能力；其余可增强但非本期必需的能力，统一归为 `后续增值服务建议`。

### 2.2 本期不单独立项的能力

以下能力不建议在本期作为独立模块写入报价交付范围：

1. 独立 `社区运营工作台`
2. 独立 `推荐内容分发平台`
3. 独立 `内容风控中心`
4. 独立 `话题运营平台`
5. 独立 `社区 BI 分析看板`

说明：

1. 以上能力如确有需要，可先通过现有 `内容审核`、`参数配置`、`文案模板`、`用户详情 Tab` 承接。
2. 若甲方后续希望做成独立菜单或专题系统，建议作为后续增值服务二期扩展。

## 3. 本期后台范围定义

PRD-05 对后台的本期范围，控制在以下 7 类能力：

1. 用户动态、评论、关注、举报记录的查询与人工排查
2. 动态、诚意贴、评论的审核与下架处理
3. 微信小程序官方机审能力的后台接入承接
4. 话题、举报原因、通知模板、空态文案配置
5. 推荐/社区互动规则与页面轻配置
6. 违规处罚、审核结果、互动通知的后台承接
7. 权限、日志、脱敏等基础安全能力对上述功能的支撑

### 4.1.3 用户生命周期管理

对齐总体后台文档：`3.2.3 用户生命周期管理`

本期重点细化与社区内容直接相关的状态联动：

1. 用户冻结、停用、注销中、已注销后：
   不再允许发布动态、诚意贴、评论、关注、点赞。
2. 用户三项认证失效或被撤销后：
   失去发布、评论、关注、点赞资格，但历史公开内容保留，按后台规则决定是否继续展示。
3. 用户被封禁后：
   历史内容可按处罚规则下架或屏蔽，新增互动能力关闭。

### 4.1.4 客服操作

对齐总体后台文档：`3.2.4 客服操作`

PRD-05 阶段建议客服/运营侧仅保留必要能力：

1. 查看用户动态、评论、关注、举报状态
2. 查看审核结果与处罚结果
3. 处理申诉与误判反馈
4. 查看互动通知是否发出

说明：

1. 本期不建议开放 `后台替用户发布动态`、`后台代评论`、`后台代关注` 这类高风险动作。
2. 如确需人工修正，建议仅向超级管理员开放，并强制记录操作日志与原因。

## 4.2 运营中心

对应总体后台文档：`3.5 运营中心`

### 4.2.1 内容审核

对齐总体后台文档：`3.5.3 内容审核`

本期不新增独立一级 `社区审核系统`，仍挂靠在既有 `内容审核` 能力下处理：

1. 社区动态审核
2. 诚意贴审核
3. 评论审核
4. 举报复核
5. 下架与处罚联动

#### A. 动态审核规则

1. 发布动态先走微信小程序官方机审
2. 文本内容走 `文本内容安全识别 /wxa/msg_sec_check`
3. 图片内容走 `多媒体内容安全识别 /wxa/media_check_async`
4. 机审不通过直接驳回
5. 机审通过后：
   - 社区动态可公开
   - 同时进入人工抽检队列

#### B. 诚意贴审核规则

1. 诚意贴同样先走微信官方机审
2. 文本和图片均通过后，进入人工审核队列
3. 仅人工审核通过后公开
4. 默认不允许留联系方式，后台开关开启后才可放开

#### C. 评论审核规则

1. 评论先走微信官方文本机审
2. 机审通过后前台展示
3. 机审不通过直接驳回
4. 支持人工复核误判内容

#### D. 举报处理规则

1. 举报对象类型：
   - 动态
   - 评论
   - 用户
2. 举报后进入人工复核
3. 核实违规后可执行：
   - 内容下架
   - 删除评论
   - 警告
   - 禁言
   - 封号
4. 审核结果需通过通知中心反馈

说明：

1. 微信官方机审只作为首层安全校验，不替代平台自身人工审核、人工抽检和举报复核。
2. 若微信官方接口异常或返回不确定结果，内容应进入 `待人工复核`，不直接公开。

### 4.2.2 文案与消息中心

对齐总体后台文档：`3.5.4 文案与消息中心`

本期补充以下模板与文案配置：

1. 动态审核通过 / 驳回通知模板
2. 诚意贴审核通过 / 驳回通知模板
3. 评论驳回通知模板
4. 举报处理结果通知模板
5. 收到点赞通知模板
6. 收到评论通知模板
7. 收到评论回复通知模板
8. 收到关注通知模板
9. 被 @ 通知模板
10. 推荐 / 社区空态文案

明确要求：

1. 点赞、评论、评论回复、关注、@、动态审核结果、诚意贴审核结果、举报处理结果均正式纳入通知中心。
2. 通知是否计入红点，继续由 `消息、私信与通知中心` 统一承接。

### 4.2.3 运营策略与内容分发预留

对齐总体后台文档：`3.5.5 运营策略`

   - 推荐

2. 若后续需要做复杂热榜、分群分发、AB 实验，建议作为增值服务。

## 4.3 系统管理中心

对应总体后台文档：`3.6 系统管理中心`

### 4.3.1 参数配置

对齐总体后台文档：`3.6.2 参数配置`

本期需补充并固化以下核心参数：

#### A. 准入与互动规则

1. 推荐 / 社区公开内容浏览：
   已登录用户可见
2. 以下动作需三项认证全部通过：
   - 发布动态
   - 发布诚意贴
   - 评论
   - 关注
   - @用户
   - 点赞
3. 举报动作：
   已登录用户可发起
4. 互相关注：
   仅形成关系标识，不开放普通私信

#### B. 发布规则

1. 动态文字上限：
   `500` 字
2. 图片上限：
   `9` 张
3. 必选话题：
   动态必须选 `1` 个
4. 诚意贴正文下限：
   `20` 字
5. @用户上限：
   每条动态最多 `5` 人

#### C. 话题与联系方式规则

1. 话题来源：
   仅允许后台预置
2. 用户不可自由创建新话题
3. 诚意贴联系方式开关：
   默认关闭

#### D. 审核规则

1. 动态：
   微信机审通过后公开，进入人工抽检
2. 诚意贴：
   微信机审通过后进入人工审核，通过后公开
3. 评论：
   微信文本机审通过后公开
4. 机审异常：
   进入待人工复核
5. 编辑动态：
   仅允许编辑文字
   编辑后需重新走微信官方机审

#### E. 排序规则

1. 社区首页保留 `推荐 + 最新`
2. 推荐算法未落地前，`推荐` 排序允许走规则流 / 人工配置

## 5. 权限、日志与脱敏要求

对应总体后台文档：`3.1 权限与账号体系`

本期继续沿用总体后台既有要求，并对推荐/社区链路补充如下约束：

1. 只有具备对应数据权限的角色，才能查看动态内容、评论内容、举报记录与审核结果。
2. 评论内容、举报补充说明、被驳回内容属于高敏信息，应限制在审核/运营主管/超级管理员角色。
3. 手机号、实名信息等敏感字段继续按总体文档要求脱敏展示。
4. 查看内容详情、改动审核状态、下架内容、处理举报结果，均需记录操作日志。
5. 高风险操作如手工通过内容、手工恢复下架内容、手工修改处罚结果，应仅向超级管理员开放，并要求二次确认。

## 8. 结论

管理后台 PRD-05 仍以总体后台报价文档为基准，不新增一级模块。移动端 `推荐模块（朋友、社区与内容互动）` 对后台的要求，首版主要落在以下既有模块中：

1. `用户管理中心`：承接用户视角的动态、评论、关注、举报辅助排查
2. `运营中心 -> 内容审核`：承接动态、诚意贴、评论审核与举报处罚联动
3. `运营中心 -> 文案与消息中心`：承接互动通知、审核结果通知、空态与提示文案
4. `系统管理中心`：承接准入规则、审核规则、排序兜底、话题与举报字典
5. `移动端配置管理`：承接推荐/社区页面轻配置、互动开关、联系方式开关与话题规则

除上述必要细化外，其余能力统一收敛为后续增值服务，确保本期范围与甲方报价口径一致。
