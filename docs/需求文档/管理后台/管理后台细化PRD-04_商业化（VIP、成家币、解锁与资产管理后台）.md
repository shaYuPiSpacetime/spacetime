# 成家立业管理后台细化 PRD-04

## 1. 文档信息

- 文档主题：商业化（VIP、成家币、解锁与资产管理后台）细化需求
- 对应移动端文档：`细化PRD-04_商业化（VIP、成家币、解锁与资产中心）`
- 对齐基准文档：`成家立业 · 管理后台产品需求文档 V1.0`
- 使用目标：在不突破现有报价文档模块边界的前提下，细化移动端 PRD-04 对后台的落地要求

## 2. 对齐原则

本文件继续以总体后台需求文档为基准，不重新定义一级模块，不默认新增独立后台系统。仅当移动端 PRD-04 已确认需求无法自然落入既有后台模块时，才补充 `必要细化` 的二级能力；其余可增强但非本期必需的能力，统一归为 `后续增值服务建议`。

### 2.1 本文档与总体后台文档的映射关系

| 本文档内容 | 对齐总体后台文档模块 | 说明 |
| --- | --- | --- |
| 用户资产摘要、用户侧订单/解锁/消费记录排查 | `3.2 用户管理中心` | 直接细化 |

### 2.2 本期不单独立项的能力

以下能力不建议在本期作为独立模块写入报价交付范围：

1. 独立 `商业化运营工作台`
2. 独立 `财务 BI 分析平台`
3. 独立 `复杂退款审批工作流系统`
4. 独立 `增长活动系统`
5. 独立 `曝光包投放分析平台`

说明：

1. 以上能力如确有需要，可先通过现有配置页、用户详情 Tab、订单记录、文案模板、日志抽屉承接。
2. 若甲方后续希望做成独立菜单或专题系统，建议作为后续增值服务二期扩展。

## 3. 本期后台范围定义

PRD-04 对后台的本期范围，控制在以下 8 类能力：

1. VIP 套餐、成家币套餐、权益和价格的配置
2. 用户资产、订单、流水、解锁记录的查询与人工排查
3. 财务中心下的订单、流水、退款、轻量对账承接
4. 商业化规则参数的统一管理
5. 会员页、充值页、支付结果页、通知文案的后台承接
6. 理想型、精选、关系反馈链路中的商业化规则承接
7. 人气曝光包的产品预留与场景占位配置
8. 权限、日志、脱敏等基础安全能力对上述功能的支撑

### 4.1.3 用户生命周期管理

对齐总体后台文档：`3.2.3 用户生命周期管理`

本期重点细化与资产和权益直接相关的状态联动：

1. 用户冻结、停用、注销中、已注销后：
   已购资产记录保留，但相关社交权益停止对外生效。
2. 用户三项认证失效或被撤销后：
   VIP/成家币资产本身仍保留，但涉及真实曝光、匹配、聊天的社交权益需暂停实际生效。
3. VIP 到期后：
   立即回退普通态；额外配额、全量查看、高级筛选、隐藏访问记录即时失效。
4. 成家币余额长期保留：
   所有成家币，包括充值所得、赠送所得、退款退回，均不过期。

### 4.1.4 客服操作

对齐总体后台文档：`3.2.4 客服操作`

PRD-04 阶段建议客服/运营侧仅保留必要能力：

1. 查看用户会员状态与资产摘要
2. 查看订单与流水记录
3. 查看单条解锁、全量解锁、理想型/精选解锁记录
4. 查看退款状态
5. 处理支付异常、到账异常、用户投诉

说明：

1. 本期不建议开放 `后台直接加币`、`后台手工赠送长期 VIP`、`后台强制改成功支付` 这类高风险动作。
2. 如确需人工修正，建议仅向超级管理员开放，并强制记录操作日志与原因。

### 4.1.5 必要新增说明

总体后台文档已经具备 `用户详情`、`行为摘要`、`客服操作` 等基础能力，但未把 `会员订单`、`成家币流水`、`解锁记录` 的后台查看方式拆开。本文件将其细化为 `用户详情下的商业化相关 Tab`，属于 `必要细化`，不建议包装成新的独立一级模块。

## 4.2 财务中心

对应总体后台文档：`财务中心`

本期商业化后台应明确将 `订单、流水、退款、轻量对账` 归入既有 `财务中心` 承接，而不是全部塞进用户管理或系统配置模块。

### 4.2.1 订单管理

本期需补充以下订单管理能力：

1. 会员订单列表
2. 成家币充值订单列表
3. 订单详情查询
4. 按订单状态、时间、用户、套餐、金额筛选
5. 支持导出

#### 搜索条件

1. 订单号
2. 用户 ID / 昵称 / 手机号
3. 订单类型：
   - VIP
   - 成家币
4. 套餐名称
5. 订单状态
6. 支付时间范围
7. 支付金额范围

#### 列表展示

1. 订单号
2. 用户信息
3. 订单类型
4. 套餐名称
5. 支付金额
6. 订单状态
7. 创建时间
8. 支付时间

说明：

1. 支付取消后的订单状态按已确认口径记为 `closed`。
2. 本期不建议在财务中心开放手工改价、手工改成功这类高风险动作。

### 4.2.2 资产流水管理

本期需补充以下流水能力：

1. 成家币充值流水
2. 成家币消费流水
3. 赠送 / 退款回退流水
4. VIP 开通与到期记录查询

#### 搜索条件

1. 用户 ID
2. 流水号
3. 资产类型
4. 业务场景
5. 时间范围

#### 列表展示

1. 流水号
2. 用户信息
3. 资产类型
4. 变动方向
5. 变动数量
6. 业务场景
7. 发生时间
8. 备注

明确要求：

1. 用户侧只展示一个成家币总余额，但财务中心流水需能区分 `充值所得`、`赠送所得`、`退款退回` 等来源。
2. 所有成家币均不过期。

### 4.2.3 退款管理

本期需承接的退款能力包括：

1. 特批退款记录查询
2. 退款状态流转查看
3. 退款原因记录
4. 退款结果回写前台状态

明确要求：

1. 前台不开放主动退款申请入口。
2. 如发生重复支付、系统故障等特批退款，订单状态需支持：
   - `refunding`
   - `refunded`
3. 财务中心应作为退款状态的主承接模块，用户详情页仅做辅助查看。

### 4.2.4 轻量对账

本期不展开完整财务对账系统，但建议财务中心保留轻量核对能力：

1. 按日查看 VIP 订单数
2. 按日查看成家币充值订单数
3. 按日查看退款订单数
4. 按支付状态统计订单数量与金额

说明：

1. 若甲方后续要求支付渠道级对账、差错追账、结算报表，建议作为后续增值服务展开。

## 4.4 系统管理中心

对应总体后台文档：`3.6 系统管理中心`

### 4.4.1 参数配置

对齐总体后台文档：`3.6.2 参数配置`

本期需补充并固化以下商业化核心参数：

#### A. VIP 规则

1. VIP 权益是否启用：
   权益集合由后台配置
2. 普通用户每日查看配额：
   默认 `10`
3. VIP 用户每日查看配额：
   默认 `20`
4. VIP 每日免费悄悄话次数：
   默认 `1`
5. VIP 到期后即时回退开关：
   默认开启

#### B. 成家币规则

1. 成家币是否允许过期：
   首版确认为 `不过期`
2. 是否区分充值币 / 赠送币双余额池：
   首版确认为 `不区分`
3. 成家币不足时承接方式：
   首版确认为 `弹出购买成家币套餐弹窗`
4. 成家币场景单价：
   按业务场景分别配置

#### C. 解锁规则

1. `喜欢我的` 单条解锁价格
2. `最近看过我的` 单条解锁价格
3. `喜欢我的` 解锁全部走 VIP
4. `最近看过我的` 解锁全部走 VIP
5. `理想型`：
   - 走成家币解锁
   - 支持单个解锁
   - 支持批量解锁
   - 单次最多勾选 `5` 个
   - VIP 不免单
   - 保留期默认 `90 天`
6. `精选主页解锁`：
   - 走成家币解锁
   - 保留期默认 `3 天`

#### D. 订单与退款规则

1. 支付取消后订单状态：
   `closed`
2. 订单状态枚举：
   - unpaid
   - success
   - closed
   - failed
   - refunding
   - refunded
3. 特批退款是否前台展示：
   首版确认为 `展示`
4. 前台是否开放主动退款申请：
   首版确认为 `不开放`

#### E. 曝光包预留规则

1. 是否启用曝光包场景预留
2. 触发场景：
   - 觅缘
   - 社区动态
3. 是否允许直接上架售卖：
   首版建议仅保留预留，不默认启用售卖

### 4.4.2 字典与枚举管理

对齐总体后台文档：`3.6.3 字典与枚举管理`

本期需要承接以下字典与枚举：

## 5. 权限、日志与脱敏要求

对应总体后台文档：`3.1 权限与账号体系`

本期继续沿用总体后台既有要求，并对商业化链路补充如下约束：

1. 只有具备对应数据权限的角色，才能查看用户资产、订单、流水、解锁记录。
2. 订单状态、退款状态、VIP 权益配置、成家币价格配置均属于高敏操作，应限制在运营主管/超级管理员角色。
3. 手机号、实名信息、支付信息等敏感字段继续按总体文档要求脱敏展示。
4. 查看订单详情、改动价格或权益规则、处理异常资产问题，均需记录操作日志。
5. 高风险操作如手工补偿、手工退款、手工加币，应仅向超级管理员开放，并要求二次确认。

## 8. 结论

管理后台 PRD-04 仍以总体后台报价文档为基准，不新增一级模块。移动端 `商业化（VIP、成家币、解锁与资产中心）` 对后台的要求，首版主要落在以下既有模块中：

1. `用户管理中心`：承接用户视角的资产摘要、会员订单、成家币流水、解锁记录辅助排查
2. `财务中心`：承接订单、流水、退款、轻量对账的主查询与主操作场景
3. `运营中心`：承接会员页/充值页/支付结果/退款状态/曝光包预留文案
4. `系统管理中心`：承接 VIP、成家币、解锁、订单、退款、曝光包预留等核心参数与枚举
5. `移动端配置管理`：承接套餐、价格、权益、配额、提示文案和曝光包场景占位配置

除上述必要细化外，其余能力统一收敛为后续增值服务，确保本期范围与甲方报价口径一致。
