# 成家立业管理后台细化 PRD-08

## 1. 文档信息

- 文档主题：推荐算法配置后台细化需求
- 对应移动端文档：`细化PRD-08_推荐算法（觅缘匹配与精选）`
- 对齐基准文档：`成家立业 · 管理后台产品需求文档 V1.0`
- 使用目标：在不突破现有报价文档模块边界的前提下，细化推荐算法模块对后台的落地要求

## 2. 对齐原则

本文件继续以总体后台需求文档为基准，不重新定义一级模块，不默认新增独立后台系统。仅当移动端 PRD-08 已确认需求无法自然落入既有后台模块时，才补充 `必要细化` 的二级能力。

### 2.1 本文档与总体后台文档的映射关系


| 本文档内容                  | 对齐总体后台文档模块                  | 说明                   |
| ---------------------- | --------------------------- | -------------------- |
| 推荐权重、阈值、规则配置           | `3.6.2 参数配置`                | 直接细化                 |
| 婚姻状态、感情状态、脱单目标枚举       | `3.6.3 字典与枚举管理`             | 直接细化（部分已在 PRD-01 定义） |
| 行为标签配置（使用时长/滑动模式/时段标签） | `3.6.3 字典与枚举管理`             | 直接细化                 |
| SHAP 特征重要度查看           | `3.2 用户管理中心` 扩展             | 在用户详情中查看推荐相关数据       |

| 权限控制、操作日志              | `3.1 权限与账号体系`、`3.1.5 审计与安全` | 直接引用                 |


### 2.2 本期不单独立项的能力

## 3. 后台配置需求

### 3.1 推荐参数配置

对齐总体后台文档 `3.6.2 参数配置`，在配置管理页新增"推荐算法配置"分组。

#### 3.1.1 通用参数


| 配置项             | 配置键                                      | 类型    | 默认值 | 说明                     |
| --------------- | ---------------------------------------- | ----- | --- | ---------------------- |
| 冷启动行为阈值         | `recommend.cold_start.threshold`         | Int   | 10  | total_actions 低于此值走冷启动 |
| 混合权重 α          | `recommend.hybrid.alpha`                 | Float | 0.6 | CF 权重，(1-α) 为 CB 权重    |

| 年龄默认范围          | `recommend.cold_start.default_age_range` | Int   | 5   | 冷启动时年龄偏好默认 ± 范围        |

#### 3.1.2 活跃度权重


| 配置项      | 配置键                                          | 类型    | 默认值  | 校验                 |
| -------- | -------------------------------------------- | ----- | ---- | ------------------ |
| 最近登录时间权重 | `recommend.activity.weight.last_login`       | Float | 0.30 | 0-1，且所有权重之和须 = 1.0 |
| 动态发布频率权重 | `recommend.activity.weight.post_freq`        | Float | 0.25 | 同上                 |
| 互动频率权重   | `recommend.activity.weight.interaction`      | Float | 0.20 | 同上                 |
| 资料完整度权重  | `recommend.activity.weight.profile_complete` | Float | 0.15 | 同上                 |
| 在线状态权重   | `recommend.activity.weight.online`           | Float | 0.10 | 同上                 |


交互要求：

1. 表单提交时前端校验权重之和 = 1.0，不满足则提示"权重之和须为 1.0"并不允许保存。
2. 操作记录在审计日志中。

#### 3.3.2 SHAP 特征重要度查看

在用户详情页提供"推荐特征解释"入口（运营/管理员可见）：

- 展示该用户最近一次推荐中，各个特征的平均 SHAP 贡献值排名。
- 展示形式：横向柱状图（正值绿色 = 正向贡献，负值红色 = 负向贡献）。
- 数据来源：离线 SHAP 计算结果，非实时。

### 3.4 推荐效果监控

复用现有的统计/看板模块，新增以下推荐相关指标卡片：


| 指标     | 计算方式                  | 刷新频率 |
| ------ | --------------------- | ---- |
| 日均推荐次数 | 推荐接口调用次数 / 天数         | 每小时  |
| 右滑率    | 右滑次数 / 推荐展示次数         | 每小时  |
| 匹配转化率  | 相互匹配次数 / 右滑次数         | 每小时  |
| 消息转化率  | 发消息匹配数 / 相互匹配数        | 每小时  |
| 约会转化率  | 持续聊天（≥ 3 轮）数 / 发消息匹配数 | 每小时  |


告警规则：

1. 右滑率突降 50%（较前 7 日均值）→ 运营人员通知。
2. 匹配转化率持续 3 天低于 5% → 算法排查 + 模型重训练。

### 3.5 操作日志

对齐 `3.1.5 审计与安全`：

1. 推荐参数修改（3.1 中各配置项）→ 记录修改前后的值、操作人、操作时间。
2. 权重修改 → 记录变更前后各权重值。
3. 字典枚举变更 → 记录新增/编辑/启停操作。
4. 模型版本切换 → 记录切换前后的模型文件 hash、版本号。
