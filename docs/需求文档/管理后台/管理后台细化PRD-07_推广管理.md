# 成家立业管理后台细化 PRD-07

## 1. 文档信息

- 文档主题：推广管理
- 对应移动端文档：`细化PRD-07_推广裂变与邀请奖励`
- 对齐基准文档：`成家立业 · 管理后台产品需求文档 V1.0`
- 使用目标：在不突破现有报价文档模块边界的前提下，细化裂变推广、校园代理、奖励统计与奖金结算的后台落地要求

说明：

- 现有正式文档中虚拟货币名称仍为 `成家币`。
- 甲方口头提到的 `千寻币`，本期按“奖励币与成家币同口径”承接；若后续统一更名，后台字段与文案可整体替换。

## 2. 对齐原则

本文件直接细化总体后台文档中的 `3.9 推广管理`，不新增新的一级后台系统。

推广模块本期拆为两条业务线：

1. `普通用户裂变奖励`
   - 以发放成家币为主
   - 核心目标是拉新和早期活跃
2. `校园代理推广管理`
   - 以统计数量和发放奖金/佣金为主
   - 核心目标是渠道增长与结算管理

## 3. 本期后台范围定义

PRD-07 对后台的本期范围，控制在以下 8 类能力：

1. 推广规则配置
2. 普通邀请关系管理
3. 普通邀请奖励流水与风控冻结
4. 校园代理信息管理
5. 校园代理二维码与推广素材管理
6. 代理统计与奖金结算台账
7. 推广异常识别与人工复核
8. 推广相关导出与操作日志

## 4. 模块细化方案

## 4.1 推广规则配置

对应总体后台文档：`3.9.1 推广规则配置`

### 4.1.1 普通用户奖励规则

#### A. 基础触发规则

支持配置以下奖励事件：

1. 首次注册并成功登录奖励
2. 基础资料完成奖励
3. 三项认证完成奖励
4. 首次购买 VIP 奖励
5. 首次充值成家币奖励

说明：

1. 普通邀请关系可在 `首次注册并成功登录` 后建立。
2. 普通用户 `成功邀请人数` 统一按 `三项认证完成` 计算。
3. 首版默认启用：
   - 首次注册并成功登录奖励
   - 基础资料完成奖励
   - 三项认证完成奖励

#### B. 奖励方式

支持以下配置：

1. 固定奖励
2. 阶梯奖励
3. 奖励上限
4. 生效时间
5. 失效时间

#### C. 阶梯规则

后台需支持按累计成功邀请人数配置阶梯：

1. 0-2 人
2. 3-8 人
3. 9-20 人
4. 21-50 人

每个阶梯支持配置：

1. 单人成家币奖励
2. 是否启用
3. 备注说明

说明：

1. 阶梯人数统计口径统一按 `完成三项认证的成功邀请人数`。

### 4.1.2 代理奖励规则

校园代理与普通用户裂变规则分开配置。

支持配置以下代理奖金事件：

1. 注册人数奖金
2. 首次成功登录人数奖金
3. 基础资料完成人数奖金
4. 三项认证完成人数奖金
5. 首购 VIP 奖金
6. 首充成家币奖金

每个事件支持：

1. 单次金额
2. 生效时间
3. 是否启用
4. 适用代理组

说明：

1. 校园代理奖励默认按 `奖金/佣金台账` 管理，不走普通用户成家币到账。
2. 首版不做自动打款。
3. 校园代理 `成功邀请人数` 统计口径统一按 `三项认证完成` 计算。

### 4.1.3 关系有效期

支持配置：

1. 普通邀请关系有效期
2. 代理推广关系有效期

规则说明：

1. 普通邀请默认仅用于首次绑定与后续奖励归属
2. 代理关系可用于统计后续激活或消费事件

### 4.1.4 风控参数

支持配置：

1. 单日奖励上限
2. 同设备邀请阈值
3. 同手机号异常阈值
4. 同支付账号异常阈值
5. 冻结开关
6. 人工复核开关

对应总体后台文档：`3.9.3 代理管理`

1. 代理 ID
2. 代理名称
3. 联系人
4. 联系电话
5. 学校
6. 校区
7. 合作状态：
   - normal
   - paused

8. 奖金规则组

### 4.4.2 校园代理二维码管理

后台需为每位代理生成：

1. 二维码编号 `qrCode`
2. 专属二维码

4. 可选二维码素材模板

1. 下载二维码
2. 重新生成
3. 停用

支持以下状态：

1. 正常推广
2. 暂停推广

状态变更后：

1. 新用户扫码仍可进入小程序
2. 是否继续计入代理奖励按规则配置

## 8. 结论

推广管理是当前正式需求中的真实漏项，且结合甲方最新要求，必须明确拆成两套逻辑：

1. `普通用户裂变`
   - 重点是让用户通过邀请 2 位新同学完成三项认证后，快速拿到足够的成家币奖励
   - 同时在注册成功登录、资料完成、三项认证完成三个节点支持分阶段发币
2. `校园代理推广`
   - 重点是按代理来源准确统计注册、登录、资料完成、三项认证和付费数据
   - 其中成功邀请口径统一按三项认证完成，并形成线下奖金结算台账

本期后台推广管理不需要新增新的一级系统，但必须把规则配置、关系记录、奖励流水、风控冻结、代理统计、结算台账这几块补完整，才能真正支撑前期裂变增长目标。
