# 成家立业管理后台细化 PRD-06

## 1. 文档信息

- 文档主题：认证与安全设置、我的页与搜索后台细化需求
- 对应移动端文档：`细化PRD-06_认证与安全设置、我的页与搜索`
- 对齐基准文档：`成家立业 · 管理后台产品需求文档 V1.0`
- 使用目标：在不突破现有报价文档模块边界的前提下，细化移动端 PRD-06 在后台侧的落地方案

## 2. 对齐原则

本文件继续以总体后台需求文档为基准，不重新定义一级模块，不默认新增独立后台系统。仅对移动端 PRD-06 已确定的能力做 `必要细化`，其余可增强但非本期必需能力，统一作为后续增值服务预留。

### 2.1 本文档与总体后台文档的映射关系

| 本文档内容 | 对齐总体后台文档模块 | 说明 |
| --- | --- | --- |
| 用户认证状态、隐私状态、黑名单与账号安全排查 | `3.2 用户管理中心` | 直接细化 |
| 公告、帮助、协议、认证权益说明、安全中心内容发布 | `3.5.4 文案与消息中心` | 直接细化 |
| 注销规则、隐私枚举、关键词规则、搜索屏蔽规则 | `3.6 系统管理中心` | 直接细化 |
| 我的页入口排序、设置页入口、H5 地址、热门词、搜索轻配置 | `3.7 移动端配置管理` | 直接细化 |
| 权限控制、日志留痕、敏感信息脱敏 | `3.1 权限与账号体系`、`3.1.5 审计与安全` | 直接引用，不新增模块 |

### 2.2 本期不单独立项的能力

以下能力不建议本期作为独立模块写入报价交付范围：

1. 独立 `安全治理数据看板`
2. 独立 `熟人屏蔽平台`
3. 独立 `搜索运营平台`
4. 独立 `客服工单系统`
5. 独立 `账号注销审批系统`

说明：

1. 本期可先通过现有 `用户管理中心`、`文案与消息中心`、`系统参数配置`、`移动端配置管理` 承接。
2. 若甲方后续希望做成独立菜单或专题工作台，建议作为后续增值服务二期扩展。

## 3. 本期后台范围定义

PRD-06 对后台的本期范围，控制在以下 6 类能力：

1. 用户认证状态、隐私设置、黑名单与账号安全状态查询
2. 我的页、设置页、帮助、公告、协议类内容配置
3. 安全中心内容发布与关键词屏蔽规则配置
4. 注销规则、隐私规则、通知开关项等系统参数维护
5. 搜索热门词、搜索屏蔽规则、搜索结果轻配置
6. 权限、日志、脱敏等基础安全能力对上述功能的支撑

2. 查看阻断原因

说明：

## 4.3 系统管理中心

对应总体后台文档：`3.6 系统管理中心`

本期负责沉淀核心规则参数、枚举和安全约束。

### 4.3.1 参数配置

#### A. 注销参数

1. 后悔期天数
2. 是否校验未到期 VIP
3. 是否校验未完成退款
4. 是否校验付费争议
5. 是否校验处罚状态

#### B. 隐私参数

1. 隐藏访问记录开关说明
2. 允许展示距离默认值
3. 隐藏活跃时间默认值
4. 婚况展示默认值
5. 资料更新动态默认值
6. 只接受异性互动默认值

#### C. 关键词屏蔽规则

1. 单用户最多可配置关键词数量
2. 单个关键词最大长度
3. 是否允许模糊匹配
4. 是否区分大小写

#### D. 搜索规则

1. 搜索结果默认排序方式
2. 是否启用后台置顶规则
3. 搜索违规词库
4. 搜索结果屏蔽规则

说明：

1. 首版搜索只保留 `综合相关度 + 后台基础排序`，不展开推荐算法排序。
2. 关键词屏蔽与搜索违规词库分开维护，避免前台逻辑混淆。

## 8. 结论

本期后台 PRD-06 不建议新增独立一级模块，建议继续挂靠在已有后台体系下落地：

1. `用户管理中心`：承接认证状态、隐私状态、黑名单、关键词、注销状态的用户侧排查
2. `运营中心 -> 文案与消息中心`：承接公告、帮助、协议、安全内容、认证权益说明、反馈查看
3. `系统管理中心`：承接注销规则、隐私规则、关键词规则、搜索规则与审计
4. `移动端配置管理`：承接我的页、设置页、安全中心、搜索页的轻配置

这样可以和总体后台 `V1.0` 保持一致，也能满足移动端 PRD-06 的首版落地需要。
