# 成家立业管理后台细化 PRD-09

## 1. 文档信息

- 文档主题：量表与测评管理后台需求（固定测评版）
- 对应移动端文档：`细化PRD-09_量表与测评（固定测评版）`
- 参考后台 demo：`/Users/外快/homeproject/home`
- 参考移动端 demo：`/Users/外快/homeproject/homeformobile`
- 版本口径：固定三套测评，不做通用量表配置平台。

## 2. 产品定位

后台只承担固定测评的运营管理、状态控制、数据查看和导出能力，不提供任意新增量表、题目编辑、公式配置、常模配置、报告拖拽编辑等平台化功能。

核心目标：

1. 管理三套固定测评的上下线和基础展示信息。
2. 查看用户作答进度和报告结果。
3. 支持运营导出数据和处理异常报告。
4. 为后续推荐算法提供固定结构化结果字段。
5. 给后续平台化建设保留升级空间。

## 3. 固定测评范围

| 测评编码 | 测评名称 | 分类 | 用途 | 推荐算法关系 |
| --- | --- | --- | --- | --- |
| `HOLLAND` | 霍兰德职业测评 | 职业 | 职业兴趣、岗位方向 | 不进入交友推荐 |
| `JUNG_8F` | 荣格认知八维测评 | 心印/性格 | 性格报告、合拍度、推荐解释 | 进入轻量推荐字段 |
| `MENTAL_HEALTH` | 心理健康测评量表 | 心理健康 | 自我了解、风险提示 | 不进入交友推荐 |

## 4. 不做范围

固定测评版后台不做：

1. 新增自定义量表。
2. 删除量表。
3. 编辑题目和选项。
4. 配置计分公式。
5. 配置常模。
6. 配置任意结果区间。
7. 报告模板拖拽搭建。
8. 婚恋匹配方案配置。
9. 推荐特征自由映射。
10. AB 灰度发布。

## 5. 菜单收敛

建议将 demo 中完整的量表菜单收敛为以下后台页面：

| 菜单 | 说明 |
| --- | --- |
| 测评列表 | 管理三套固定测评基础信息和状态 |
| 题目查看 | 只读查看题目、选项和分值 |
| 报告记录 | 查看用户报告和结果 |
| 作答进度 | 查看进行中、已完成、异常作答 |
| 数据导出 | 导出答题、结果、统计数据 |
| 文案管理 | 管理少量引导文案、报告固定文案 |
| 测评统计 | 查看完成率、平均耗时、报告查看率 |

demo 中的常模管理、公式管理、解释管理、报告配置、发布管理、岗位管理、婚恋性格匹配保留为后续平台化入口，不作为本版本交付范围。

## 6. 测评列表

### 6.1 页面目标

展示三套固定测评，并允许运营调整基础信息和启停状态。

### 6.2 列表字段

| 字段 | 说明 |
| --- | --- |
| 测评名称 | 固定，不允许新增 |
| 测评编码 | 固定，不允许修改 |
| 分类 | 职业、心印/性格、心理健康 |
| 简介 | 可编辑 |
| 预计耗时 | 可编辑 |
| 题目数 | 只读 |
| 当前版本 | 只读或由发版维护 |
| 展示模块 | 成家、立业、我的 |
| 状态 | 启用、停用 |
| 更新时间 | 自动记录 |
| 更新人 | 自动记录 |

### 6.3 操作

1. 查看详情。
2. 编辑简介、引导文案、预计耗时。
3. 启用/停用。
4. 查看题目。
5. 查看报告记录。
6. 查看统计。

### 6.4 启停规则

1. 停用后移动端不再展示入口。
2. 已完成报告仍可查看。
3. 进行中用户进入时提示“测评维护中，暂不可继续”。
4. 启停操作必须记录日志。

## 7. 题目查看

### 7.1 页面目标

只读查看固定题库，方便运营、客服和测试核对。

### 7.2 字段

| 字段 | 说明 |
| --- | --- |
| 测评名称 | 所属测评 |
| 题号 | 固定 |
| 题目内容 | 只读 |
| 题型 | 单选、量尺 |
| 选项 | 只读 |
| 分值 | 只读 |
| 所属维度 | 只读 |
| 是否反向计分 | 只读 |

### 7.3 操作

1. 按测评筛选。
2. 按题号搜索。
3. 导出题库。

不允许后台直接编辑题目。如需修改题目，应通过开发发版或后续平台化版本处理。

## 8. 文案管理

### 8.1 页面目标

允许运营修改少量安全文案，不触碰计分逻辑。

### 8.2 可编辑文案

| 文案 | 示例 |
| --- | --- |
| 测评简介 | 用于列表卡片 |
| 测评说明页文案 | 用于开始前说明 |
| 报告页提示文案 | 如免责声明 |
| 分享文案 | 分享标题和摘要 |
| 维护提示 | 测评停用时展示 |

### 8.3 不可编辑内容

1. 题目。
2. 选项。
3. 分值。
4. 公式。
5. 报告结构。
6. 结果分段。

## 9. 作答进度

### 9.1 页面目标

查看用户测评过程状态，支持客服排查。

### 9.2 查询条件

1. 用户 ID。
2. 手机号。
3. 昵称。
4. 测评名称。
5. 状态。
6. 时间范围。

### 9.3 列表字段

| 字段 | 说明 |
| --- | --- |
| 用户 | 用户 ID、昵称、手机号脱敏 |
| 测评 | 三套之一 |
| 状态 | 未开始、进行中、已提交、生成失败、异常 |
| 当前进度 | 已答题数/总题数 |
| 开始时间 | 首次作答时间 |
| 更新时间 | 最近保存时间 |
| 耗时 | 累计耗时 |
| 异常标记 | 过快、全同选项等 |

### 9.4 操作

1. 查看进度详情。
2. 查看已答答案。
3. 标记异常。
4. 清除进行中进度，仅管理员可操作。

## 10. 报告记录

### 10.1 页面目标

查看用户完成测评后的报告结果。

### 10.2 字段

| 字段 | 说明 |
| --- | --- |
| 报告 ID | 唯一 |
| 用户 | 用户 ID、昵称、手机号脱敏 |
| 测评名称 | 三套之一 |
| 核心结果 | 霍兰德代码、人格类型、心理健康风险等级 |
| 维度分 | 可展开查看 |
| 报告状态 | 成功、生成中、失败 |
| 展示状态 | 是否展示到个人主页 |
| 完成时间 | 提交时间 |
| 报告版本 | 固定版本号 |

### 10.3 操作

1. 查看报告。
2. 查看原始答案。
3. 查看反应时。
4. 重新生成报告。
5. 导出单条报告。
6. 标记异常报告。

## 12. 推荐算法接口

### 12.1 本版本输出

后台/服务端只向推荐域提供固定字段：

```text
jungCompleted
jungType
jungTopFunctions
personalityTags
```

### 12.2 本版本不提供

1. 任意量表字段映射。
2. 可配置推荐权重。
3. 可配置婚恋匹配公式。
4. 可配置 MBTI 配对矩阵。

推荐算法若需要使用荣格结果，由推荐服务按固定字段自行处理。

## 13. 数据导出

### 13.1 导出类型

1. 用户作答明细。
2. 测评结果汇总。
3. 报告记录。
4. 测评统计。
5. 异常作答清单。

### 13.2 导出权限

| 数据 | 权限要求 |
| --- | --- |
| 霍兰德结果 | 运营/管理员 |
| 荣格结果 | 运营/管理员 |
| 心理健康结果 | 管理员，需二次确认 |
| 原始答案 | 管理员或数据权限 |

### 13.3 脱敏规则

1. 手机号脱敏。
2. 用户真实姓名脱敏。
3. 心理健康结果导出需记录用途。
4. 导出操作记录日志。

## 14. 测评统计

按测评展示：

1. 展示人数。
2. 开始人数。
3. 完成人数。
4. 完成率。
5. 平均耗时。
6. 中断率。
7. 报告查看率。
8. 报告分享率。
9. 主页展示开启率。
10. 荣格结果用户的合拍度入口点击率。

## 15. 异常规则

后台自动标记：

| 异常 | 规则 |
| --- | --- |
| 作答过快 | 总耗时低于阈值 |
| 单题过快 | 大量题目反应时低于阈值 |
| 全同选项 | 单选题答案高度一致 |
| 报告生成失败 | 服务端计算或模板渲染失败 |
| 高风险心理结果 | 心理健康风险等级为高 |

高风险心理结果只进入后台受限查看，不触发公开展示。

## 18. 待甲方确认清单

| 问题 | 建议 |
| --- | --- |
| 三套固定测评是否确认？ | 建议确认为霍兰德、荣格八维、心理健康。 |
| 题目和计分规则由谁提供？ | 需甲方提供最终版和授权依据。 |
| 后台是否允许编辑题目？ | 固定版建议不允许，避免公式和报告联动风险。 |
| 后台是否允许编辑报告文案？ | 建议只开放简介、说明、免责声明等轻量文案。 |
| 心理健康高风险结果如何处理？ | 建议受限查看，不公开，不进推荐。 |
| 原始答案是否允许导出？ | 建议管理员权限，并记录用途和日志。 |
| 荣格结果是否影响觅缘推荐？ | 建议作为轻量加权和推荐解释，不作为硬过滤。 |
| 用户隐藏荣格报告后是否影响推荐？ | 建议隐藏只影响展示；算法使用需在隐私说明中告知。 |
| 报告重新生成是否覆盖旧报告？ | 建议保留历史，默认展示最新。 |
| 后续何时升级为通用量表平台？ | 建议等三套测评上线并验证完成率、报告查看率、推荐转化后再评估。 |
