# 认证审核页面说明

> 对应 PRD：`管理后台细化PRD-01_用户准入与资料认证初始化.md`
> 口径说明：本页面仅依据需求文档梳理，结合后台常规设计补充必要字段与交互，不受当前代码实现约束。

## 1. 页面范围

- 实名认证审核
- 学历认证审核
- 头像认证审核

## 2. 列表字段

- 用户头像
- 昵称
- 用户 UUID
- 认证类型
- 认证状态
- 提交时间
- 认证方式
- 驳回原因
- 认证等级

## 3. 查询条件

- 用户 UUID
- 昵称
- 认证状态
- 认证类型
- 提交时间范围

## 4. 操作按钮

- 通过
- 驳回
- 重置
- 查看详情

## 5. 详情 / 弹窗

- 审核弹窗
- 驳回原因输入框
- 审核状态选择

## 5.1 实名认证详情字段

- 用户 UUID
- 昵称
- 头像
- 提交时间
- 当前状态
- 微信返回摘要
- 最近失败原因
- 认证流水
- 人工跟进记录

## 5.2 学历认证详情字段

- 用户 UUID
- 昵称
- 学校
- 认证方式
- 提交时间
- 当前状态
- 驳回原因
- 认证材料摘要

## 5.3 头像认证详情字段

- 用户 UUID
- 昵称
- 当前主头像
- 历史头像
- 提交时间
- 审核状态
- 驳回原因
- 重新上传提示

## 6. 交互说明

- 三类认证共用同一页面框架，按路由切换不同列表。
- `待审核` 状态才显示 `通过` / `驳回`。
- 驳回时必须填写原因。
- 提交后刷新列表。
- 认证详情建议放在右侧抽屉或弹窗中，审核动作保留在列表和详情两侧都可触发。
- 实名认证默认只允许查看状态和处理异常，人工通过需更高权限。
- 三类认证状态、驳回原因、审核记录必须彼此独立，不可混用。
