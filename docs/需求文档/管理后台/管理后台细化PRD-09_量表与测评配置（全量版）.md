# 成家立业管理后台细化 PRD-09

## 1. 文档信息

- 文档主题：量表与测评配置后台全量需求
- 对应移动端文档：`细化PRD-09_量表与测评（全量版）`
- 管理后台 demo：`/Users/外快/homeproject/home`
- 移动端 demo：`/Users/外快/homeproject/homeformobile`
- 目标：定义后台从量表配置、题库、维度、常模、公式、解释、报告、发布、结果记录到推荐算法联动的完整能力。

## 2. 模块定位

后台量表与测评配置是移动端测评链路的运营和配置中枢，核心职责：

1. 配置量表主档、题目、维度、选项和计分规则。
2. 配置常模、公式、解释、建议和报告模板。
3. 配置发布范围、前置逻辑、展示模块和版本灰度。
4. 管理用户作答进度、报告记录和数据导出。
5. 将测评结果映射为用户画像和推荐算法特征。

## 4. 名称管理

### 4.1 页面目标

维护量表主档，是所有题目、维度、公式、报告和发布配置的基础。

### 4.2 列表字段

| 字段                | 说明                              |
| ------------------- | --------------------------------- |
| 量表名称            | 对内外展示名称                    |
| 量表编码            | 唯一编码，如 `HOLLAND`、`JUNG_8F` |
| 分类                | 心印、婚恋、职业、心理健康、知音  |
| 备用名称            | 面向不同模块展示的别名            |
| 简介/引导说明       | 移动端说明页文案                  |
| 题目数              | 自动统计                          |
| 当前版本            | 当前发布版本                      |
| 状态                | 草稿、已发布、已下架、停用        |
| 添加时间/添加人     | 审计信息                          |
| 最后更新时间/更新人 | 审计信息                          |

### 4.3 操作

1. 新增量表。
2. 编辑量表。
3. 复制量表。
4. 启用/停用。
5. 上架/下架。
6. 删除草稿。
7. 查看配置完整度。
8. 进入题目、维度、公式、报告配置。

## 5. 题目管理

### 5.1 页面目标

维护每份量表的题目、题型、选项、分值和展示规则。

### 5.2 支持题型

| 题型   | 配置项                             |
| ------ | ---------------------------------- |
| 单选题 | 选项文案、选项分值、是否自动下一题 |
| 多选题 | 选项文案、分值、最少/最多选择数    |
| 量尺题 | 最小值、最大值、步长、左右端文案   |
| 排序题 | 排序项、计分方式                   |
| 填空题 | 字数限制、是否参与文本分析         |
| 说明题 | 富文本内容，不计分                 |

### 5.3 题目字段

| 字段         | 说明                               |
| ------------ | ---------------------------------- |
| 所属量表     | 必填                               |
| 题目编号     | 自动或手动                         |
| 题目内容     | 支持富文本                         |
| 所属维度     | 可单维度或多维度                   |
| 题型         | 单选、多选、量尺、排序、填空、说明 |
| 是否必答     | 必答题不可跳过                     |
| 是否反向计分 | 用于心理/人格量表                  |
| 排序         | 控制移动端展示顺序                 |
| 状态         | 启用、停用                         |

### 5.4 导入导出

1. 下载题目模板。
2. Excel 批量导入题目和选项。
3. 导入前预览校验结果。
4. 导出当前量表题库。
5. 导入错误需展示行号、字段和错误原因。

## 6. 维度管理

### 6.1 页面目标

维护量表的维度体系，例如霍兰德 R/I/A/S/E/C、荣格八维、依恋焦虑/回避等。

### 6.2 字段

| 字段     | 说明               |
| -------- | ------------------ |
| 所属量表 | 必填               |
| 维度名称 | 如现实型、内倾直觉 |
| 维度代码 | 如 `R`、`Ni`       |
| 所属题目 | 关联题目列表       |
| 描述     | 维度含义           |
| 排序     | 报告和配置展示顺序 |
| 状态     | 启用、停用         |

### 6.3 规则

1. 同一量表内维度代码唯一。
2. 题目可关联一个或多个维度。
3. 维度被公式、常模或报告引用时，不允许直接删除。

## 7. 常模管理

### 7.1 页面目标

维护标准化计算所需的常模数据。

### 7.2 字段

| 字段     | 说明                       |
| -------- | -------------------------- |
| 所属量表 | 必填                       |
| 字段名称 | 如常模均值、标准差、分位数 |
| 字段代码 | 如 `mean_R`、`sd_R`        |
| 适用人群 | 全体、性别、年龄段、学历等 |
| 常模数据 | JSON                       |
| 状态     | 启用、停用                 |

### 7.3 校验

1. JSON 格式必须合法。
2. 常模字段被公式引用时，字段代码不可修改。
3. 同一量表、同一人群、同一字段代码只能有一条启用记录。

## 8. 公式管理

### 8.1 页面目标

配置量表计分和结果判断逻辑。

### 8.2 公式类型

| 类型       | 示例                                        |
| ---------- | ------------------------------------------- |
| 维度原始分 | `R = SUM(Q1,Q2,Q3)`                         |
| 标准分     | `T = 50 + 10 * Z`                           |
| 总分       | `TOTAL = R + I + A + S + E + C`             |
| 类型判断   | `TYPE = TOP3(R,I,A,S,E,C)`                  |
| 风险等级   | `IF TOTAL >= 60 THEN HIGH`                  |
| 匹配分     | `Score = TypeComp * 0.4 + ValueAlign * 0.6` |

### 8.3 配置字段

| 字段     | 说明                       |
| -------- | -------------------------- |
| 所属量表 | 必填                       |
| 计算维度 | 可为空，表示总分或全局公式 |
| 公式名称 | 对运营可读                 |
| 公式代码 | 对系统唯一                 |
| 表达式   | 可执行表达式               |
| 变量说明 | 引用题目、维度、常模       |
| 测试用例 | 输入样例和预期输出         |
| 状态     | 启用、停用                 |

### 8.4 公式校验

1. 保存前校验语法。
2. 校验引用题目、维度、常模是否存在。
3. 支持试算，展示中间值和最终结果。
4. 发布前必须至少通过一组测试用例。
5. 历史报告必须保留当时使用的公式版本。

## 9. 解释管理

### 9.1 页面目标

将分数、类型和标签转化为用户可读解释。

### 9.2 字段

| 字段       | 说明                                   |
| ---------- | -------------------------------------- |
| 所属量表   | 必填                                   |
| 分类       | 维度解释、类型解释、风险解释、关系解释 |
| 解释名称   | 如 INTJ、焦虑型依恋、R 型              |
| 解释类型   | 高分、低分、区间、类型                 |
| 临界值范围 | 如 `0-30`、`>=60`                      |
| 解释内容   | 富文本                                 |
| 标签       | 可回写到用户画像                       |
| 状态       | 启用、停用                             |

### 9.3 规则

1. 同一维度的解释区间不可重叠。
2. 类型解释必须能被公式结果命中。
3. 下架解释不影响历史报告展示。

## 11. 报告管理

### 11.1 报告列表

管理报告模板，字段包括：

| 字段            | 说明                             |
| --------------- | -------------------------------- |
| 报告名称        | 如 MBTI 性格类型报告             |
| 所属量表        | 关联量表                         |
| 类型            | 性格、职业、婚恋、知音、心理健康 |
| 适用结果        | 全部或指定结果类型               |
| 状态            | 草稿、已启用、已禁用             |
| 更新时间/更新人 | 审计信息                         |

### 11.2 报告配置

支持可视化配置：

1. 页面尺寸：A4、A5、手机。
2. 背景：纯色、渐变、图片。
3. 主题色。
4. 组件库：标题、文本、图片、标签、卡片、匹配推荐、雷达图、条形图、维度表、建议块。
5. 数据绑定：绑定 `resultType`、`dimensionScores`、`resultTags`、`matchReasonTags` 等字段。
6. 移动端预览和 PC 预览。
7. 保存草稿、发布模板、复制模板。

### 11.3 报告生成规则

1. 用户提交测评后，根据量表、版本和结果类型匹配报告模板。
2. 若没有精确模板，使用默认模板。
3. 报告生成失败时记录错误，允许后台重试。
4. 历史报告固定使用生成时模板快照。

## 15. 婚恋性格匹配

### 15.1 页面目标

配置测评结果如何计算用户之间的合拍度，并服务心印合拍、智能推荐和通用匹配。

### 15.2 字段

| 字段         | 说明                               |
| ------------ | ---------------------------------- |
| 方案名称     | 如 MBTI 经典配对方案               |
| 方案代码     | 如 `DM-MBTI-CLASSIC`               |
| 关联测评     | 荣格、依恋、爱情风格、婚恋价值观等 |
| 适用场景     | 心印合拍、智能推荐、通用           |
| 匹配公式     | 计算双人合拍分                     |
| 推荐理由模板 | 生成合拍原因                       |
| 状态         | 启用、停用                         |

### 15.3 规则

1. 同一场景同一时间至少有一个启用方案。
2. 方案启用前必须试算通过。
3. 可按人群、城市、会员等级、AB 实验选择不同方案。
4. 方案变更需记录操作日志，并保留旧版本用于回溯。

## 16. 测评结果与推荐算法映射

### 16.1 特征映射配置

| 字段       | 说明                                  |
| ---------- | ------------------------------------- |
| 来源量表   | 如荣格八维                            |

### 16.2 推荐输出回写

## 18. 数据导出

导出范围：

1. 量表配置。

## 20. 数据监控

每个量表展示：

1. 曝光人数。
2. 开始人数。
3. 完成人数。
4. 完成率。
5. 平均耗时。
6. 放弃题号分布。
7. 报告生成成功率。
8. 分享率。
9. 主页展示开启率。
10. 对推荐转化的影响：右滑率、匹配率、消息转化率、投诉率。

## 21. 异常与风控

1. 极短时间完成：标记低置信度。
2. 全部同选项：标记异常。
3. 多次重测结果剧烈变化：提示结果稳定性较低。
4. 心理健康高风险：按合规规则提示用户和平台处理，不直接公开。
5. 公式或报告生成失败：告警并进入待处理列表。

## 22. 待甲方确认清单

| 问题                                     | 建议                                                                     |
| ---------------------------------------- | ------------------------------------------------------------------------ |
| 是否真的需要全量可配置平台？             | 全量版可支持长期运营，但开发量显著高于固定三套测评。                     |
| 首批量表有哪些？                         | 当前候选包括霍兰德、荣格八维、心理健康、亲密关系、爱情风格、婚恋价值观。 |
| 量表内容版权和专业来源由谁负责？         | 需甲方确认授权、专家审核和免责声明。                                     |
| 公式是否允许运营自由编辑？               | 建议公式编辑限制给管理员/产品，并强制试算。                              |
| 报告模板是否必须可视化拖拽？             | 若需要全量平台化，应做；若只固定三套，可降级为固定模板。                 |
| 是否支持版本灰度和回滚？                 | 全量版建议必须支持，否则无法安全迭代量表。                               |
| 测评结果是否默认展示到用户主页？         | 建议默认关闭，由用户主动开启。                                           |
| 心理健康数据是否可导出？                 | 建议严格限制权限，并做脱敏、审计和合规提示。                             |
| 推荐算法使用哪些量表结果？               | 建议心印/婚恋类进入推荐，职业类和心理健康类默认不进入交友推荐。          |
| 隐藏报告后是否仍参与推荐？               | 需甲方明确；建议区分“对外展示”和“算法使用授权”。                         |
| 婚恋匹配公式由后台配置还是算法服务固定？ | 建议后台配置方案元数据和权重，复杂计算由算法服务实现。                   |
| 合拍的人是否付费解锁？                   | 需与商业化 PRD 对齐成家币/VIP 规则。                                     |
| 是否提供学术研究原始数据导出？           | 若提供，需要单独授权、协议和数据脱敏流程。                               |
