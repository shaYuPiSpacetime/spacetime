## 3. 总体结论

### 3.1 商业化（PRD-04）

后端接口已较完整（VIP/成家币/订单/解锁/资产摘要/流水，12 个接口中 12 个已实现），但存在以下核心问题：

1. **支付渠道未确认**：PRD 写微信支付，demo 展示支付宝+连续订阅。决定订单表结构和支付回调逻辑。
2. **前端 service 路径与后端不一致**：如 `/miniapp/payment/vip/packages` vs `/miniapp/vip/packages`。
3. **解锁与 VIP 权益边界模糊**：VIP 解锁理想型是否扣成家币？各场景保留期未最终确认。
4. **后台商业化页面范围不明确**：挂财务中心二级页还是独立菜单？
5. **成家币默认金额、赠送规则、不过期策略未最终拍板**。

### 3.2 社区/推荐（PRD-05）

后端基础接口已完整（动态/评论/点赞/关注/举报，11 个接口全部已实现），但以下问题阻塞开发：

1. **"推荐"Tab 下的二级结构未定**：觅知音/悦目/诚意贴三者的定义和数据源，**PRD 和 demo 完全反向**（D-08）。
2. **社区准入逻辑依赖 C-01**：发布/评论/点赞需要三项认证还是仅实名？
3. **机审+人工审核流程细节缺失**：什么情况自动公开、什么情况进入人工复核？
4. **悦目/诚意贴数据模型未定**：独立表还是社区动态 subtype？
5. **demo 有 11 个社区分类，PRD 只有 3 个子页**，差异巨大。

### 3.3 测评（PRD-09）

**三个 PRD 中差距最大的模块**——全量版描述的是一个完整 SaaS 测评平台，但首版交付边界完全模糊。

1. 全量版包含量表/题库/维度/公式/常模/报告模板/发布/灰度/回滚/推荐映射——远超首版需求。
2. 首批测评清单未定：霍兰德/荣格/成人依恋/沟通模式/爱情价值观/Super 职业价值观，到底做哪几套？
3. 公式引擎和报告模板系统是独立产品级别的工作量。
4. demo 中"心印测试"（SoulTest）与 PRD-09 测评体系的关系未定义。

### 6.4 首版 MVP 建议交付边界

> 以下为技术侧建议，需需求方确认。

```
首版 MVP（建议）：
├── 量表管理：固定题型（单选/多选/量表），不支持运营新增题型
├── 题目管理：CRUD + 排序 + 分组
├── 维度配置：维度定义 + 固定计分规则（不开放公式编辑器）
├── 报告模板：固定模板 + 变量替换（不做拖拽）
├── 作答记录：存储 + 查看
├── 报告生成：按固定规则生成，不做灰度/回滚
└── 立业首页：聚合测评入口 + 最近报告

二期（全量版）：
├── 公式引擎（运营可编辑）
├── 可视化报告模板
├── 灰度发布 + 版本回滚
├── 常模管理
├── 推荐映射配置
└── 数据导出
```
