# 一期上线目标 - 移动端与管理后台

> 口径：本文档从 `docs/相关资料/移动端demo页面清单.md`、`docs/需求文档/移动端/`、`docs/需求文档/管理后台/` 收敛而来，只描述一期上线目标；demo 盘点清单和各模块 PRD 作为原始参考保留。
>
> 编号说明：移动端使用 `MVP-PAGE` / `MVP-POP` 作为一期上线目标编号；管理后台使用 `ADM-MVP` 作为一期上线目标编号；`demo 原编号`用于回溯原 demo 页面清单。
>
> 不列纯容器、底部 Tab、无实际入口的保留路由；底部 Tab 仅作为菜单树层级展示。

## 1. 一期上线总目标

一期上线以“小程序核心闭环可用、后台可运营可审核可追踪”为目标，移动端和管理后台必须成组交付：

| 目标方向 | 移动端上线内容 | 管理后台上线内容 |
| --- | --- | --- |
| 用户准入与资料 | 微信登录、资料初始化、编辑资料、用户主页、实名/学历/头像认证、单身承诺 | App 用户管理、用户详情、准入状态、资料完整度、冻结/解冻、启用/停用、认证审核 |
| 推荐与关系 | 觅缘推荐、心动列表、喜欢/相互喜欢、偏好设置、见面偏好、理想型、用户主页 | 用户关系摘要、推荐/匹配相关配置、推荐数据查看、用户状态处置 |
| 社区与内容 | 千寻成家信息流、话题、发布动态、动态详情、评论/互动、举报 | 内容审核、评论审核、举报处理、社区配置、资料照片与文字审核 |
| 消息与通知 | 消息列表、私信、官方助手、官方消息、悄悄话、通知中心、邀请响应 | 消息/通知记录查询、模板文案配置、举报与处罚联动、操作日志 |
| 商业化 | VIP 会员、成家币充值、明细、权益弹窗、解锁/悄悄话消耗 | VIP 权益配置、VIP 套餐配置、成家币套餐配置、订单管理、流水管理、退款管理 |
| 推广裂变 | 推荐给好友、活动规则、邀请响应、邀请奖励展示入口 | 推广规则、邀请关系、奖励审核、校园代理、代理结算 |
| 我的与安全 | 我的页、帮助客服、反馈箱、设置、隐私设置、账号注销、协议清单 | 反馈箱、注销申请、公告与帮助内容、协议配置、搜索热词/屏蔽词 |
| 测评与职业 | 心印测试、职业推荐、职业测评报告、推荐职位 | 固定测评列表、题目查看、报告记录、作答进度、统计与导出 |

## 2. 一期范围边界确认

| 边界项 | 一期确认口径 |
| --- | --- |
| 聊天关系规则 | 普通私信必须在双方相互喜欢后才能开聊；未相互喜欢时不开放普通私信。 |
| 破冰能力 | 悄悄话/小纸条作为一期破冰能力保留，和普通私信区分；其次数、消耗、审核、举报等细节进入后续详细设计。 |
| 举报与安全闭环 | 聊天举报纳入一期；推荐卡、用户主页、社区详情、会话页需保留举报及场景内屏蔽/不再推荐能力。本期不做完整黑名单管理页、关键词屏蔽页、安全中心独立页。 |
| 后台举报承接 | 后台统一承接用户举报、内容举报、聊天举报、悄悄话举报等举报来源，按来源区分处理上下文和处置记录。 |
| 心印测试 | 一期作为固定测评入口保留，产出报告，并参与推荐；不做通用低代码量表配置平台，不开放独立统一测评中心。 |
| 知音场景 | 觅知音作为知音关系发现入口保留；悦目作为偏视觉浏览与悄悄话破冰入口保留；诚意贴作为征友/自我介绍贴发布与互动入口保留。 |
| 搜索范围 | 一期搜索按入口限定搜索对象，不定义为全站统一搜索；例如理想型页面搜索用户标签或兴趣爱好，其他入口按所属场景收敛搜索范围。 |
| 通知机制 | 不做用户自助通知设置页，也不做后台完整通知偏好开关；保留业务节点通知和站内通知。微信订阅消息模板申请作为上线前置风险确认项。 |
| 登录与手机号 | 微信登录和手机号绑定作为一期强制准入要求；用户未完成手机号绑定时不进入核心使用流程。 |
| 福利中心 | 一期福利中心仅保留每日签到，不包含任务系统、活动福利或其他福利玩法。 |
| 公告入口 | 我的 -> 帮助与客服不保留公告栏；公告栏仅保留在我的 -> 设置 -> 关于我们下。 |
