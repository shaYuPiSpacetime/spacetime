# PRD 标准文件夹 - 导航

> 本文件夹定义"一份合格 PRD 长什么样"，并提供可复用模板。**第一次进来先看本文件。**

---

## 1. 这里有什么（永久文件）

| 文件                           | 是什么                                                                                               | 你什么时候用它         |
| ------------------------------ | ---------------------------------------------------------------------------------------------------- | ---------------------- |
| `00_PRD编写规范.md`            | **总纲**。12 条硬要求、分层结构、ID 规范、字段/操作/状态/验收标准、反模式、自检清单                  | 写 PRD 前通读一遍      |
| `05_PRD评审清单.md`            | **评审清单**。产品/技术/一致性三维度逐项检查，P0 阻塞 / P1 加分                                      | 每个 PRD 定稿前        |
| `10_模板_全局定义_共享层.md`   | **共享层模板**（全项目唯一一份）：术语、枚举、状态机、业务规则、第三方、配置、通知、错误码、接口约定 | 项目启动时建，持续维护 |
| `11_模板_全局定义_端专属层.md` | **端专属层模板**（后台一份、移动端一份）：菜单树、权限矩阵、UI 空态、文案                            | 项目启动时建，持续维护 |
| `20_模板_模块PRD.md`           | **模块 PRD 模板**：目标、范围、边界、流程、数据模型、依赖、页面清单                                  | 每个模块开始前         |
| `30_模板_页面规格.md`          | **页面规格模板**：路由、字段表、操作表、联动、状态、验收                                             | 每个页面一份           |

## 2. 怎么用（顺序）

```
①  通读 00_PRD编写规范.md（理解 12 条要求 + ID 规范）
        │
②  建全局定义（项目级，只做一次，持续维护）
    ├── 10 共享层（术语/枚举/状态机/错误码…，全项目唯一）
    └── 11 端专属层（后台一份 + 移动端一份：菜单/权限/UI）
        │
③  每个模块：用 20_模板 写模块 PRD（目标/范围/边界/流程/数据/依赖/页面清单）
        │
④  每个页面：用 30_模板 写页面规格（字段表/操作表/状态/验收）
        │   —— 凡涉及枚举/状态/权限/错误码/文案，只引用全局定义 ID，不复述
        │
⑤  定稿前：对照 05_PRD评审清单 逐项打勾（P0 全过 + P1 ≥80%）
```

**铁律（来自本项目踩坑）：**

- 字段只写名字 = 不合格，必须填满字段表 10 列
- 状态只写 happy path = 不合格，必须穷举 6 类状态
- "待确认"遗留到定稿后 = 不合格，拍板后回写正文
- 共享层的东西（术语/枚举/状态机/错误码/业务规则）只写一份，端专属层不复述

## 3. 使用建议

> 以下不是硬性门禁，是降低踩坑概率的工作习惯。

**3.1 PRD 与技术方案对齐（对应 05 清单 C4）**
技术方案(tcdesign)通常在 PRD 之后写，是 PRD↔技术方案漂移的高发区。建议技术方案定稿时，回头对一遍接口前缀、状态机、字段、错误码是否与全局定义一致；发现不一致，改全局定义（单一事实源），不要在技术方案里另立一套。

**3.2 全局定义变更要通知下游**
全局定义是被 N 个页面引用的单一事实源，改它影响面大。建议：

- 改全局定义时，**变更记录里必须列出受影响的页面 ID**（10/11 模板的变更记录表头已注明）。
- 改完后通知引用方同步更新；尤其是枚举值增减、状态机改动、权限矩阵调整。
- 能加版本号的（如枚举字典），前端按版本号失效缓存，避免老缓存。

**3.3 ID 唯一性**
当前产品 PRD 由一人输出，ID 唯一性由本人保证即可，无需额外注册表。若后续多人并行写 PRD，再约定"模块号 = 人，各管号段"或引入 ID 注册表。

**3.4 善用 ID 串起全链路**
设计稿、代码、测试用例、Bug 都引用页面/字段/操作 ID。这样改一处能反查影响面，文档漂移也能被发现。这是"可追溯"落地的关键，别嫌麻烦。
