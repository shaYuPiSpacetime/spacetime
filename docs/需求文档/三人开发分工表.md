# 成家立业项目三人开发分工表

## 1. 分工前提

1. 移动端前端代码由专人负责，不纳入本次三人分工。
2. 本次三人均负责：
   - 后端代码
   - 管理后台前端页面
   - 给移动端提供接口
3. 分工以 `领域边界清晰、尽量减少交叉` 为原则。
4. `推荐算法` 与 `测评` 相关内容纳入首批，只是当前详细 PRD 尚未完全输出，先按“底座 + 接口 + 可扩展框架”推进。
5. `C 同学` 工作量相对降低，因为还需兼顾整体需求工作。

## 2. 总体分工结论

| 负责人 | 负责域 | 权重 | 说明 |
| --- | --- | --- | --- |
| A 同学 | 成家主链路 + 推荐算法域 | 高 | 核心用户关系链、推荐结果生产、成家主页面相关 |
| B 同学 | 社区内容 + 商业化 + 测评域 | 高 | 社区互动、VIP/成家币、量表与测评、立业模块 |
| C 同学 | 推广管理 + 权限账号体系 + 平台配置域 | 中 | 推广裂变、代理统计、后台权限、公共配置与搜索/我的等平台型模块 |

## 5. 必须提前统一的公共边界

这几项一定要最早对齐，否则三个人后面一定会互相影响：

| 公共边界 | 牵头人 | 参与人 | 需确认内容 |
| --- | --- | --- | --- |
| 用户主表与扩展表 | A | B、C | 哪些字段归主表，哪些字段归扩展表；谁能改哪些字段 |
| 通知中心消息类型 | A | B、C | 通知类型枚举、模板参数、红点口径 |
| 成家币流水来源枚举 | B | C | 商业化发币与推广发币统一流水类型 |
| 标签与字典体系 | C | A、B | 搜索标签、推荐标签、测评标签是否共用字典 |
| 后台权限点清单 | C | A、B | 每个后台页面、按钮、导出、审核动作对应权限点 |
| 风控状态枚举 | A | B、C | 邀请风控、社区风控、账号风控的状态口径是否统一 |

## 7. 研发协作要求

1. 不允许三个人同时改同一张核心表结构而不先同步。
2. 后端接口在联调前，必须先出接口文档或 Swagger。
3. 每个负责人都要对自己模块的后台页面权限点给 C 同学。
4. 所有写入通知中心的模块，必须复用统一通知类型枚举。
5. 所有写入成家币流水的模块，必须复用统一流水来源枚举。
6. 推荐算法和测评虽然细 PRD 未完全输出，但首批必须先把 `数据结构、接口骨架、后台配置入口` 搭起来。
7. C 同学工作量已压低，但他负责的平台能力会影响所有人，A/B 不能等他全部做完才提需求，必须第一周就把依赖清单给他。

## 8. 一句话版分工

1. `A 同学`：成家主链路 + 推荐算法域
2. `B 同学`：社区内容 + 商业化 + 测评域
3. `C 同学`：推广管理 + 权限账号体系 + 平台配置域
