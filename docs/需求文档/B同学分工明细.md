# B 同学分工明细

> 从 `三人开发分工表.md` 抽取，专注 B 同学负责范围。

---

## 1. 分工前提（全员适用）

1. 移动端前端代码由专人负责，不纳入本次三人分工。
2. 本次三人均负责：后端代码、管理后台前端页面、给移动端提供接口。
3. 分工以 `领域边界清晰、尽量减少交叉` 为原则。
4. `推荐算法` 与 `测评` 相关内容纳入首批，只是当前详细 PRD 尚未完全输出，先按"底座 + 接口 + 可扩展框架"推进。

---

## 2. 负责域

**社区内容 + 商业化 + 测评域**（权重：高）

社区互动、VIP/成家币、量表与测评、立业模块。

---

## 3. 负责模块

| 模块 | 备注 |
| --- | --- |
| 商业化（VIP/成家币/订单/流水） | 含财务中心相关后台页面 |
| 推荐模块（朋友/社区/内容互动） | 社区动态、诚意贴、评论、举报 |
| 立业模块 | 当前纳入首批，测评驱动 |
| 量表与测评 | 测评作答、报告、标签回写 |
| 后台量表与测评管理 | 独立后台域 |

对应 PRD：
- `PRD-04 商业化（VIP、成家币、解锁与资产中心）`
- `PRD-05 推荐模块（朋友、社区与内容互动）`
- `立业模块`
- `量表与测评`
- `后台量表与测评管理`

---

## 4. 后端职责

- VIP、成家币、订单、流水、退款状态、解锁记录
- 社区动态、诚意贴、评论、点赞、关注、举报、审核流
- 量表、题目、选项、作答、报告、结果标签回写
- 立业首页/作答/报告接口

---

## 5. 管理后台前端职责

- 商业化后台、财务中心相关页面
- 社区内容审核页面
- 量表与测评管理整块后台
- 立业测评配置页、报告模板页

---

## 6. 提供给移动端接口

- 充值、会员、订单、流水、解锁接口
- 社区动态、评论、话题、举报接口
- 量表列表、测评作答、测评报告、立业首页与报告接口
