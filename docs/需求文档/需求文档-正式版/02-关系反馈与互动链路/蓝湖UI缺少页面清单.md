## 设计补充重点

1. 前台失效态必须展示失效原因，不做静默隐藏。
2. 普通用户模糊态不得展示昵称、年龄、学校、清晰头像等强识别字段。
3. 最近看过我的需要明确“今日访客 UV / 今日浏览量 PV / 总浏览量 PV”的视觉层级。
4. 单条解锁与解锁全部必须复用 PRD-04 付费引导，不单独画支付页。
5. 单条解锁弹窗需要画 PRD-02 场景内容差异，但扣币、余额不足和充值承接仍复用 PRD-04 组件。
6. 后台不需要全局关系列表或关系反馈配置页设计稿，只需要 App 用户管理列表字段增强、用户详情内关系记录区块。
7. 关系反馈展示窗口、访客去重、模糊展示条数、隐藏访问记录入口、悄悄话触发匹配等规则按代码固定实现，不要求蓝湖补配置页、保存确认弹窗或配置日志抽屉。
