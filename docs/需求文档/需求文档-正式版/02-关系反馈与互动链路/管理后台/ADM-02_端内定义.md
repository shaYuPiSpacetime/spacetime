# ADM-02 端内定义 - 关系反馈与互动链路

> 本文件登记 PRD-02 在管理后台内专属的信息架构、权限矩阵、审计要求与 UI 状态。
> 管理后台全局通用规则见 `../../全局定义/端专属层_管理后台.md`；PRD-02 跨端业务定义见 `../PRD-02_模块公共定义.md`。
> 首版不新增独立一级“关系反馈工作台”、全局关系列表或关系反馈配置页，能力挂靠在 ADM-01 App 用户管理与运营文案能力中。

|------|------|--------|----------|
| 版本03 | 2026-07-02 | Codex | 按确认口径取消关系反馈配置页，规则参数改为代码固定实现 |

## 1. 信息架构（菜单 / 页面树）

| 一级菜单 | 二级页面 / 区块 | 页面 ID | 一句话职责 | 优先级 |
|----------|----------------|---------|------------|--------|
| 用户管理中心 | App 用户管理列表-关系反馈字段补充 | `ADM-02-PAGE-user-list-relation-fields` | 在既有 App 用户管理列表补充关系反馈筛选条件和统计字段，不新增独立关系列表 | P0 |
| 用户管理中心 | App 用户详情-关系记录区块 | `ADM-02-PAGE-user-relation-section` | 在用户详情中查看关系反馈摘要、喜欢记录、访客记录、匹配记录、解锁记录 | P0 |
| 运营中心 | 文案与消息中心-关系反馈分组 | `ADM-GLB-PAGE-copy-message-center` | 配置匹配成功、空态、失效原因、解锁引导等文案模板 | P1 |
| 内容审核/风控处理 | 处罚联动 | 由 PRD-03/运营审核页面承接 | 处罚、封禁、拉黑等结果触发关系失效 | P0 |

> 不新增全局关系列表页；客服排查从 App 用户管理页定位到用户详情后查看关系记录区块。关系反馈展示窗口、去重、模糊上限、隐藏访问记录开关、悄悄话触发匹配等规则按 `M02-PARAM-*` 代码固定实现，不提供后台配置入口。若后续需要按关系维度全局搜索或运营可配置，另立二期需求。

## 2. 角色与权限矩阵

### 2.1 角色引用

| 角色 code | 显示名 | 本模块中承担的职责 |
|-----------|--------|-------------------|
| `GLB-ROLE-customer-service` | 客服 | 查看用户关系记录，协助排查“看不到喜欢/访客/匹配”的客诉 |
| `GLB-ROLE-operation` | 运营 | 查看关系反馈数据，维护空态、引导文案等通用文案 |
| `GLB-ROLE-risk` | 风控 | 查看因拉黑、封禁、冻结导致的失效记录 |
| `GLB-ROLE-super-admin` | 超级管理员 | 查看全部记录、处理授权导出与审计 |

### 2.2 权限矩阵（ADM-02 涉及操作）

| 操作 / 能力 | 客服 | 运营 | 风控 | 超管 |
|-------------|------|------|------|------|
| 查看关系反馈摘要 | 允许 | 允许 | 允许 | 允许 |
| 查看用户列表关系反馈字段 | 允许 | 允许 | 允许 | 允许 |
| 查看喜欢记录 Tab | 允许 | 允许 | 允许 | 允许 |
| 查看访客记录 Tab | 允许 | 允许 | 允许 | 允许 |
| 查看匹配记录 Tab | 允许 | 允许 | 允许 | 允许 |
| 查看解锁记录摘要 | 允许 | 只读 | 允许 | 允许 |
| 导出关系记录 | 禁止 | 禁止 | 按授权 | 按授权 |
| 手工制造/恢复匹配 | 禁止 | 禁止 | 禁止 | 禁止 |
| 手工补喜欢/访客记录 | 禁止 | 禁止 | 禁止 | 禁止 |

### 2.3 审计要求

| 操作 | 是否二次确认 | 是否记录审计日志 | 审计日志保留 |
|------|-------------|-----------------|--------------|
| 查看用户关系记录详情 | 否 | 是 | >= 1 年 |
| 导出关系记录 | 是 | 是 | >= 1 年 |

## 3. 后台展示标识规则

| 对象 | 后台展示标识 | 说明 |
|------|--------------|------|
| 用户 | 用户 UUID + 昵称/手机号（按权限脱敏） | 不直接展示数据库内部主键 |
| 喜欢记录 | `LIK-yyyymmdd-xxxx` | 用于客服排查 |
| 访客记录 | `VIS-yyyymmdd-xxxx` | 可关联访客来源场景 |
| 匹配记录 | `MAT-yyyymmdd-xxxx` | 展示匹配来源和失效原因 |
| 解锁记录 | `ULK-yyyymmdd-xxxx` | 具体资产流水由 PRD-04 承接 |

---

## 4. UI 空态 / 错误态 / 降级态

| 场景 | 表现 | 默认文案 |
|------|------|----------|
| 用户无关系记录 | Tab 内表格空态 | 暂无关系记录 |
| 搜索/筛选无结果 | 表格空态 + 清除筛选 | 未找到相关记录 |
| 无权限 | Tab 或按钮隐藏/置灰 | 无操作权限 |
| 关系已失效 | 状态标签 + 失效原因 | 已失效：{invalidReason} |
| PRD-04 资产服务不可用 | 解锁记录展示基本信息，资产流水入口提示不可用 | 资产数据暂不可用 |
