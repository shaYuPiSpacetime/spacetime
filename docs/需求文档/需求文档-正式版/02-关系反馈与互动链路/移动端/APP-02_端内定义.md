## 1. 信息架构（页面树）

| 一级入口 / 场景 | 二级页面 / 弹窗 | 页面 ID | 一句话职责 | 优先级 |
|----------------|----------------|---------|------------|--------|
| 荐/婚恋反馈入口 | 喜欢我的列表页 | `APP-02-PAGE-likes-me` | 展示喜欢过我的异性用户，承接模糊展示、单条解锁、会员全量查看和回看主页 | P0 |
| 荐/婚恋反馈入口 | 最近看过我的列表页 | `APP-02-PAGE-recent-viewers` | 展示最近 7 天访问过我婚恋主页的异性用户，承接模糊展示、单条解锁和会员全量查看 | P0 |
| 荐/消息快捷入口 | 相互喜欢列表页 | `APP-02-PAGE-mutual-matches` | 展示当前有效或已失效的匹配成功关系，并提供聊天/主页跳转或失效原因 | P0 |
| 匹配成功流程 | 匹配成功弹窗 | `APP-02-PAGE-match-success-modal` | 首次匹配成功时即时反馈，并引导看主页或进入聊天 | P0 |
| 付费触点 | 单条解锁弹窗场景规格 | `APP-02-PAGE-single-unlock-modal` | 定义喜欢/访客单条解锁的标题、副标题、模糊头像和按钮组合；实际扣费与弹窗基础组件复用 PRD-04 | P0 |
| 付费触点 | 业务场景付费引导弹窗 | `APP-04-PAGE-paywall-modal` | 喜欢/访客单条解锁与解锁全部的统一付费承接 | P0 |
| 用户主页 | 婚恋用户主页 | 由 PRD-05/用户主页规格承接 | PRD-02 只定义进入主页是否生成访客记录及主页按钮状态 | P0 |

## 2. 移动端登录态与核心准入

| 用户状态 | 可见页面/能力 | 不可用能力 | 不满足时处理 |
|----------|---------------|------------|--------------|
| 未登录 | 不可查看关系反馈个人数据 | 喜欢我的、最近看过我的、相互喜欢、喜欢动作、访客记录 | 点击入口后引导登录 |
| 已登录但未完成三重认证 | 可看到入口或强引导态 | 不可进入真实反馈列表，不可形成真实喜欢、访客、匹配、聊天 | 展示 `M02-TXT-core-access-block`，跳转 PRD-01 核心准入拦截/认证引导 |
| 三重认证通过且账号正常 | 可使用关系反馈全部基础能力 | 付费权益仍受 PRD-04 会员/千寻币规则限制 | 正常展示 |
| 账号冻结/停用/注销中 | 仅可查看必要提示 | 不可形成或继续关系反馈 | 按账号状态提示；历史关系前台置失效 |

## 3. 移动端 UI 状态

| 场景 | 页面表现 | 默认文案/引用 |
|------|----------|---------------|
| 喜欢我的无数据 | 空态 + 去完善资料/去看看推荐入口 | `M02-TXT-likes-empty` |
| 最近看过我的无数据 | 空态 + 去完善资料/提高曝光建议 | `M02-TXT-viewers-empty` |
| 相互喜欢无数据 | 空态 + 去推荐页看看 | `M02-TXT-mutual-empty` |
| 未完成三重认证 | 拦截页或弹层 | `M02-TXT-core-access-block` |
| 关系失效 | 卡片置灰/弱化，展示失效原因 | `M02-TXT-relation-invalid` |
| 普通用户未解锁 | 模糊头像/弱识别标签，不展示强识别字段 | `M02-RULE-blur-display` |
| 单条解锁弹窗 | 标题“查看对方资料”，按喜欢/访客区分副标题，按钮为“只看 ta”“解锁全部” | `M02-RULE-single-unlock-modal-content` |
| 会员生效 | 全量清晰展示喜欢/访客记录 | `M04-ENUM-vip-benefit-type=heart_list/visitor_list` |
| 单条已解锁 | 当前记录清晰展示 | `M02-RULE-unlock-visibility` |
| 网络失败 | toast + 保留上次数据 + 重试 | 移动端全局通用态 |

## 4. 页面跳转口径

| 来源 | 目标 | 跳转条件 | 备注 |
|------|------|----------|------|
| 喜欢我的清晰卡片 | 婚恋用户主页 | 记录 `displayStatus=clear` 且未失效 | 进入主页会按 `M02-RULE-visit-generate` 为对方生成访客记录，若用户开启隐藏访问记录则不展示给对方 |
| 最近看过我的清晰卡片 | 婚恋用户主页 | 记录 `displayStatus=clear` 且未失效 | |
| 相互喜欢有效卡片 | 聊天页 | `matchStatus=matched` 且 PRD-03 会话可用 | 需遵守 PRD-03 女性保护机制 |
| 相互喜欢有效卡片 | 婚恋用户主页 | `matchStatus=matched` | 主页主动作展示聊天 |
| 失效卡片 | 无跳转或仅查看原因 | `displayStatus=invalid` 或 `matchStatus=invalid` | 前台展示失效原因，不删除记录 |
| 单条模糊卡片 | PRD-04 付费引导弹窗 | 普通用户未单条解锁 | 单条走千寻币 |
| 解锁全部按钮 | PRD-04 付费引导弹窗 | 普通用户非会员 | 全量走时空邂逅会员 |
