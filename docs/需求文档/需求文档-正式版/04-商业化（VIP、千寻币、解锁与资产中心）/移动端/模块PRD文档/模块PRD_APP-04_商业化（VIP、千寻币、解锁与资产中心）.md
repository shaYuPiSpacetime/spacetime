## 1. 模块目标

移动端 PRD-04 统一承接小程序内所有商业化能力：持续型权益走时空邂逅会员，按次消费走千寻币，会员状态、千寻币余额、订单与流水分别由会员中心、千寻币充值页、会员订单记录页和千寻币流水页承接。首版仅提供“时空邂逅会员”一个会员层级，不做 SVIP、高端服务或人工高端会员商品，也不做独立资产中心页。模块目标是在不破坏核心准入门槛的前提下，让用户能清楚理解“为什么付费、付费得到什么、何时生效、如何追溯”。

**用户故事：** 作为已登录用户，我想在触发受限场景时快速开通会员或充值千寻币，并能通过千寻币流水页和会员订单记录页追溯资产变化，以便持续使用平台的高级权益和按次解锁能力。

**核心指标：** 会员中心访问转化率、千寻币充值转化率、支付成功率、付费后权益可用率、支付/到账客诉率。

## 3. 关键业务流程

### 3.1 会员购买与权益生效流程

```
入口：我的 -> 时空邂逅会员，或业务场景会员引导弹窗
正常路径：
  1. 用户进入会员中心，查看当前会员状态、9 项权益和套餐；权益列表图标取后台 `M04-CFG-vip-benefit-list.mobileIcon`
  2. 选择普通套餐或连续订阅套餐；不出现 SVIP/高端服务套餐；连续订阅套餐接入微信真实自动续费扣款
  3. 勾选对应协议后发起微信支付
  4. 支付成功回调进入 `M04-SM-trade-order=success`
  5. 会员状态进入 `M04-SM-vip-status=active`
  6. 来源页/会员中心刷新权益
  7. 若购买连续订阅套餐，会员中心提供连续订阅管理入口，可查看订阅状态与取消续费指引
异常：
  - 支付失败：订单为 failed，不变更资产，可重试
  - 支付取消或 30 分钟未支付：订单为 closed，不变更资产
  - 未完成三重认证：可开通，但使用真实曝光、喜欢、私信、社区权益时按 `M01-RULE-core-access` 拦截
出口：返回来源页、会员中心或会员订单记录页
```

### 3.2 千寻币充值与按次消费流程

```
入口：我的 -> 千寻币，或余额不足/单条解锁/悄悄话等付费弹窗
正常路径：
  1. 用户查看当前余额和后台配置的千寻币套餐
  2. 查看按后台配置收口的 8 个消费场景说明；消费场景展示名称取后台 `M04-CFG-coin-scene-price.mobileDisplayName`，图标取后台 `M04-CFG-coin-scene-price.mobileIcon`
  3. 选择套餐并勾选千寻币充值协议后发起支付
  4. 支付成功后写充值流水，刷新余额
  5. 若来自业务场景，回到来源场景继续解锁/发送/查看
按次消费：
  - 余额充足：确认后扣减千寻币，写 `consume` 流水和解锁记录
  - 余额不足：在弹窗内展示 1-2 个快捷充值套餐和更多套餐入口，不允许透支
异常：
  - 重复点击/重复回调：按 `M04-RULE-order-idempotency` 幂等处理
  - 解锁对象失效：不扣币，提示对象不可用
```

### 3.3 资产记录追溯流程

```
入口：我的页相关入口、会员中心、千寻币充值页，或支付结果页快捷入口
用户可查看：
  1. 会员中心展示当前会员状态、到期时间、免费悄悄话等会员权益摘要
  2. 千寻币充值页和千寻币流水页展示当前千寻币余额
  3. 千寻币流水页展示充值、消费、奖励、退款退回等记录
  4. 会员订单记录页展示会员订单、连续订阅标识和退款状态
异常：
  - 流水/订单为空：展示空态
  - 网络失败：保留上次数据并提供重试
```

## 9. 非功能性需求

|------|------|
