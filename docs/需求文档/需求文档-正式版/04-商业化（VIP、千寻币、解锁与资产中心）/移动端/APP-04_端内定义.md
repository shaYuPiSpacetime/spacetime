## 1. 信息架构（页面树）

| 一级入口 / 场景 | 二级页面 / 弹窗 | 页面 ID | 一句话职责 | 优先级 |
|----------------|----------------|---------|------------|--------|
| 我的 | 时空邂逅会员中心 | `APP-04-PAGE-vip-center` | 展示会员状态、权益、套餐并承接会员购买 | P0 |
| 我的 | 千寻币充值页 | `APP-04-PAGE-coin-recharge` | 展示余额、套餐并承接千寻币充值 | P0 |
| 我的 | 资产中心页 | `APP-04-PAGE-asset-center` | 汇总会员状态、千寻币余额、流水和订单入口 | P0 |
| 资产中心/会员中心 | 连续订阅管理页 | `APP-04-PAGE-subscription-manage` | 查看连续订阅状态、续费说明和取消续费入口 | P0 |
| 资产中心 | 千寻币流水页 | `APP-04-PAGE-coin-flow` | 分页展示充值、消费、奖励、退款等千寻币流水 | P0 |
| 资产中心 | 会员订单记录页 | `APP-04-PAGE-vip-orders` | 分页展示会员订单、支付状态与退款状态 | P0 |
| 支付流程 | 支付结果承接页/弹窗 | `APP-04-PAGE-payment-result` | 承接支付成功、失败、取消后的反馈与回跳 | P0 |
| 业务付费触点 | 付费引导弹窗 | `APP-04-PAGE-paywall-modal` | 统一承接 VIP 引导、千寻币余额不足、单条解锁确认 | P0 |

> 入口展示名继承移动端全局定义：`我的 -> 时空邂逅会员`、`我的 -> 千寻币`。旧称“成家币”不得出现在正式前台页面文案中。

## 2. 登录态与核心准入

| 用户状态 | 可见页面/能力 | 不可用能力 | 不满足时处理 |
|----------|---------------|------------|--------------|
| 未登录 | 可见部分业务触点 | 不可查看资产、不可下单、不可解锁 | 点击后引导登录 |
| 已登录但未完成三重认证 | 可进入会员中心、千寻币充值、资产中心、可购买资产 | 涉及真实曝光、喜欢、私信、社区的权益实际使用受限 | 购买前/购买成功后提示 `M04-TXT-core-access-purchase-tip`；使用时按 `M01-ERR-core-access-blocked` 拦截 |
| 三重认证通过且账号正常 | 可购买并使用当前权益范围内能力 | 无 PRD-04 额外限制 | — |
| 账号冻结/停用/注销中 | 可按账号状态查看必要订单记录（若产品允许） | 社交权益、支付、解锁不可用 | 按账号状态提示，资产记录保留 |

## 3. 移动端 UI 空态 / 错误态 / 降级态

| 场景 | 页面表现 | 默认文案 |
|------|----------|----------|
| 套餐无可售数据 | 套餐区空态 + 不展示支付按钮 | 暂无可购买套餐 |
| 千寻币流水无数据 | 居中空态 | 暂无千寻币流水 |
| 会员订单无数据 | 居中空态 | 暂无会员订单 |
| 支付服务不可用 | 支付按钮置灰 + toast | 支付暂不可用，请稍后重试 |
| 支付成功但刷新失败 | 成功页展示成功结果 + 资产刷新重试 | 支付成功，资产刷新中 |
| 未完成三重认证 | 购买页/成功页提示条 | 完成三重认证后，相关社交权益才会实际生效 |
| 余额不足 | 付费弹窗提示余额不足并展示 1-2 个快捷充值套餐和更多套餐入口 | 千寻币余额不足，请先充值 |
| 连续订阅服务不可用 | 连续订阅 Tab 置灰；订阅管理页展示重试或取消指引 | 连续订阅暂不可用 |
