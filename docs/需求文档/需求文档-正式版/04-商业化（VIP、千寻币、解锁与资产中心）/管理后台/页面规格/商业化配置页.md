## 1. 页面定位

- **目标用户**：运营、超级管理员
- **核心任务**：配置会员权益/套餐、千寻币套餐、场景单价、保留期、社交与订单参数和曝光包预留
- **页面类型**：多 Tab 配置页

---

## 2. 布局

```text
Tab: 会员权益 | 会员套餐 | 千寻币套餐 | 场景单价 | 解锁保留期 | 社交与订单参数 | 曝光包预留
Tab 内容配置表/表单 -> 底部固定保存按钮（二次确认） -> 右侧变更日志抽屉
```

| 区块 | 位置 | 内容 | 是否可折叠 | 是否记住展开状态 |
|------|------|------|-----------|----------------|
| 会员权益 Tab | Tab1 | 系统内置权益启停；免费悄悄话额外配置每日次数；名称、类型、说明只读；不提供排序 | 否 | 是 |
| 会员套餐 Tab | Tab2 | 普通套餐/连续订阅套餐、原价、优惠价、时长/周期、标签、上架状态、协议状态 | 否 | 是 |
| 千寻币套餐 Tab | Tab3 | 千寻币套餐、原价、优惠价、到账币数、赠送币数、标签、推荐档、上架状态 | 否 | 是 |
| 场景单价 Tab | Tab4 | 6 个千寻币消费场景单价，不包含 `invite_*` 奖励场景 | 否 | 是 |
| 解锁保留期 Tab | Tab5 | 理想型批量上限、理想型保留天数、精选主页回看天数 | 否 | 是 |
| 社交与订单参数 Tab | Tab6 | 普通/会员每日查看配额、会员到期提醒提前天数、退款展示开关；未支付订单固定 30 分钟仅做说明 | 否 | 是 |
| 曝光包预留 Tab | Tab7 | 曝光包场景占位开关与说明 | 否 | 是 |

### 2.2 编辑控件口径

| 配置区域 | 展现形式 | 编辑控件 | 新增/删除方式 | 保存方式 |
|----------|----------|----------|----------------|----------|
| 会员权益 | 固定表格 | 启停开关；免费悄悄话每日次数数字输入 | 不允许新增、删除、排序；名称、类型、说明均只读 | 底部统一保存 |
| 会员套餐 | 列表表格 | 编辑弹窗、状态开关、标签多选 | 新增/编辑均打开弹窗；下架需二次确认 | 底部统一保存 |
| 千寻币套餐 | 列表表格 | 编辑弹窗、数字输入、推荐单选 | 新增/编辑均打开弹窗；推荐档互斥 | 底部统一保存 |
| 场景单价 | 固定场景表格 | 数字输入、启停开关 | 不允许新增场景；场景由 `M04-ENUM-coin-biz-scene` 消费场景限定 | 底部统一保存 |
| 解锁保留期 | 表单 | 数字输入、步进器 | 不涉及新增 | 底部统一保存 |
| 社交与订单参数 | 表单 | 数字输入、开关、单位选择 | 不涉及新增 | 底部统一保存 |
| 曝光包预留 | 表单 | 开关、说明文本域 | 不涉及新增 | 底部统一保存 |

## 6. 数据联动规则

| 触发字段 | 触发事件 | 影响字段 | 联动行为 | 备注 |
|----------|----------|----------|----------|------|
| 连续订阅套餐 | 上架 | 微信连续订阅商品/协议 | 未绑定微信连续订阅商品、周期或协议时阻止上架 | |
| 千寻币套餐 | 设置推荐档 | 其他套餐推荐标记 | 同一时间最多 1 个推荐档 | |
| 场景单价 | 关闭场景 | 移动端触点 | 对应触点隐藏或不可点击 | |
| 社交与订单参数 | 修改查看配额 | APP 关系/推荐链路 | 新配额只影响保存后的新访问判定，不回滚历史已用次数 | |
| 会员到期提醒提前天数 | 修改 | 通知任务 | 保存后影响后续未触发提醒的会员 | `M04-NTF-vip-expire-remind` |
| 退款展示开关 | 关闭 | APP 订单记录 | 前台隐藏退款标签，后台订单/退款台账仍完整展示 | `M04-RULE-refund-display` |
| 曝光包预留开关 | 开启 | 移动端曝光包占位 | 仅展示预留说明，不允许购买 | `M04-RULE-exposure-pack-reserve` |

## 7. 状态与异常

| 状态类型 | 触发场景 | 页面表现 | 用户可做的操作 | 引用 |
|----------|----------|----------|---------------|------|
| 加载态 | 首次加载配置 | 骨架屏 | 无 | — |
| 空态 | 配置未初始化 | 展示初始化提示 | 新增配置 | `M04-ERR-config-missing` |
| 错误态 | 保存失败 | toast + 表单错误定位 | 修正后重试 | — |
| 无权限态 | 无配置权限 | 页面只读或菜单隐藏 | 无 | `../ADM-04_端内定义.md` |
| 业务态-套餐下架 | 套餐状态=`off` | 行置灰，历史订单仍可查 | 重新上架/编辑 | `M04-ENUM-package-status` |
| 降级态 | 微信连续订阅商品校验失败 | 连续订阅套餐不可上架，提示补齐商品/协议配置 | 完善配置后重试 | `M04-SRV-wechat-subscription` |

## 9. 验收标准

```text
AC-ID: ADM-04-AC-config-save
Given 运营具备配置权限，修改一个千寻币套餐的到账币数
When  点击保存并确认二次弹窗
Then  M04-CFG-coin-package-list 更新，移动端配置缓存失效，审计日志记录操作人、变更前后值和时间

AC-ID: ADM-04-AC-config-subscription-block
Given 连续订阅套餐未绑定微信连续订阅商品或协议
When  运营尝试上架连续订阅套餐
Then  保存被阻断，提示补齐微信连续订阅商品、周期和协议配置

AC-ID: ADM-04-AC-config-benefit-readonly
Given 运营进入会员权益 Tab
When  查看全量查看、高级筛选、隐私权益和免费悄悄话
Then  名称、类型、说明均不可编辑，不展示排序控件；只有启停可配置，免费悄悄话额外展示每日次数输入
```

### 验收标准清单

| AC ID | 场景 | 类型 | 优先级 |
|-------|------|------|--------|
| `ADM-04-AC-config-save` | 配置保存生效 | 正常 | P0 |
| `ADM-04-AC-config-subscription-block` | 连续订阅配置缺失拦截 | 异常 | P0 |
| `ADM-04-AC-config-benefit-readonly` | 会员权益只读字段与可配置字段 | 正常 | P0 |
