## 1. 页面定位

- **目标用户**：客服、财务、运营、风控、超级管理员
- **核心任务**：围绕单个用户排查资产、订单、流水、解锁和退款问题；首版只查看会员状态/订单，不提供连续订阅状态同步或代用户取消续费操作
- **页面类型**：用户管理详情内嵌 Tab/弹窗内容区；不作为 ADM-04 独立菜单页

---

## 2. 布局

```text
商业化摘要卡 + 疑似异常提示 -> Tab: 会员订单 | 千寻币流水 | 解锁记录 | 退款记录 -> 详情抽屉/人工修正弹窗
```

| 区块 | 位置 | 内容 | 是否可折叠 | 是否记住展开状态 |
|------|------|------|-----------|----------------|
| 商业化摘要卡 | Tab 顶部 | 会员状态、到期时间、千寻币余额、免费悄悄话、累计充值、累计消费、最近购买、最近解锁；不展示订阅同步按钮 | 否 | 否 |
| 疑似异常提示 | 摘要卡下方 | 余额与流水不一致、退款回退失败、幂等异常等风险提示 | 条件展示 | 否 |
| 会员订单 Tab | 明细区 Tab1 | 当前用户会员订单和千寻币充值订单摘要 | 否 | 是 |
| 千寻币流水 Tab | 明细区 Tab2 | 当前用户资产流水 | 否 | 是 |
| 解锁记录 Tab | 明细区 Tab3 | 当前用户单条/按次解锁记录 | 否 | 是 |
| 退款记录 Tab | 明细区 Tab4 | 当前用户退款记录 | 否 | 是 |
| 人工修正弹窗 | 页面居中 | 修正资产类型、变动数量、原因、二次确认 | 否 | 否 |

## 3. 筛选与搜索

本页嵌入用户详情，用户身份由路由 `:id` 固定，不提供跨用户搜索。各明细 Tab 内仅保留轻量筛选：时间范围、状态/类型筛选；默认展示最近 30 天。

## 8. 查询与列表

- **默认排序**：各 Tab 按发生时间倒序。
- **分页**：每个 Tab 独立分页，默认 10 条/页。
- **默认范围**：最近 30 天；用户详情内不提供跨用户搜索。
- **批量操作**：首版不支持批量人工修正、批量退款或批量解锁回退。

## 9. 验收标准

```text
AC-ID: ADM-04-AC-user-asset-summary
Given 客服进入一个已开通会员且有千寻币余额的用户详情
When  切换到商业化 Tab
Then  摘要区展示会员状态、到期时间、千寻币余额、累计充值和最近购买时间

AC-ID: ADM-04-AC-user-asset-adjust-permission
Given 客服角色打开商业化 Tab
When  查看人工修正资产入口
Then  入口不展示或置灰；只有超管可操作且必须填写原因

AC-ID: ADM-04-AC-user-asset-no-subscription-sync
Given 客服或财务角色打开商业化 Tab
When  查看会员摘要和会员订单
Then  页面只展示会员状态与订单结果，不展示连续订阅状态同步或代用户取消续费入口
```
