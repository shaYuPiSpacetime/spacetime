## 1. 模块目标

后台 PRD-04 的目标是支撑移动端商业化的配置、查询、排查、轻量财务核对和合规审计。首版不建设独立商业化运营工作台，不做复杂退款审批流和 BI 平台；后台只查看会员状态/订单，不提供连续订阅状态同步、代用户取消续费等操作；demo 中积分管理不属于 PRD-04 首版，千寻币为唯一付费虚拟币口径。核心能力收敛到移动端配置、用户管理详情、财务中心、文案与消息中心、系统参数和审计。

**用户故事：** 作为运营、客服、财务或超级管理员，我想配置会员/千寻币商品，查询用户资产、订单、流水和退款记录，以便保障移动端付费链路可上线、可排查、可追溯。

**核心指标：** 配置生效准确率、订单查询命中率、到账异常处理时长、财务导出准确率、敏感操作审计覆盖率。

## 3. 关键业务流程

### 3.1 商业化配置流程

```
入口：移动端配置管理 -> 商业化配置
正常路径：
  1. 运营进入商业化配置页
  2. 配置 9 项会员权益、会员套餐、千寻币套餐、千寻币消费场景、解锁保留期、社交与订单参数、曝光包预留；会员权益和千寻币消费场景均维护移动端图标
  3. 高风险配置保存时二次确认
  4. 保存后立即生效，写审计日志
异常：
  - 套餐价格/币数缺失：校验阻断
  - 上架套餐无协议/支付配置：校验阻断或置为下架
  - 连续订阅商品/协议未配置：对应套餐不允许上架
  - 出现 SVIP/高端服务或积分管理配置：不纳入 PRD-04 首版，页面不展示
```

### 3.2 客服排查流程

```
入口：用户管理中心 -> 用户详情 -> 商业化详情 Tab
客服可查看：
  1. 用户会员状态、到期时间、千寻币余额
  2. 会员订单、千寻币订单/流水、解锁记录、退款记录
  3. 支付异常或到账异常关联订单
边界：
  - 客服不直接改订单状态、加币或延长会员
  - 客服不触发微信连续订阅状态同步，不代用户取消续费
  - 如需人工修正，升级给超级管理员并记录原因
```

### 3.3 财务查询与轻量对账流程

```
入口：财务中心 -> 商业化订单/资产流水/退款/轻量对账
正常路径：
  1. 财务按订单号、用户、时间、状态、金额筛选
  2. 查看订单详情、流水明细或退款记录；需要特批退款时在订单详情中发起退款
  3. 按权限导出
  4. 轻量对账页按日汇总订单数、支付金额、退款金额
边界：
  - 不做支付渠道级自动差错追账
  - 不做连续订阅状态后台同步或渠道订阅状态主动拉取
  - 不做复杂退款审批工作流
```

## 9. 非功能性需求

### 9.1 权限

权限矩阵以 `../ADM-04_端内定义.md` 第 2 节为准；后端接口必须校验权限，不允许仅前端隐藏按钮。

### 9.2 安全与合规

| 字段 | 加密存储 | 脱敏规则 | 留存时长 | 注销后处理 | 是否可导出 |
|------|---------|----------|----------|-----------|-----------|
| 手机号 | 是/按用户体系 | 138****1234 | 按用户体系 | 按用户体系 | 按权限 |
| 订单号/流水号 | 否 | 不脱敏 | 长期 | 保留财务台账，用户标识匿名化 | 是 |
| 支付渠道流水号 | 否/按支付规范 | 后台按权限展示 | 长期 | 保留财务台账 | 是 |
| 退款原因 | 否 | 后台按权限展示 | 长期 | 保留财务台账 | 是 |

### 9.3 性能

| 场景 | 要求 |
|------|------|
| 主列表查询 | 默认 10 条/页，常规筛选 500ms-1s 内返回 |
| 导出 | 异步生成，进入下载中心 |
| 配置保存 | 保存后 1 分钟内移动端配置缓存失效 |
| 轻量对账 | 按日聚合，可接受分钟级延迟 |

### 9.4 并发与幂等

| 需幂等的操作 | 并发场景 | 幂等方案建议 |
|-------------|----------|-------------|
| 配置保存 | 多管理员同时修改 | 版本号/乐观锁 + 审计 |
| 支付成功回调 | 渠道重复回调 | orderNo + channelTradeNo 唯一 |
| 退款状态同步 | 渠道重复回调/状态重复同步 | refundNo 唯一 |
| 资产人工修正 | 超管重复提交 | requestId + 审计原因 |
| 导出 | 重复点击导出 | 异步任务去重或生成多任务均可审计 |
