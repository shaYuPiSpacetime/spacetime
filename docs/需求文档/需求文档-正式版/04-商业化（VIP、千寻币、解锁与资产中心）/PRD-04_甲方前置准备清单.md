## 1. 甲方必须准备的商业化资料

| 序号 | 准备项               | 甲方需要给什么                                                       |
| ---- | -------------------- | -------------------------------------------------------------------- |
| 1    | 会员协议             | 正式协议文本、协议名称、生效时间、适用主体、客服/投诉联系方式        |
| 2    | 连续订阅服务协议     | 自动续费周期、扣费规则、取消续费说明、续费前提示口径、协议生效时间   |
| 3    | 千寻币购买与使用说明 | 千寻币用途、不过期说明、不可提现/不可转让/不可兑换现金等限制文案     |
| 4    | 退款与售后说明       | 前台不开放主动退款申请的说明口径、客服处理入口、特批退款适用场景     |
| 5    | 支付结果与到账说明   | 支付成功、支付失败、支付取消、到账延迟、资产刷新失败时给用户看的文案 |
| 6    | 会员权益展示文案     | 心动名单、访客、免费悄悄话、额外浏览、高级筛选、曝光、隐私权益、三天回放、每日心动机会的最终对外说明文案 |
| 7    | 移动端展示名称与图标素材 | 会员权益 9 项图标素材，以及千寻币消费场景 8 项的移动端展示名称、最终图标素材、图标命名或资源 ID、默认/禁用态展示规范；需包含“立业-职业推荐”图标素材 |
| 8    | 商业化客服入口       | 小程序内客服入口、客服电话/邮箱、客服在线时间、客诉流转负责人        |
| 9    | 财务主体信息         | 收款主体、开票主体、公司名称、纳税人识别号、财务联系人               |

## 2. 甲方必须准备的微信支付与订阅条件

| 序号 | 准备项           | 甲方需要确认什么                                                 |
| ---- | ---------------- | ---------------------------------------------------------------- |
| 1    | 微信支付商户能力 | 小程序是否已绑定可用微信支付商户号，商户号主体是否与业务主体一致 |
| 2    | 连续订阅能力     | 微信小程序连续订阅能力是否已开通，商品是否已创建并通过微信侧审核 |
| 3    | 订阅商品映射     | 每个连续订阅套餐对应的微信商品 ID、周期、金额、协议链接          |
| 4    | 退款能力         | 商户号是否允许后台发起退款，退款权限由哪个角色/账号持有         |
