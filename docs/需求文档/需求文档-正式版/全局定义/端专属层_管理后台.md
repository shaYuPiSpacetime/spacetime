# 全局定义（端专属层）- 管理后台

> 本文件是管理后台正式端专属层母文件，统一登记管理后台全局通用的信息架构原则、角色基线、权限控制原则、敏感操作审计基线与 UI 通用态。
> 单模块菜单树、模块权限矩阵、模块审计清单、模块端内 UI 状态放在对应模块的管理后台端内定义中，不放本文；跨端或多页面复用的模块文案 key 放模块公共定义。
> 跨模块项目级业务定义在 `共享层_项目级.md`；单模块跨端业务定义在对应模块公共定义中。

## 2. 角色基线

| 角色 code | 显示名 | 全局职责口径 |
|-----------|--------|--------------|
| `GLB-ROLE-operation` | 运营 | 负责运营配置、活动/内容运营与业务数据查看 |
| `GLB-ROLE-customer-service` | 客服 | 负责用户咨询、申诉协助、账号状态处理与受控人工纠错 |

| `GLB-ROLE-super-admin` | 超级管理员 | 负责全后台配置、权限、敏感操作与审计查看 |

## 3. 权限控制原则

| 原则 | 说明 |
|------|------|
| 菜单权限 | 无菜单权限时不展示对应入口 |
| 页面权限 | 无页面权限时页面不可访问或展示无权限态 |
| 操作权限 | 无操作权限时按钮隐藏或置灰，禁止仅前端控制 |
| 数据权限 | 数据范围按角色/组织/负责人控制，模块需在端内定义补充 |
| 导出权限 | 导出敏感数据必须单独授权并记录审计日志 |

## 4. 敏感操作审计基线

| 操作类型 | 是否二次确认 | 是否记录审计日志 | 审计日志保留 |
|----------|-------------|-----------------|-------------|
| 修改高风险配置 | 是 | 是 | >= 1 年 |
| 人工处理风险/异常记录 | 是 | 是 | >= 1 年 |
| 生成/变更结算或财务状态 | 是 | 是 | >= 1 年 |
| 导出敏感数据 | 是 | 是 | >= 1 年 |

## 5. UI 通用空态 / 错误态 / 降级态

| 场景 | 表现 | 默认文案 |
|------|------|----------|
| 列表无数据 | 居中插画 + 文案 | 暂无数据 |
| 搜索无结果 | 居中插画 + 清除筛选建议 | 未找到相关记录 |
| 请求失败（网络/服务端） | toast + 重试按钮 | 加载失败，请重试 |
| 无权限 | 菜单/按钮不展示或置灰 | 无操作权限 |
| 导出生成中 | loading + 异步生成提示 | 导出生成中，请稍后在下载中心查看 |

## 6. 字段展示通用规则

| 字段类型 | 管理后台展示要求 | 说明 |
|----------|------------------|------|
| 数据库内部主键 ID | 展示层尽量不直接展示；接口、路由参数、内部定位可继续使用内部 ID | 页面可见字段优先使用业务编号、UUID、订单号、流水号、名称等业务标识；模块需在端内定义补充具体映射 |
| 枚举 / 状态 | 只展示中文名称 | 列表、详情、筛选项、状态胶囊、操作条件、验收标准、导出均展示中文；code 仅作为接口/字典内部值或文档引用 ID |
| datetime 时间 | `yyyy-MM-dd HH:mm:ss` | 所有列表、详情、导出中的 datetime 字段统一展示到秒 |

## 7. 后台 UI 布局与组件规范

> 本节定义管理后台所有业务页面的通用 UI 结构与组件口径。模块页面规格只描述业务字段、筛选条件、操作和异常态；默认继承本节布局与组件规范，除非页面规格明确说明差异。

### 7.1 全局布局

| 区域 | 规范 |
|------|------|
| 左侧导航 | 固定左侧菜单，宽度 170px；支持多级菜单展开/收起；当前页面菜单高亮；菜单项使用 lucide 图标。 |
| 顶部栏 | 固定顶部区域，高度 68px；承载全局标题、账号、登出等统一操作。 |
| 内容区 | 内容区避让左侧导航与顶部栏，默认内边距 24px；页面背景使用后台统一浅色背景。 |
| 页面间距 | 页面根容器默认使用纵向 16px 间距；多个业务区块按从上到下排列。 |

### 7.2 页面主体结构

| 页面类型 | 默认结构 |
|----------|----------|
| 列表页 | `Card` 容器 → `CardHeader` 标题/页面级操作 → `CardContent` 筛选栏、表格、分页。 |
| 表单页 | `Card` 容器 → 标题区 → 表单区 → 底部操作区；复杂配置可使用 Tab 分组。 |
| 详情页 | 顶部概览区 → 详情字段区/时间轴/明细 Tab；关键操作放在概览区右侧。 |
| 多 Tab 页 | 顶部使用横向 Tab；Tab 下方复用列表页或表单页结构；Tab 切换后查询条件按页面需要重置或保留。 |

### 7.3 筛选栏

| 项 | 规范 |
|----|------|
| 位置 | 列表页表格上方，位于 `CardContent` 首部。 |
| 布局 | 横向排列，空间不足时换行；控件间距 12px。 |
| 控件宽度 | 文本搜索框常用 176-224px；状态/枚举下拉常用 128-176px；日期范围按页面需要加宽。 |
| 触发方式 | 简单列表可输入即查询；复杂筛选默认点击「查询」触发，并提供「重置」。页面规格可明确。 |
| 当前筛选 | 如需 URL 持久化或筛选标签，由页面规格单独声明；默认不强制。 |

### 7.4 表格

| 项 | 规范 |
|----|------|
| 基础样式 | 使用后台统一 `Table` 组件；外层支持横向滚动；表格文字默认 14px。 |
| 表头 | 表头高度 66px，左对齐，常规字重，使用次级文字颜色。 |
| 表体 | 行高 56px；行 hover 使用浅色背景；最后一行不显示底部分隔线。 |
| 空/加载 | 加载中、暂无数据、搜索无结果使用整行居中文案；复杂页面可补充插画。 |
| 操作列 | 默认放最右侧；图标型操作使用 `ghost + icon` 按钮，必要时加 `title` 或 tooltip；文本操作使用 `ghost + sm`。 |
| 字段展示 | 空值统一显示 `-`；枚举/状态显示中文；时间展示到秒；敏感字段按权限脱敏。 |

### 7.5 分页

| 项 | 规范 |
|----|------|
| 位置 | 列表页表格下方，靠左展示总数和页码控件。 |
| 默认分页 | 默认每页 10 条；可选 10/20/50 条/页；业务 PRD 要求默认 20 条时，页面规格需明确。 |
| 控件 | 显示首页、上一页、页码、下一页、末页；超过 7 页时使用省略号。 |
| 数据交互 | 页码从 1 开始；翻页只改变页码，不清空筛选条件。 |

### 7.6 弹窗

| 项 | 规范 |
|----|------|
| 使用场景 | 新增/编辑表单、处理确认、详情预览、分配配置等短流程操作。 |
| 基础样式 | 居中弹窗；默认最大宽度约 512px，复杂表单可放宽到约 672px；遮罩点击和 Esc 可关闭。 |
| 结构 | `DialogHeader` 包含标题与说明；主体表单位于标题下方；底部右侧放「取消」和主操作按钮。 |
| 表单布局 | 少量字段用单列；较多字段可用双列网格；字段标签使用 14px 中等字重。 |
| 高风险操作 | 删除、作废、发放、结算、规则生效等敏感操作必须二次确认，必要时填写备注。 |

### 7.7 按钮、状态标签与图标

| 类型 | 规范 |
|------|------|
| 主操作按钮 | 使用主色按钮；放在标题区右侧或弹窗底部右侧，例如「新增」「保存」「确认」。 |
| 次操作按钮 | 使用 outline 或 ghost；例如「重置」「取消」「查看」。 |
| 危险操作 | 使用 destructive 语义或危险色图标；必须配合权限与二次确认。 |
| 图标 | 优先使用 lucide 图标；图标按钮尺寸常用 32px 或 36px；不使用自绘图标替代已有图标。 |
| 状态标签 | 使用 `Badge`；成功/启用用 success，待处理/冻结/已确认用 warning，禁用/无效/终止/取消用 destructive 或 secondary。 |
