## 1. 页面定位

- **目标用户**：超级管理员
- **核心任务**：配置移动端准入、资料、认证规则
- **页面类型**：配置工作台
- **配置事实源**：配置项 ID、默认值与约束引用 `../../PRD-01_模块公共定义.md` 的 `M01-CFG-*`；用户字段来源引用 `M01-DATA-user-input-fields`；本页只定义管理后台如何分组、展示、行内开关、表单编辑、按 Tab 保存和审计。

## 2. 页面布局

| 区域 | 内容 | 规则 |
|------|------|------|
| 顶部栏 | 页面名、面包屑、`查看变更日志` | 顶部不放页面级保存，只保留日志查看入口 |
| 风险提示区 | 高风险项保存需二次确认并记录审计；配置页不是分页列表页，不展示公共分页 | 固定展示在 Tab 上方 |
| Tab 区 | 准入门槛、字段配置、资料完整度、上传限制、审核 SLA、文案配置、安全策略 | 每个 Tab 的展示和编辑结构不同；切换 Tab 前如有未保存变更需二次提示 |
| 配置内容区 | 当前 Tab 的配置卡片、配置表格、模板列表和 Tab 内操作栏 | 配置项优先在当前 Tab 内直接编辑；保存按钮放在当前 Tab 内 |

UI 样式必须继承管理后台全局规范：左侧导航 + 顶部栏 + 浅色内容背景 + 白色配置卡片；所有 Tab 都在同一主内容卡片内切换，表格使用后台统一表格样式，输入框、开关、按钮、状态标签遵循 `../../../全局定义/端专属层_管理后台.md` 7.7。

### 2.1 顶部操作定义

| 操作 | 功能说明 | 展示内容 | 数据来源 |
|------|----------|----------|----------|
| 查看变更日志 | 查看当前页面全部配置项的历史变更记录 | 配置项、所属 Tab、操作人、操作时间、变更前值、变更后值、变更原因、是否高风险、审计单号 | 管理后台配置变更审计；对应 `ADM-01_端内定义.md` 的“修改准入配置/系统字典”审计 |

> 顶部栏不承接保存。每个 Tab 自己展示编辑和保存动作；保存只对当前 Tab 生效，不跨 Tab 批量保存。若多个 Tab 都有草稿，Tab 标题展示未保存标记，需分别进入对应 Tab 保存。

### 2.3.1 准入门槛 Tab 编辑口径

| 区块 | 字段 | 编辑方式 | 默认值/约束 |
|------|------|----------|-------------|
| 年龄范围 | 最小年龄、最大年龄 | 数字输入框或步进器直接编辑 | 默认 18-60；最小年龄不得小于 18，最大年龄不得小于最小年龄 |
| 核心准入 | 实名认证通过、头像认证通过、学历认证通过 | 锁定开关，只读展示 | 三项默认全部开启，不允许降级为单实名或双认证 |
| 账号状态限制 | 账号正常、未冻结、未注销 | 开关或只读状态项 | 账号异常不得开放核心准入 |
| 影响能力 | 曝光、喜欢、私信、社区发布 | 只读标签 | 引用 `M01-RULE-core-access`，不在本页单独改业务范围 |

准入门槛没有单独“编辑”按钮。管理员直接修改输入框/开关后，右下角 `保存准入门槛` 才生效；保存时要求填写变更原因，并触发高风险二次确认。

### 2.3.2 字段配置 Tab 列表口径

字段配置 Tab 必须将 `M01-DATA-user-input-fields` 正式字段池全部列出。表格列固定如下：

| 列名 | 展示/操作口径 |
|------|---------------|
| 字段组 | 直接展示字段池中的字段组，例如登录账号、轻量资料、基础资料、扩展资料、联系方式、头像认证、实名认证、学历认证 |
| 显示名 | 直接展示字段池显示名；字段 ID 可作为悬浮提示或显示名下方小字，不单独占一列 |
| 页面承载 | 直接展示字段池页面承载；多个页面用逗号或换行展示 |
| 是否展示 | 开关展示，列表级可操作；不可编辑字段置灰并展示默认开/关或条件展示状态 |
| 是否必填 | 开关展示，列表级可操作；不可编辑字段置灰并展示固定必填/条件必填状态 |
| 是否计分控制 | 开关展示，列表级可操作；不可编辑字段置灰并展示固定不计分状态；打开后进入资料完整度 Tab |

字段配置联动规则：

1. 不提供新增字段配置、编辑字段、删除字段按钮；字段来源只允许来自 `M01-DATA-user-input-fields`。
2. `是否展示=关` 时，系统必须同步置为 `是否必填=关`、`是否计分控制=关`；保存时不得出现“不展示但必填”或“不展示但计分”的组合。
3. `是否必填=开` 前必须校验 `是否展示=开`；若用户先关闭展示，再打开必填，页面需阻断并提示先开启展示。
4. `是否计分控制=开` 后，该字段进入资料完整度 Tab；关闭后从资料完整度 Tab 移除，已有分值草稿同步清空。
5. 登录账号、协议确认、实名认证、学历认证流程材料、定位坐标、短信验证码、微信授权信息等流程或系统字段不可编辑对应开关；开关置灰，默认值按本节“字段开关默认值”展示。

字段开关默认值：

| 字段范围 | 是否展示默认 | 是否必填默认 | 是否计分默认 |
| -------- | ------------ | ------------ | ------------ |
| 登录账号、协议确认 | 固定展示或按登录方式条件展示 | 固定必填或流程校验 | 固定不计分 |
| 性别、出生日期 | 开 | 开 | 开 |
| 身份、最高学历、现居地 | 开 | 关 | 开 |
| 基础资料、扩展资料中的后台配置字段 | 开 | 关 | 由资料完整度初始化权重决定；纳入计分的字段默认开 |
| 定位坐标、微信授权信息、短信验证码 | 条件采集或系统流程字段，不作为资料展示字段 | 按流程条件校验 | 固定不计分 |
| 实名认证、学历认证流程字段 | 固定展示在对应流程或审核页 | 固定必填或条件必填 | 固定不计分，除字段池已明确允许计分的头像/资料展示类字段外 |

### 2.3.3 资料完整度 Tab 列表口径

资料完整度 Tab 只展示字段配置 Tab 中 `是否计分控制=开` 的字段。表格列固定如下：

| 列名 | 展示/操作口径 |
|------|---------------|
| 字段组 | 继承字段配置 Tab 的字段组 |
| 显示名 | 继承字段配置 Tab 的显示名 |
| 页面承载 | 继承字段配置 Tab 的页面承载 |
| 分值 | 数字输入框，可编辑；按当前身份分支展示和保存 |

资料完整度保存规则：

1. Tab 顶部使用分支切换：在校生、职场人；表格中的 `分值` 只编辑当前分支。
2. 右上或表格上方展示当前分支总分，例如 `在校生 100/100`、`职场人 96/100`。
3. 右下角固定展示 `保存完整度配置`；无新增按钮，无编辑抽屉。
4. 保存时必须同时校验在校生、职场人两个分支总分均为 100；任一分支不等于 100 时阻断保存并定位到对应分支。
5. 字段配置 Tab 关闭某字段的计分控制后，该字段从资料完整度列表移除；再次打开时按默认分值或 0 分进入列表，由管理员重新调整。

### 2.3.4 其他 Tab 输入框编辑口径

| Tab | 输入控件要求 |
|-----|--------------|
| 上传限制 | 图片数量、文件大小使用数字输入框/步进器；文件格式使用多选；失败提示使用文本域 |
| 审核 SLA | 学历审核时限、临期提醒阈值使用数字输入框；超时标记使用开关；头像/实名无 SLA 说明用只读提示 |
| 文案配置 | 文案内容使用文本域；文案启用状态使用开关；开放文本长度使用最小/最大数字输入框；驳回模板可表格内新增一行模板内容 |
| 安全策略 | 倒计时、有效期、发送上限、错误上限使用数字输入框；微信授权失败兜底和语音安全策略使用开关/下拉；失败提示使用文本域 |

## 4. 规则

1. 年龄范围、用户字段展示/必填/计分配置、核心准入门槛、资料完整度权重、短信验证码规则、语音内容安全策略属于高风险配置，保存需二次确认。
2. 核心准入默认必须实名 + 头像 + 学历三项通过，不允许默认降级为单实名；页面可展示该门槛但不能保存低于三重认证的配置。
3. 字段配置 Tab 必须全量展示 `M01-DATA-user-input-fields`；不提供新增字段、删除字段、编辑字段按钮。
4. 字段配置行内开关遵循本页 2.3.2：不展示时不能必填，不能计分；固定必填、固定不计分、流程条件采集等字段开关置灰。
5. 资料完整度 Tab 的数据源只来自字段配置 Tab 中 `是否计分控制=开` 的字段；无新增按钮，分值只能在当前分支列表内编辑。
6. 资料完整度保存时必须校验在校生、职场人两个分支总分均为 100；任一分支不满足则阻断保存。
7. 上传限制、审核 SLA、文案配置、安全策略均需提供输入框、数字步进器、文本域、开关或下拉等可编辑控件，不允许只展示静态表格。
8. 配置变更只影响变更后的判定，历史资料按原始冗余值展示；资料完整度、核心准入状态可按最新配置重算，但原始用户填写内容不被改写。
9. 修改文案不得改变产品已确认口径，例如“身份证号”“千寻币”“三重认证”。
10. 语音介绍只配置微信音频机审策略和提示文案，首版不提供单独人工语音审核开关。
11. 保存按钮必须位于当前 Tab 内容区右下角，不允许放在页面右上角；保存必须填写变更原因；保存成功后记录配置版本、操作人、变更前后值、风险等级和审计单号。
