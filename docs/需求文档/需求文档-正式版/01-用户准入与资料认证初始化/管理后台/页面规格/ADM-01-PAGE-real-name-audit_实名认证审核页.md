## 1. 页面定位

- **目标用户**：审核员、风控、超级管理员
- **核心任务**：审核用户提交的姓名 + 身份证号 + 已绑定手机号三要素实名认证信息，给出通过或驳回结论
- **页面类型**：审核列表 + 审核详情
- **字段事实源**：实名认证审核字段来源以 `M01-DATA-user-input-fields` 的实名认证字段组与登录账号 `phone` 为准；后台只定义脱敏、权限和审核操作。

## 5. 审核规则

1. 不展示/不要求人脸核身结果。
2. 字段文案统一为“身份证号”。
3. 姓名 + 身份证号 + 已绑定手机号作为实名三要素核验口径，身份证号 + 手机号作为唯一性辅助判定。
4. 单身承诺函仅校验用户是否勾选，不作为第四认证。
5. 驳回原因自由输入，并原文同步至用户站内信。
6. 审核通过/驳回后实时刷新当前列表和统计；待审核筛选下已处理记录移出当前列表。
