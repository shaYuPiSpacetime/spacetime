## 1. 页面定位

- **目标用户**：运营、客服、审核员、风控、超级管理员
- **核心任务**：以卡片宫格查看 App 用户资料、认证状态、准入状态，并通过右侧画像抽屉查看详情
- **页面类型**：管理列表页 + 右侧画像抽屉（独立画板）
- **字段事实源**：用户字段来源、显示名、敏感级别与字段启用状态以 `M01-DATA-user-input-fields` 为准；本页字段表只定义后台组合展示与脱敏方式。

## 2.2 卡片宫格展示结构

| 区域 | 展示内容 | UI 要求 |
|------|----------|---------|
| 头像与身份 | 头像、真实姓名、性别/年龄、居住地、身份标签、VIP 标签 | 卡片顶部左侧为头像，右侧为姓名与基础摘要；VIP 使用低饱和标签，不抢主操作视觉 |
| 资料摘要 | 职场人展示职业、公司、年收入；在校生展示学历、学校、专业 | 最多两行，超长省略；字段缺失显示 `-` |
| 认证进度 | 头像认证、实名认证、学历认证三个状态点亮展示 | 通过为绿色，待审为橙色，未提交/未通过为灰色或红色；三个认证必须同时可见 |
| 资产与联系 | 资料完成度、千寻币余额、微信号脱敏值 | 资产字段未上线时隐藏；微信号按权限脱敏 |
| 操作区 | `画像`、`头像审核` | 操作区固定在卡片底部，按钮不挤压字段内容 |

## 2.3 画像抽屉展示结构

> 画像抽屉按「顶部概览 + 资料信息 + 状态资产 + 处理记录 + 固定底部操作」组织。基本信息由轻量资料和基础资料组成；扩展资料独立成组展示，不并入基本信息。抽屉主体可纵向滚动，底部高风险操作固定在抽屉底部。

| 区域 | 展示内容 | UI 要求 |
|------|----------|---------|
| 抽屉标题 | `画像详情`、关闭入口 | 右侧抽屉宽度建议 640px；Figma 单独画板中可放宽至 680-720px 便于展示资料分组。主列表背景使用半透明遮罩缩略 |
| 顶部概览 | 头像、真实姓名/昵称、用户编号、手机号脱敏、注册时间、用户状态、VIP 摘要 | 首屏顶部横向排版：头像与姓名在左，用户状态/核心标签居中，VIP 摘要小卡在右；高敏字段默认脱敏 |
| 基本信息-轻量资料 | 性别、年龄/生日、身份、最高学历、现居省市区、定位状态 | 使用 2 列字段栅格；字段来源为 `M01-DATA-user-input-fields` 轻量资料字段组 |
| 基本信息-基础资料 | 昵称、身高、体重、家乡、户口、行业、职业、公司、年收入、学校、专业、婚姻状况 | 使用 2 列字段栅格；职场人优先展示职业/公司/年收入，在校生优先展示学校/专业；字段缺失显示 `-` |
| 扩展资料 | 脱单目标、感情状态、关于我/希望 TA 了解、个人标签、相册/背景图、语音介绍、MBTI、爱听的歌曲、见面偏好、资料问答、住房/购车/子女/结婚计划/宗教/吸烟/饮酒/宠物态度 | 独立分组展示，建议用「摘要卡 + 标签/缩略图/文本预览」组合；开放文本最多展示 2 行并提供查看入口；相册展示 3-4 张缩略图；长字段不撑开抽屉 |
| 认证与准入 | 核心准入状态、头像认证、实名认证、学历认证、资料完整度 | 使用状态标签 + 资料完整度进度条；三重认证状态必须同时可见 |
| 千寻币/VIP | 千寻币余额、最近流水表：时间、类型、金额、余额、来源/用途；VIP 状态、到期时间 | 展示最近 3-5 条流水，更多流水跳转 PRD-04 对应资产页面；VIP 摘要与顶部概览保持一致 |
| 客服/风控处理记录 | 最近处理记录：处理人、时间、类型、处理说明 | 只展示客服/风控处理，不恢复蓝湖红娘、牵线或跟进字段 |
| 底部操作 | 冻结/解冻、查看高敏、特殊修改性别、定位头像审核 | 固定在抽屉底部；危险操作使用二次确认；无权限时隐藏或置灰 |

## 6. 导出字段

导出字段固定为：用户姓名、用户昵称、用户身份证号、性别、出生日期、身份、婚姻状况、个人标签、资料完整度、首登引导状态、核心准入状态、头像认证状态、实名认证状态、学历认证状态。导出文件不做掩码，必须二次确认并记录审计；其中用户填写字段的字段 ID、显示名、敏感级别以 `M01-DATA-user-input-fields` 为准，状态类字段按系统状态枚举导出。

| 状态类型         | 场景                   | 页面表现                   |
