## 1. 模块目标

管理后台 PRD-01 承接移动端用户准入、资料与认证审核的运营能力，核心目标是让审核员、客服、运营在一个可控、可审计的后台中完成 App 用户查看、认证审核、资料内容审核、准入配置和字典维护。

**后台目标：** App 用户管理采用卡片宫格 + 右侧画像抽屉；实名、头像、学历认证独立审核；资料图片和开放性文字独立审核；敏感字段按权限脱敏，原图下载与完整字段查看必须审计。

**核心指标：** 待审核量、审核通过率、驳回率、学历 24 小时内处理率、核心准入开放率、敏感字段查看次数。

## 3. 关键业务流程

### 3.1 App 用户管理流程

```
进入 App 用户管理
-> 按搜索/筛选查看卡片宫格
-> 点击用户卡片打开右侧画像抽屉
-> 查看基础资料、认证状态、准入状态、千寻币/VIP摘要
-> 需要处理头像认证时点击“头像审核”
-> 跳转到头像认证审核页对应记录
```

### 3.2 认证审核流程

```
用户提交实名/头像/学历材料
-> 后台认证审核页生成待审核记录
-> 审核员查看材料与历史
-> 通过：状态变更为已通过，站内信通知用户，重新计算核心准入
-> 驳回：自由输入驳回原因，站内信原文告知用户并引导联系客服
-> 当前列表实时刷新，已处理记录移出待审核筛选结果
```

### 3.3 配置与字典流程

```
管理员进入准入与认证配置/准入字典管理
-> 修改年龄、资料完整度、准入门槛、上传限制、审核 SLA、文案等
-> 高风险配置二次确认
-> 记录变更前后值与操作人
-> 新增数据按新配置生效，历史数据按冗余值展示
```

## 6. 数据与状态要求

后台用户资料、认证材料、字段显示名、字段来源与字段配置统一以 `../../PRD-01_模块公共定义.md` 的 `M01-DATA-user-input-fields` 为准。后台页面可以按运营场景组合摘要、筛选、脱敏、导出和审核字段，但不另行定义一套用户字段全集。

| 状态对象 | 后台可见字段 | 后台可操作 |
|----------|--------------|------------|
| 首登状态 | 未开始、轻量引导中、强引导中、游客模式、已完成 | 查看、筛选 |
| 核心准入 | 资料未完成、游客、认证未完成、已开放 | 查看、筛选 |
| 实名认证 | 未提交、审核中、已通过、已驳回 | 通过、驳回、查看历史 |
| 头像认证 | 未提交、审核中、已通过、已驳回、已失效 | 通过、驳回、查看大图和历史 |
| 学历认证 | 未提交、审核中、已通过、已驳回、已失效 | 通过、驳回、查看材料 |
| 资料图片 | 未提交、审核中、已通过、已驳回 | 通过、驳回 |
| 开放文字 | 未提交、审核中、已通过、已驳回 | 通过、驳回 |

状态隔离原则：

1. 实名、头像、学历、资料图片、开放性文字五类审核状态独立。
2. 主头像通过不代表资料图片通过。
3. 资料图片通过不代表开放性文字通过。
4. 任一认证通过/驳回都要记录审核历史与操作人。

## 7. 权限、审计与脱敏

| 操作 | 权限要求 | 审计要求 | 二次确认 |
|------|----------|----------|----------|
| 查看完整手机号/姓名/身份证号 | 高敏字段权限 | 记录操作人、字段、用户、理由、时间 | 是 |
| 下载实名/学历/头像原图 | 原图下载权限 | 记录材料类型、用户、理由、时间 | 是 |
| 导出用户/审核数据 | 导出权限 | 记录导出字段、筛选条件、下载时间、下载人；导出文件字段不做掩码 | 是 |
| 修改性别 | 客服特殊权限 | 记录变更前后值、原因、操作人 | 是 |
| 冻结/解冻、启用/停用 | 客服/风控权限 | 记录目标用户、动作、原因 | 是 |
| 修改配置/字典 | 管理员权限 | 记录变更前后值 | 是 |
| 审核通过/驳回 | 审核权限 | 记录对象、结果、驳回原因 | 是 |

导出字段固定为：用户姓名、用户昵称、用户身份证号、性别、出生日期、身份、婚姻状况、个人标签、资料完整度、首登引导状态、核心准入状态、头像认证状态、实名认证状态、学历认证状态。其中用户填写字段的字段 ID、显示名与敏感级别以 `M01-DATA-user-input-fields` 为准；状态类字段由系统状态枚举维护。

## 10. 上线 / 迁移 / 回滚

| 项 | 说明 |
|----|------|
| 存量用户 | 补齐首登状态、认证状态、资料完整度；缺失字段按空态展示 |
| 存量头像 | 如需要纳入认证，生成待审核记录；否则保留原状态并在用户换头像后提审 |
| 未成年存量 | 产品确认线下人工处理，后台可通过筛选辅助定位 |
| 配置上线 | 高风险配置建议灰度或管理员二次确认后生效 |
| 回滚策略 | 隐藏新增审核入口或停止新提审；已保存材料和审核历史不删除 |
