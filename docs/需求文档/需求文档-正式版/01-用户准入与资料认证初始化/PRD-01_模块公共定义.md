## 2. 模块术语

| 术语 ID                         | 统一术语             | 定义                                                             |
| ------------------------------- | -------------------- | ---------------------------------------------------------------- |
| `M01-TERM-light-guide`          | 首登轻量引导         | 新用户首次登录后依次填写性别、生日、身份、学历、现居地的轻量步骤 |
| `M01-TERM-strong-guide`         | 完善资料和认证强引导 | 轻量引导后进入的四步资料与认证流程                               |
| `M01-TERM-visitor-mode`         | 游客模式             | 用户跳过强引导后进入的受限状态，不进入真实曝光/推荐/互动链路     |
| `M01-TERM-triple-certification` | 三重认证             | 实名认证、头像认证、学历认证三项认证集合                         |
| `M01-TERM-core-access`          | 核心准入             | 用户进入真实曝光、喜欢、私信、社区等核心链路的资格               |
| `M01-TERM-profile-score`        | 资料完整度           | 用 0-100 分衡量资料填写完整程度，并驱动补全引导                  |

| `sms_code` | 手机号验证码 | 通过手机号 + 短信验证码登录/注册 | 启用 |

### 3.2 `M01-ENUM-first-guide-status` 首登引导状态

| 值               | 显示名     | 说明                                 | 状态 |
| ---------------- | ---------- | ------------------------------------ | ---- |
| `not_started`    | 未开始     | 新用户尚未进入首登流程               | 启用 |
| `light_guiding`  | 轻量引导中 | 性别/生日/身份/学历/现居地步骤未完结 | 启用 |
| `strong_guiding` | 强引导中   | 已进入完善资料和认证四步向导         | 启用 |
| `visitor`        | 游客模式   | 用户跳过强引导                       | 启用 |
| `completed`      | 已完成     | 达到首登资料初始化完成标准           | 启用 |

### 3.3 `M01-ENUM-identity-type` 身份类型

| 值        | 显示名 | 说明                   | 状态 |
| --------- | ------ | ---------------------- | ---- |
| `student` | 在校生 | 当前在校学生           | 启用 |
| `worker`  | 职场人 | 已进入职场或非在校学生 | 启用 |

| 值              | 显示名 | 说明                                 | 状态 |

| `rejected`      | 已驳回 | 审核驳回，可重新提交或联系客服       | 启用 |
| `expired`       | 已失效 | 材料过期或因资料变更失效             | 启用 |

| 值                      | 显示名     | 说明                                   | 状态 |

## 5. 模块状态机

### 5.1 `M01-SM-first-guide` 首登引导状态机

| 起始状态         | 触发                   | 目标状态         | 副作用                                                               |
| ---------------- | ---------------------- | ---------------- | -------------------------------------------------------------------- |
| `not_started`    | 首次登录成功           | `light_guiding`  | 创建账号与首登进度                                                   |
| `light_guiding`  | 完成或跳过轻量步骤     | `strong_guiding` | 保存已填写字段，强引导反显                                           |
| `strong_guiding` | 点击“暂不认证/先看看”  | `visitor`        | 写入游客模式                                                         |
| `strong_guiding` | 完成强引导页面必填资料 | `completed`      | 触发资料完整度计算；不等于核心准入开放，核心准入仍以三重认证通过为准 |
| `visitor`        | 继续完善资料           | `strong_guiding` | 恢复强引导                                                           |

### 5.2 `M01-SM-triple-certification` 三重认证状态机

三项认证分别维护状态，不共用一条记录：

| 对象     | 状态流转                                                                                           |
| -------- | -------------------------------------------------------------------------------------------------- |
| 实名认证 | `not_submitted -> reviewing -> approved/rejected`；已通过后用户端不支持修改实名信息                |
| 头像认证 | `not_submitted -> reviewing -> approved/rejected`；更换主头像后已通过状态变为 `expired` 并重新提审 |
| 学历认证 | `not_submitted -> reviewing -> approved/rejected/expired`；已通过后重新认证需重新审核              |

### 5.3 `M01-SM-core-access` 核心准入状态机

| 起始状态                | 触发                       | 目标状态                                    | 说明                |
| ----------------------- | -------------------------- | ------------------------------------------- | ------------------- |
| `blocked_profile`       | 用户跳过强引导             | `visitor`                                   | 不进入真实推荐/曝光 |
| `blocked_profile`       | 首登资料完成               | `blocked_certification`                     | 等待三重认证        |
| `blocked_certification` | 实名、头像、学历三项均通过 | `allowed`                                   | 开放核心链路        |
| `allowed`               | 任一认证失效或账号冻结     | `blocked_certification` / `blocked_profile` | 按失效原因回退      |

## 6. 模块业务规则

| 规则 ID                                    | 规则描述           | 判定逻辑                                                                                                  |
| ------------------------------------------ | ------------------ | --------------------------------------------------------------------------------------------------------- |
| `M01-RULE-login-phone`                     | 手机号强制绑定     | 微信授权或短信登录均需绑定手机号；实名唯一性以身份证号 + 手机号作为判定依据                               |
| `M01-RULE-light-guide`                     | 首登轻量引导       | 性别、生日必填；身份、学历、现居地可跳过；跳过仍保存已填字段                                              |
| `M01-RULE-gender-lock`                     | 性别锁定           | 用户端提交后不可修改；客服特殊修改需权限与审计，并重算推荐/曝光/匹配                                      |
| `M01-RULE-age-range`                       | 年龄准入           | 默认 18-60，配置项可调整；不符合时阻断继续使用并提示                                                      |
| `M01-RULE-profile-scope`                   | 本期资料字段范围   | `M01-DATA-user-input-fields` 为正式字段池；资料类字段是否展示、必填和计分由后台配置控制                   |
| `M01-RULE-profile-score`                   | 资料完整度         | 资料完整度总分、参与字段、在校生/职场人权重均由后台配置；资料完整度只用于补全引导和展示，不等同于核心准入 |
| `M01-RULE-real-name`                       | 实名认证           | 用户填写姓名 + 身份证号，系统取已绑定手机号；调用实名三要素核验服务做一致性校验，后台人工审核兜底；身份证号字段文案必须展示为“身份证号” |
| `M01-RULE-single-commitment`               | 单身承诺函         | 作为协议查看与勾选，不作为第四项认证，不影响准入                                                          |
| `M01-RULE-education-realname-precondition` | 学历认证实名前置   | 必须先提交实名认证，再进入学历认证流程                                                                    |
| `M01-RULE-education-method`                | 学历认证分流       | 在校生走学生证材料；毕业生走学信网验证码/证书编号/上传证书                                                |
| `M01-RULE-education-identity-change`       | 身份切换与学历认证 | 身份切换不影响已通过学历认证；处于审核中的学历认证需重新提交；用户主动重新认证需重新审核                  |
| `M01-RULE-core-access`                     | 核心准入门槛       | 核心曝光、喜欢、私信、社区发布要求实名 + 头像 + 学历三项通过且账号正常                                    |
| `M01-RULE-match-gender`                    | 异性匹配           | 首版仅支持男女异性匹配，不采集性取向字段                                                                  |
| `M01-RULE-content-audit-isolation`         | 审核隔离           | 实名、学历、头像、资料图片、开放性文字五类状态独立存储、独立驳回                                          |
| `M01-RULE-voice-safety`                    | 语音介绍安全       | 语音介绍上传后必须走微信音频内容安全机审；首版不单设后台人工语音审核页，机审不通过则前台不展示            |
| `M01-RULE-reject-reason`                   | 驳回原因           | 后台自由输入，用户端原文展示，同时告知可联系客服                                                          |

### 6.1 `M01-DATA-user-input-fields` 用户字段总表

本节作为 APP 用户需填写、选择或授权提交的字段池。

除登录账号、协议确认、实名认证、头像认证、学历认证等流程/合规字段外，资料类字段的是否展示、是否必填、是否参与资料完整度计分、在校生/职场人权重均由后台字段配置控制。字段未启用时，移动端不展示、不校验、不计分。

#### 6.1.1 正式字段池

| 字段组   | 字段 ID                      | 显示名                | 类型          | 页面承载                               | 必填控制                   | 计分控制   | 备注                                               |
| -------- | ---------------------------- | --------------------- | ------------- | -------------------------------------- | -------------------------- | ---------- | -------------------------------------------------- |
| 登录账号 | `loginMethod`                | 登录方式              | enum          | 登录授权页                             | 固定必填                   | 固定不计分 | 微信授权/手机号验证码                              |
| 登录账号 | `phone`                      | 手机号                | string        | 登录授权页/绑定手机号                  | 固定必填                   | 固定不计分 | 微信授权与短信登录均需绑定                         |
