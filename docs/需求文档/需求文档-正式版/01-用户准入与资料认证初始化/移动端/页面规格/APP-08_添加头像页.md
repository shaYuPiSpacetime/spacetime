## 1. 页面定位

- **目标用户**：强引导用户、需要头像认证的用户
- **核心任务**：引导上传本人清晰头像，完成裁剪并提交头像认证
- **页面类型**：强引导四步第 2 步
- **字段事实源**：头像认证字段 ID、显示名、页面承载与固定采集规则以 `M01-DATA-user-input-fields` 的头像认证字段组为准。

---

## 2. 布局

| 区块 | 内容 | 规则 |
|------|------|------|
| 四步进度 | 基本资料、添加头像、自我介绍、三重认证 | 添加头像高亮 |
| 说明卡片 | `选一张你满意的头像`，示例：本人照片、能看清长相、展示完美的你 | 蓝湖已覆盖 |
| 反例说明 | 非人物照、风景照、模糊遮挡、无正脸 | 作为上传前引导，不代表自动禁止 AI/过度美颜 |
| 选择动作 | `知道了，去选照片` | 打开拍照/相册选择 |
| 裁剪页 | 虚线裁剪框、提示 `请将人物主体放虚线框内`、取消/确认 | 必须裁剪后提交 |
| 审核态 | 头像预览 + `审核中` 标签 + 下一步 | 提交后进入审核中 |

---

## 5. 规则

1. 支持相册和拍照。
2. 必须裁剪，且裁剪框内需要看到头像主体。
3. 不在用户端禁止 AI、盗图、换脸、过度美颜；后台需通过人像识别接口辅助判定是人像而非动物/风景。
4. 主头像提交后自动进入头像认证审核中。
5. 更换主头像后，已通过的头像认证需失效并重新提审。

---

## 6. 状态与异常

| 状态类型 | 场景 | 页面表现 |
|----------|------|----------|
| 未上传 | 首次进入 | 展示头像规范和去选照片按钮 |
| 选择来源 | 点击去选照片 | action sheet：拍照、从相册选取、取消 |
| 裁剪中 | 已选图片 | 裁剪页展示虚线框 |
| 审核中 | 上传成功 | 头像预览 + 审核中标签 |
| 上传失败 | 网络/文件失败 | toast「上传失败，请重试」 |
| 权限拒绝 | 相机/相册未授权 | 引导开启权限 |
