## 1. 页面定位

- **目标用户**：实名认证审核中或已通过、待学历认证用户
- **核心任务**：按在校生/中国大陆毕业生分流，提交学历认证信息或材料
- **页面类型**：认证分流页 + 认证表单页

## 5. 规则

1. 必须先提交实名认证，再进入学历认证流程；不允许先提交学历材料。
2. 在校生上传学生证/在读证明材料，最多 4 张，明确展示 0/4。
3. 毕业生支持三种方式：学信网在线验证码、毕业证或学位证书编号、上传毕业证或学位证书。
4. 学信网方式标记为推荐；证书编号和上传证书标记较慢，24 小时内完成审核。
5. 海外/港澳台学历首版展示暂不支持。
6. 学校字典先查本地冗余，未命中调用全国大学院校查询 API 后入库。
7. 身份从在校生切换为职场人或反向切换时，已通过学历认证不自动失效；审核中的学历认证需重新提交。
8. 用户主动重新认证学历时，新材料提交后进入审核中，原已通过状态在新审核通过前不自动覆盖。

## 6. 状态与异常

| 状态类型 | 场景 | 页面表现 |
|----------|------|----------|
| 实名未提审 | 从三重认证页进入学历 | 弹顺序提示并引导去实名 |
| 默认态 | 进入学历页 | 按身份反显推荐分段 |
| 材料上传 | 点击上传区域 | action sheet：拍照、从相册选取、取消 |
| 达到上限 | 已上传 4 张 | 上传入口置灰或隐藏 |
| 审核中 | 提交成功 | 三重认证页学历状态审核中 |
| 重新认证中 | 已通过后重新提交材料 | 保留原已通过结果，新材料显示审核中 |
| 已驳回 | 后台驳回 | 展示驳回原因，引导联系客服或重新提交 |
| 暂不支持 | 海外/港澳台 | 展示当前学历地区暂不支持认证 |
| 第三方不可用 | 学历服务不可用 | 提示稍后重试，保留已填内容 |
