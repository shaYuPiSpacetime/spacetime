## 1. 页面定位

- **目标用户**：待实名认证用户
- **核心任务**：填写姓名与身份证号，结合已绑定手机号，查看并勾选单身承诺/认证协议后提交实名认证
- **页面类型**：认证表单页
- **字段事实源**：实名认证字段 ID、显示名、页面承载与固定必填规则以 `M01-DATA-user-input-fields` 的实名认证字段组为准；手机号取登录账号字段组的已绑定 `phone`。

## 5. 规则

1. 首版不做人脸核身。
2. 实名三要素以姓名 + 身份证号 + 已绑定手机号做一致性核验。
3. 第三方核验不可用或需复核时，保留提交记录并进入后台人工审核兜底。
4. 单身承诺函仅为协议查看和勾选，不作为第四认证，不影响核心准入项数。
5. 驳回后站内信展示后台自由输入的原文原因，并告知联系客服申诉。
6. 实名认证已通过后，用户端不支持修改真实姓名和身份证号；如需处理异常，只能走客服/风控线下流程。

---

## 6. 状态与异常

| 状态类型 | 场景 | 页面表现 |
|----------|------|----------|
| 默认态 | 未填写 | 输入框空，提交置灰 |
| 可提交 | 姓名、身份证号、勾选完成 | 提交按钮高亮 |
| 格式错误 | 身份证号格式不合法 | 字段下提示或 toast |
| 审核中 | 提交成功 | 三重认证页展示实名审核中 |
| 已驳回 | 后台驳回 | 展示驳回原因，引导联系客服或重新提交 |
| 第三方不可用 | 如后续接核验服务不可用 | 提示认证服务暂不可用，保留已填内容 |
