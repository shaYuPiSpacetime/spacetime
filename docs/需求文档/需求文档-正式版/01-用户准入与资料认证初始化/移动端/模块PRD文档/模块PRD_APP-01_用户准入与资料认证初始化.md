## 1. 模块目标

PRD-01 是小程序的准入层、资料初始化层与信任建立层，目标是在用户首次进入时完成账号登录、最小资料采集、强引导资料补齐，并通过实名、头像、学历三重认证决定用户是否能进入真实曝光、喜欢、私信、社区等核心链路。

**用户故事：** 作为新用户，我希望用微信或手机号快速登录，并通过分步填写和认证逐步建立可信资料，以便进入严肃、可靠的婚恋交友场景。

**核心指标：** 登录成功率、轻量引导完成率、强引导完成率、三重认证提交率、三重认证通过率、核心准入开放率。

## 3. 关键业务流程

### 3.1 新用户标准路径

```
进入小程序
-> 登录授权页点击“立即使用”
-> 协议弹窗同意
-> 微信授权登录或手机号验证码登录
-> 绑定/确认手机号
-> 首登轻量引导：性别 -> 出生日期 -> 身份 -> 学历 -> 现居地
-> 强引导：基本资料 -> 添加头像 -> 自我介绍 -> 三重认证
-> 三重认证：头像认证、实名认证、学历认证
-> 三项认证均通过
-> 开放核心曝光、喜欢、私信、社区
```

分支：

| 分支 | 处理 |
|------|------|
| 轻量引导身份/学历/现居地未填写 | 字段配置为非必填时允许空值下一步；保存已填写字段，后续强引导反显；未填字段继续可补 |
| 资料填写未完成时离开流程 | 不生成旧的受限浏览状态；仅可访问资料未完成可访问白名单页面，白名单外页面和动作统一跳转“完善资料和认证”拦截页 |
| 拦截页左图态 | 除轻量资料之外的基本资料字段一个都未填写；轻量资料只在用户首次登录出现，第二次登录、换绑或改绑手机号等场景不再回跳轻量引导 | 展示蓝湖左图 `完善资料和认证 解锁更多专属权益`，点击“立即完善”进入资料补全 |
| 拦截页右图态 | 除轻量资料之外的基本资料字段已填写一部分，但资料和认证仍未完成 | 展示蓝湖右图三项卡片：基本资料、头像简介、三重认证，点击“立即完善”进入当前缺口补全 |
| 年龄不符合 | 阻断进入后续流程，提示平台年龄范围 |
| 定位失败/拒绝定位 | 允许手动选择城市，保存城市；定位成功时保存经纬度 |
| 学历认证未实名提交 | 弹窗提示先提交实名，仅可去实名或关闭提示；页面流程必须先提交实名再进入学历认证 |

### 3.2 老用户回流路径

```
已有登录态进入小程序
-> 查询首登状态、资料完整度、认证状态
-> 若首次登录且轻量引导未完成：回到轻量引导未完成步骤
-> 若非首次登录且资料未完成：白名单外统一展示“完善资料和认证”拦截页
-> 若强引导未完成：按基本资料填写状态展示左图态或右图态，再由“立即完善”进入补全流程
-> 若三重认证未通过：核心能力触发准入拦截
-> 若三重认证通过：正常进入核心链路
```

## 8. 非功能性需求

### 8.1 权限与隐私

| 字段/材料 | 敏感级别 | 移动端展示 | 留存 |
|-----------|----------|------------|------|
| 手机号 | 敏感 | 仅用户本人可见完整值 | 随账号留存 |
| 姓名、身份证号 | 高敏 | 认证页填写；结果页不反复明文展示 | 永久保存，注销不删除 |
| 学历材料 | 高敏 | 用户本人可查看提交状态 | 永久保存，注销不删除 |
| 头像原图 | 敏感 | 用于头像认证和展示 | 永久保存，注销不删除 |

### 8.2 可用性

| 场景 | 要求 |
|------|------|
| 分步保存 | 每一步提交后立即保存，中断后恢复到未完成步骤 |
| 弱网上传 | 上传失败保留当前页和用户已填内容，允许重试 |
| 协议更新 | 用户再次进入登录/授权链路时重弹协议弹窗 |
| 第三方不可用 | 展示可理解提示，不清空用户已填内容 |

### 8.3 埋点

| 埋点事件 | 触发时机 | 关键参数 |
|----------|----------|----------|
| `m01_login_show` | 登录页曝光 | loginMethodAvailable |

| `m01_light_step_submit` | 轻量引导单步提交 | step, hasValue, required |
| `m01_strong_step_submit` | 强引导单步提交 | step |
| `m01_avatar_submit` | 头像提交认证 | source |
| `m01_realname_submit` | 实名提交 | hasCommitmentChecked |
| `m01_education_submit` | 学历提交 | identityType, method |
| `m01_core_access_block` | 核心入口被拦截 | entry, missingCertifications |

## 10. 上线 / 迁移 / 回滚

| 项 | 说明 |
|----|------|
| 存量用户 | 按资料与认证状态补齐 `firstGuideStatus`、资料完整度、三重认证状态；缺失字段不自动造值 |
| 未成年存量 | 产品确认线下人工处理，不在本期产品功能内自动处置 |
| 性别存量异常 | 仅客服特殊权限处理并记录审计 |
| 灰度策略 | 可按新注册用户比例灰度强引导和三重认证准入拦截 |
| 回滚策略 | 关闭强引导入口与核心拦截配置；已保存资料和认证材料不删除 |
