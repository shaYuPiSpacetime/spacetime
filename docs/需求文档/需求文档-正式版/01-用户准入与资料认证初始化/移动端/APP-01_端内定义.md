## 2. 登录态与核心准入

| 用户状态 | 可见页面/能力 | 不可用能力 | 不满足时处理 |
|----------|---------------|------------|--------------|
| 未登录 | 登录授权页、协议页 | 资料、认证、推荐、互动、资产 | 引导登录 |
| 已登录但轻量引导未完成 | 轻量引导当前步骤 | 真实推荐/曝光/互动 | 回到未完成步骤 |
| 强引导中 | 基本资料、头像、自我介绍、三重认证 | 真实推荐/曝光/互动 | 可继续填写或跳过进游客模式 |
| 游客模式 | `M01-CFG` 游客白名单页面、资料补全、认证入口 | 白名单外页面、真实推荐流、曝光、喜欢、私信、社区发布 | 按缺口跳转资料补全/三重认证/实名/学历认证 |
| 资料完成但三重认证未通过 | 资料编辑、认证中心、受限浏览 | 核心曝光、喜欢、私信、社区发布 | 提示完成三重认证 |
| 三重认证通过且账号正常 | 核心曝光、喜欢、私信、社区发布 | 无本模块限制 | — |

游客模式白名单以 `PRD-01_模块公共定义.md` 的 `7.4 游客模式白名单` 为准。未列入白名单的页面和动作一概不可进入，不做模糊可进的默认扩展。

## 3. 移动端 UI 状态

| 场景 | 页面表现 | 默认文案 |
|------|----------|----------|
| 协议未同意 | 协议弹窗保持展示，不进入下一步 | 同意后才可以继续使用 |
| 年龄不符合 | 阻断继续，展示说明 | 当前年龄不符合平台使用要求 |
| 定位失败 | toast + 手动选择入口 | 定位失败，请手动选择城市 |
| 上传失败 | toast + 保留当前页面 | 上传失败，请重试 |
| 头像审核状态 | 添加头像页与三重认证页展示审核中/通过/驳回，不单设结果页 | 头像认证审核中 |
| 实名失败 | 状态页/站内信展示驳回原因 | 实名认证未通过，如需申诉请联系客服 |
| 学历失败 | 状态页/站内信展示驳回原因 | 学历认证未通过，如需申诉请联系客服 |
| 第三方不可用 | 置灰或提示稍后重试 | 认证服务暂不可用，请稍后重试 |
| 核心准入拦截 | 弹窗/横幅提示并给认证入口 | 完成三重认证后才可使用该功能 |
| 海外/港澳台学历 | 当前分流下展示暂不支持 | 当前学历地区暂不支持认证 |

## 4. 页面规格蓝湖核对要求

移动端页面规格落文档前必须完成以下核对：

1. 登录授权、性别、生日、身份、学历、现居地页面以蓝湖为准。
2. 强引导四步“基本资料 -> 添加头像 -> 自我介绍 -> 三重认证”以蓝湖为准。
3. 实名认证页字段文案必须按确认结论修正为“身份证号”，即使设计稿仍显示“证件号码”。
4. 学历认证页需核对在校生、毕业生、中国大陆、学信网验证码、证书编号、上传证书各状态。
5. 异常状态页（年龄不符、定位失败、上传失败、实名失败、学历失败、第三方不可用）需补设计稿后再最终定稿。
