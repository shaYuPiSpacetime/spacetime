# PRD-01 甲方前置准备清单（简版）

> 面向甲方沟通使用。只列上线前必须由甲方准备或确认的内容，其他技术细节由研发侧在技术方案阶段承接。

| 版本   | 日期       | 修改人 | 变更摘要             |
| ------ | ---------- | ------ | -------------------- |
| 版本01 | 2026-06-26 | Codex  | 简化为甲方必要准备项 |

---

## 1. 甲方必须准备的资料

| 序号 | 准备项           | 甲方需要给什么                                                           |
| ---- | ---------------- | ------------------------------------------------------------------------ |
| 1    | 用户协议         | 正式文本、协议名称、生效时间                                             |
| 2    | 隐私政策         | 正式文本、生效时间、客服联系方式                                         |
| 3    | 单身承诺函       | 正式文本，用于实名认证页勾选确认                                         |
| 4    | 实名认证授权说明 | 允许平台使用“姓名 + 身份证号 + 手机号”做实名认证的授权文案               |
| 5    | 学历认证授权说明 | 允许平台处理学校、学历、学信网验证码、证书编号、证书图片等材料的授权文案 |

---

## 2. 甲方必须确认的接口/账号

| 序号 | 准备项              | 甲方需要确认什么                                   |
| ---- | ------------------- | -------------------------------------------------- |
| 1    | 短信验证码接口      | 使用哪家短信服务商；短信签名和验证码模板是否已开通 |
|      | 实名认证三要素接口  | 是否已有服务商，支持“姓名 + 身份证号 + 手机号”核验 |
| 4    | 学历认证方式        | 是否已有服务商，接第三方学历认证接口；             |
| 5    | 学校库/学校查询接口 | 是否提供学校基础数据，或确认使用学校查询接口       |
| 6    | 内容安全能力        | 是否使用微信内容安全能力审核文字、图片、语音       |
| 7    | 客服入口            | 小程序内“联系客服”跳转到哪里，谁负责处理申诉       |
