## 3. 筛选与搜索

### 3.1 搜索

| 项       | 说明                                                 |
| -------- | ---------------------------------------------------- |
| 搜索字段 | 邀请关系编号、邀请人 UUID、邀请人手机号、被邀请人 UUID、被邀请人手机号 |
| 搜索方式 | 精确匹配                                             |
| 搜索触发 | 点击查询                                             |
| 清空搜索 | 有清空按钮，恢复默认列表                             |

### 3.2 筛选条件

| 筛选 ID                                             | 筛选名       | 类型     | 选项来源                          | 是否多选 | 默认值 | 是否可清除 |
| --------------------------------------------------- | ------------ | -------- | --------------------------------- | -------- | ------ | ---------- |
| `ADM-07-PAGE-invite-relation-list-FILTER-source`    | 来源类型     | 下拉     | 普通用户/校园代理（来源：`M07-ENUM-invite-source`） | 否       | 全部   | 是         |
| `ADM-07-PAGE-invite-relation-list-FILTER-status`    | 关系状态     | 下拉     | 已注册/已完成资料完善/已完成认证/风险冻结中/无效（来源：`M07-ENUM-invite-relation-status`） | 是       | 全部   | 是         |
| `ADM-07-PAGE-invite-relation-list-FILTER-bind-time` | 绑定时间范围 | 日期范围 | —                                 | —        | 无     | 是         |

### 3.3 筛选交互

- 筛选项变化后：点击「查询」触发
- 显示当前筛选标签：是（可逐个删除）
- URL 持久化：是

## 9. 验收标准

```
AC-ID: ADM-07-AC-relation-unfreeze
Given 一条状态=风险冻结中的邀请关系，当前为风控角色（GLB-ROLE-risk）
When  点击解除冻结并确认
Then  关联奖励 M07-SM-invite-reward → 已到账，写 invite_risk_release 流水，
      发 M07-NTF-invite-reward-arrive，记审计日志，列表状态刷新

AC-ID: ADM-07-AC-relation-mark-invalid
Given 一条非无效的关系，风控角色
When  点击标记无效并确认
Then  关系→无效，关联奖励→无效，发 M07-NTF-invite-reward-invalid，记审计日志

AC-ID: ADM-07-AC-relation-export
Given 具备导出权限的后台用户
When  设置筛选条件后点击导出
Then  按当前筛选异步生成明细，下载中心可下载；导出字段包含手机号且不脱敏，操作记审计
```

### 验收标准清单

| AC ID                             | 场景     | 类型 | 优先级 |
| --------------------------------- | -------- | ---- | ------ |
| `ADM-07-AC-relation-unfreeze`     | 解除冻结 | 正常 | P0     |
| `ADM-07-AC-relation-mark-invalid` | 标记无效 | 正常 | P0     |
| `ADM-07-AC-relation-export`       | 导出     | 正常 | P1     |
