## 1. 模块目标

在不突破现有报价文档模块边界的前提下，首版上线推广管理，细化裂变推广、校园代理、奖励统计与奖金结算的后台落地。拆为两条业务线：普通用户裂变奖励（发千寻币，拉新+早期活跃）、校园代理推广管理（统计数量 + 发奖金/佣金，渠道增长+结算管理）。

**用户故事：** 作为运营/渠道运营/财务/风控，我想配置裂变规则、查看邀请与代理数据、处理风控冻结、管理系统生成的奖金结算台账，以便支撑前期裂变增长并完成线下奖金结算。

**核心指标：** 成功邀请数（按后台配置统计节点）、代理带来的注册/认证/首次会员/首次充值转化、奖金结算准确率。

## 3. 关键业务流程

### 3.1 普通邀请奖励处理流程

```
入口：移动端触发奖励事件 → 后台落奖励流水
正常路径：
  1. 奖励事件命中且未触发风控 → 奖励已到账，写千寻币流水
  2. 命中风控（M07-RULE-invite-antifraud）→ 奖励冻结中，进冻结队列
分支（人工处理）：
  - 风控/超管在冻结队列：确认有效并发放 → 已到账（写 invite_risk_release 流水）
  - 确认无效并作废 → 无效
异常：单日超上限 → 超出转冻结或不发
出口：所有人工动作记审计日志
```

### 3.2 代理奖金结算流程

```
入口：代理推广带来用户事件 → 按代理奖励规则生成待结算奖金记录
正常路径：
  1. PRD-07 系统结算任务按结算周期汇总代理奖金记录，生成待结算结算单
  2. 标记已确认 → 线下发放 → 标记已发放
  3. 导出结算明细供线下打款
说明：首版不做自动打款；ADM-07 代理结算管理页只做台账查询、确认、发放标记与导出，不提供手动生成结算单按钮；结算任务无页面配置入口，按 `M07-RULE-agent-settlement-generate` 执行
```

### 3.3 代理付费归因规则

> A07-06 解释：代理统计是否包含后续付费归因，指“某用户通过代理二维码注册后，之后首次成为会员或首次充值千寻币时，这笔付费是否继续归因到原代理，并计入代理统计/奖金”。

1. 首版保留付费归因统计字段：首次会员人数、首次充值千寻币人数、对应奖金金额。
2. 归因关系跟随永久代理关系，不另设时间窗口；但奖励是否发放、金额多少由后台代理奖励规则控制。
3. 若 PRD-04 支付/会员能力未上线，付费归因字段展示为 0 或空态，不阻塞注册/认证统计。
4. 防刷口径沿用 `M07-RULE-invite-antifraud`，支付类奖励只响应 PRD-04 支付成功回调且按 userId+eventType 幂等。
5. 不做续费、复购、长期消费分成；后续如要做需另补 PRD。

## 9. 非功能性需求

### 9.1 权限

引用 ADM-07_端内定义.md 第 2 节。最终权限以后台菜单管理/页面操作权限控制为准；可看到对应页面的人可处理该页面授权事项。

### 9.2 安全与合规

- **敏感字段清单及处理：**

| 字段 | 加密存储 | 脱敏规则 | 留存时长 | 注销后处理 | 是否可导出 |
|------|---------|----------|----------|-----------|-----------|
| 被邀请人/被推广用户手机号 | 是 | 页面展示可按权限脱敏；导出不脱敏 | 随账号 | 匿名化 | 是，含明文手机号，需审计 |
| 代理联系电话 | 是 | 页面展示可按权限脱敏；导出不脱敏 | 合作期+留存 | 保留 | 是 |
| 收款相关字段 | 是 | 页面展示可按权限脱敏；导出不脱敏 | 合作期+留存 | 保留 | 首版不采集；若后续补充则导出不脱敏并审计 |

- **后台敏感信息查看控制：** 解除冻结、判无效、修改规则、变更结算状态均需二次确认 + 审计日志。
- **导出安全：** 推广导出字段包含手机号/收款信息且不脱敏，操作必须由页面权限控制并记录审计日志。

### 9.3 性能

- 列表查询 < 500ms，详情 < 200ms；列表默认 20 条/页，最大 100 条/页。
- 代理统计为聚合数据，可走预聚合/定时刷新避免实时大查询。

### 9.4 并发与幂等

| 需幂等的操作 | 并发场景 | 幂等方案建议 |
|-------------|----------|-------------|
| 奖励发放 | 同事件重复触发 | (relationId + eventType) 唯一键 |
| 解除冻结/判无效 | 多人并发处理同一记录 | 乐观锁/状态前置校验，已处理拒绝 |
| 结算单生成 | PRD-07 系统结算任务重复生成同周期 | (agentNo + period) 唯一键 |
| 标记已发放 | 重复点击 | 状态机前置校验 |

### 9.5 后台展示规范

引用 `../ADM-07_端内定义.md` 第 2.4、2.5 节。后台列表、详情、搜索条件、导出字段等展示层尽量不直接展示数据库内部主键 ID；接口、路由参数、内部定位可继续使用内部 ID；页面可见信息优先展示业务编号/用户 UUID/名称等业务标识。所有枚举、状态在页面与导出中只展示中文名称；所有 datetime 字段展示格式统一为 `yyyy-MM-dd HH:mm:ss`。

### 9.6 埋点

后台操作埋点按系统统一审计日志承载，不单列业务埋点。
