## 1. 页面定位

- **目标用户**：已登录普通用户（邀请人）
- **核心任务**：展示裂变收益、邀请进度，提供生成二维码与分享入口
- **页面类型**：活动页（仪表盘 + 操作入口）

## 2. 布局

### 2.1 整体布局

```
┌─────────────────────────────────────────┐
│ 顶部活动区：标题 + 副标题 + 活动说明入口   │
```

## 7. 状态与异常

| 状态类型 | 触发场景 | 页面表现 | 用户可做的操作 | 引用 |
|----------|----------|----------|---------------|------|
| 加载态 | 首次加载 | 骨架屏 | 无 | — |
| 空态（无记录） | 暂无邀请记录 | 记录区「还没有邀请记录，快邀请同学一起玩」 | 分享 | — |
| 错误态（网络） | 请求超时/断网 | toast + 保留上次数据 | 重试 | — |
| 错误态（服务端） | 500/503 | toast + 重试按钮 | 重试 | `M07-ERR-*` |
| 无权限态 | 未登录点击入口 | 引导登录页 | 登录 | — |
| 业务态-有进度 | 已有成功邀请 | 进度区展示人数与币数、下一档位 | 继续分享 | `M07-RULE-invite-ladder` |
| 业务态-含冻结/无效记录 | 记录含冻结/无效 | 对应记录标记态 + 提示 | 查看记录详情 | `M07-SM-invite-reward` |
| 降级态（二维码） | 微信小程序码不可用 | 二维码区占位 + 重试 | 重试 | `M07-SRV-wx-qrcode` / `M07-ERR-7006` |
| 降级态（分享） | 微信分享不可用 | 隐藏分享按钮，引导保存二维码 | 保存图片 | `M07-SRV-wx-share` |

## 9. 验收标准

```
AC-ID: APP-07-AC-home-show
Given 已登录用户，已存在 2 个达到 `M07-RULE-invite-success` 统计节点的邀请关系，后台已配置阶梯规则
When  进入推荐给好友页
Then  进度区展示「成功邀请 2 人」+ 已到账千寻币累计值 + 下一档位说明，埋点 invite_home_show 上报

AC-ID: APP-07-AC-share-success
Given 已登录用户，微信分享可用
When  点击底部「分享」并完成微信分享
Then  调起微信分享卡片，埋点 invite_share_click 与 invite_share_success 上报

AC-ID: APP-07-AC-qr-fail-degrade
Given 微信小程序码服务不可用（M07-SRV-wx-qrcode）
When  进入页面
Then  二维码区展示占位 + 重试按钮（M07-ERR-7006），页面其余区块正常渲染，不阻塞

AC-ID: APP-07-AC-save-no-permission
Given 用户未授权相册权限
When  点击保存二维码
Then  引导开启相册权限，不报错崩溃
```

### 验收标准清单

| AC ID | 场景 | 类型 | 优先级 |
|-------|------|------|--------|
| `APP-07-AC-home-show` | 进度展示与埋点 | 正常 | P0 |
| `APP-07-AC-share-success` | 分享成功 | 正常 | P0 |
| `APP-07-AC-qr-fail-degrade` | 二维码降级 | 异常 | P0 |
| `APP-07-AC-save-no-permission` | 保存无权限 | 异常 | P1 |
