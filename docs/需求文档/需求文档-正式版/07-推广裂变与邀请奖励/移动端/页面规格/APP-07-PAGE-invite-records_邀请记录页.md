## 1. 页面定位

- **目标用户**：邀请人
- **核心任务**：让邀请人清楚知道「邀请是否成功、奖励是否到账」
- **页面类型**：列表页

---

## 2. 布局

### 2.1 整体布局

```
┌─────────────────────────────────────────┐
│ 顶部统计摘要：累计成功 / 已奖励币数        │
├─────────────────────────────────────────┤
│ 状态筛选 Tab：全部/已成功/已奖励/冻结/无效 │
├─────────────────────────────────────────┤
│ 记录列表（下拉刷新 + 上拉加载更多）        │
│  · 脱敏昵称 + 状态胶囊                    │
│  · 绑定时间 + 已发奖励                    │
│  · 无效/冻结原因                          │
└─────────────────────────────────────────┘
```

### 2.2 区块说明

| 区块 | 位置 | 内容 | 是否可折叠 | 是否记住展开状态 |
|------|------|------|-----------|----------------|
| 统计摘要 | 顶部 | 累计成功邀请人数、已奖励千寻币 | 否 | 否 |
| 筛选 Tab | 摘要下 | 全部/已成功/已奖励/冻结中/无效 | 否 | 是（记住选中 Tab） |
| 记录列表 | 主体 | 逐条邀请记录 | 否 | 否 |

## 7. 状态与异常

| 状态类型 | 触发场景 | 页面表现 | 用户可做的操作 | 引用 |
|----------|----------|----------|---------------|------|
| 加载态 | 首次加载/切 Tab | 骨架屏 | 无 | — |
| 空态（无数据） | 列表 0 条 | 插画 +「还没有邀请记录，快邀请同学一起玩」 | 返回首页分享 | — |
| 空态（筛选无结果） | 某 Tab 无记录 | 插画 +「该状态暂无记录」 | 切 Tab | — |
| 错误态（网络/服务端） | 请求失败 | toast + 重试 | 重试 | — |
| 业务态-已成功 | 达到 `M07-RULE-invite-success` 统计节点 | 绿色「已成功」胶囊 | 查看 | `M07-SM-invite-relation` / `M07-RULE-invite-success` |
| 业务态-冻结 | 状态=冻结中 | 「风险冻结中」胶囊 + 提示 | 查看原因 | `M07-SM-invite-reward` |
| 业务态-无效 | 状态=无效 | 「无效」胶囊 + 原因 | 查看原因 | `M07-RULE-invite-invalid` |

## 8. 查询与列表

- **默认排序**：绑定时间倒序
- **分页方式**：上拉加载更多，默认 20 条/页
- **下拉刷新**：支持
- **批量选择**：不支持

## 9. 验收标准

```
AC-ID: APP-07-AC-records-filter
Given 用户有多条不同状态的邀请记录
When  切换到「冻结中」Tab
Then  仅展示冻结中的记录，每条展示脱敏昵称、绑定时间（yyyy-MM-dd HH:mm:ss）、冻结提示，埋点 invite_record_show 上报

AC-ID: APP-07-AC-records-empty
Given 用户无任何邀请记录
When  进入记录页
Then  展示空态插画与分享引导

AC-ID: APP-07-AC-records-invalid-reason
Given 一条状态=无效的记录
When  点击查看原因
Then  展示无效原因（M07-ENUM-risk-hit-reason）与文案 M07-TXT-invite-reward-invalid
```

### 验收标准清单

| AC ID | 场景 | 类型 | 优先级 |
|-------|------|------|--------|
| `APP-07-AC-records-filter` | 状态筛选 | 正常 | P0 |
| `APP-07-AC-records-empty` | 空态 | 正常 | P1 |
| `APP-07-AC-records-invalid-reason` | 无效原因 | 异常 | P1 |
