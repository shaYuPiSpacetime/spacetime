# 成家立业项目 14 周排期、甲方汇报时间轴与上线准备建议

## 1. 说明与排期假设

- 本排期基于《成家立业 · 移动端（微信小程序）产品需求文档 V1.0》与《成家立业 · 管理后台产品需求文档 V1.0》整理。
- 排期口径按 `T0 = 项目启动周` 计算，总周期 `14 周`。
- 采用“产品、UI、技术同步开工”的并行方式，不按“产品做完再给 UI、UI 做完再给技术”的串行模式安排。
- 本排期默认首版以“交友主链路 + 认证 + 会员/成家币 + 举报审核 + 管理后台核心配置”作为上线主目标；`社区公开发帖 / 复杂活动 / AB测试 / 高级数据看板` 建议按上线风险决定是否压到二期。
- 若按 `2026-04-28` 启动，`W14` 结束时间约为 `2026-08-02`。

## 3. 产品、UI、技术各自必须卡住的截止点

### 3.1 产品关键截止点

- `W1 末`：版本边界、功能优先级、信息架构必须确认。
- `W2 末`：PRD、流程图、接口清单、字段字典、埋点清单必须完成第一版冻结。
- `W4 末`：后台配置规则、举报处罚规则、支付规则、审核规则必须冻结。
- `W6 末`：需求变更截止点 1，之后只允许补充细节，不再新增核心功能。
- `W10 末`：上线文案、协议、审核说明、提审资料包必须定稿。
- `W12 末`：甲方验收口径、审核回复口径、上线清单必须最终确认。

### 3.2 UI 关键截止点

- `W1 末`：视觉方向和设计规范初稿完成。
- `W2 末`：交友主链路核心页面高保真第一批完成。
- `W4 末`：后台关键页面高保真完成。
- `W6 末`：全量页面设计稿、组件库、空态/异常态、标注切图完成。
- `W8 末`：审核版 UI 收口完成。
- `W10 末`：上线素材包定稿。

### 3.3 技术可以优先同步做的事情

- `第 1 天就能启动`：工程初始化、环境搭建、数据库设计、账号权限、日志审计、上传服务、对象存储、CI/CD。
- `不等最终 UI 也能做`：登录鉴权、用户/认证/标签/订单/支付/举报/内容审核的数据结构和接口骨架。
- `不等最终 PRD 也能做`：管理后台 RBAC、配置中心、字典管理、操作日志、基础埋点框架、消息模板、客服工单骨架。
- `必须提前做`：内容安全能力接入、风控与黑名单、举报处罚链路、支付商户联调、域名证书备案、提审资料预检。

## 4. 建议的版本范围控制

### 4.1 首版建议必须上线的 P0

- 登录注册
- 三重认证：实名、学历、头像
- 成家主链路：觅缘推荐、用户主页、喜欢/跳过/举报/屏蔽
- 会员与成家币：套餐、充值、消费、订单
- 理想型搜索基础版
- 喜欢我的 / 最近看过我的
- 消息基础版
- 举报、黑名单、处罚、申诉
- 管理后台：用户、认证审核、内容审核、会员配置、成家币配置、系统参数、菜单权限、客服工单

### 4.2 建议放到二期或灰度的 P1/P2

- 社区公开发帖与复杂互动
- 朋友模块完整能力
- 复杂活动运营、AB测试、数据看板
- 量表高级配置与完整择业链路
- 推广体系全量功能

### 4.3 原因

- 该项目属于交友/婚恋类小程序，审核和资质要求明显高于普通工具型小程序。
- 若首版同时上“婚恋 + 社区公开发帖 + 笔记/论坛式内容广场”，通常会增加类目、内容安全、审核解释和二审风险。
- 因此首版更适合先把“婚恋主链路 + 审核治理后台”做稳，再扩社区。

## 6. 上线前必须准备的资料清单

## 6.1 主体与账号资料

- 非个人主体微信小程序账号
- 营业执照 / 统一社会信用代码证
- 小程序管理员、主体负责人、项目负责人身份证件
- 微信认证资料
- 对外客服信息：客服电话、客服邮箱、投诉处理联系人

## 6.2 类目与资质资料

- 至少需要评估并申请以下社交类目之一或组合：
- `社交-婚恋`
- `社交-陌生人交友 / 熟人交友`
- 若首版包含公开社区/论坛型 UGC：评估补充 `社交-社区/论坛`
- 若首版包含公开图文笔记分享：评估补充 `社交-笔记`
- `《增值电信业务经营许可证》`，且核准服务项目需包含“互联网信息服务业务”
- 如涉及社区/论坛类开放能力，按对应类目准备所需备案/资质材料

## 6.3 备案资料

- 主办单位证件
- 主体负责人证件
- 小程序负责人证件
- 前置审批 / 专项审批文件（如被要求）
- 授权书、情况说明、承诺书等补充材料
- 小程序负责人核验照片 / 人脸核身配合

## 6.4 技术基础设施

- 生产服务器
- 测试服务器 / 预发服务器
- 数据库
- 对象存储（用户头像、认证资料、社区图片）
- CDN
- HTTPS 证书
- 日志平台
- 监控告警
- 备份与容灾策略
- 短信能力
- 微信支付商户号（若首版包含会员/成家币充值）

## 6.5 小程序域名与网络要求

- 请求域名必须提前配置在微信后台
- 仅支持 `https` 和 `wss`
- 不支持 `IP`、`localhost`
- 域名必须完成 `ICP 备案`
- 不支持配置父域名，应使用子域名
- `https` 域名如配置端口，调用时必须与配置端口完全一致
- 业务域名（`web-view`）若使用，也必须是 `https`，且完成 ICP 备案；新备案域名一般需 `24 小时后` 才可配置
- HTTPS 证书需有效、信任链完整、域名匹配，`TLS 1.2` 及以上

## 6.6 审核与合规资料

- 用户协议
- 隐私政策
- 注销协议 / 注销流程说明
- 未成年人保护说明（如涉及）
- 举报机制说明
- 黑名单 / 封禁 / 申诉规则
- 内容审核机制说明
- 三重认证规则说明
- 审核测试账号
- 审核路径说明文档
- 后台审核账号（必要时）

## 7. 微信交友类小程序特别注意事项

### 7.1 本项目不是普通类目小程序

- 按 PRD 功能形态判断，“成家立业”明显属于 `交友 / 婚恋` 场景。
- 若按婚恋、陌生人交友、熟人交友形态上线，不能只靠普通企业主体直接提审，必须先匹配正确类目与资质。

### 7.2 首次提审要预留二次审核时间

- 微信官方说明：选择 `社交-陌生人交友 / 熟人交友` 或 `社交-婚恋` 后，`首次提交代码审核` 需经当地互联网主管机关审核确认，`预计审核时长 7 天左右`。
- 这 7 天不能压缩，所以必须在 `W12` 前把代码、资料、测试账号、审核备注全部准备齐。

### 7.3 内容安全不是可选项

- 交友类小程序只要有用户发布文本、图片、动态、聊天、评论、悄悄话等场景，就必须把内容安全能力接入到所有发布入口。
- 至少应覆盖：
- 文本内容安全识别
- 图片/音频等多媒体内容安全识别
- 举报、删除、禁言、封禁、申诉闭环
- 后台审核记录与操作日志留痕

### 7.4 社区能力会抬高审核要求

- PRD 中有 `社区动态`、`评论`、`朋友模块` 等公开交流能力。
- 依据微信官方开放类目规则，这些能力可能触发 `社交-社区/论坛` 或 `社交-笔记` 的补充要求。
- 因此建议首版优先上线婚恋主链路，社区公开发帖能力谨慎首发。

## 8. 对甲方的建议话术

- 这个项目按产品形态不只是“做个小程序页面”，而是标准的交友/婚恋类平台。
- 真正影响上线时间的不是开发本身，而是 `资质、类目、备案、内容审核机制、首次属地二审`。
- 因此 14 周排期里必须把合规准备放进主计划，而不是等开发完成后再补。
- 若甲方暂时无法补齐 `婚恋/交友类目 + 增值电信业务经营许可证`，建议先调整首版范围，避免开发完成后因类目不符无法上线。
