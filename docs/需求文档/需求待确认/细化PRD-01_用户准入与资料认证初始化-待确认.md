# PRD-01 用户准入与资料认证初始化 - 产品/技术确认清单

> 用法：逐行填写“确认结论”。P0 先确认，P1 开发前确认，P2 可后续优化。

| P-03 | P0 | 学历认证实名前置 | 学历认证依赖实名认证“已提交”还是“已通过”？是否允许先提交学历材料、实名后再审核？ |  |

| P-11 | P0 | 性别修改 | 性别是否提交后不可改？后台客服是否允许特殊修改？修改后是否重算推荐/曝光/匹配？ |  |
| P-12 | P0 | 年龄规则 | 最小年龄、最大年龄、年龄计算规则是什么？未成年人已创建账号和已采集资料如何处理？ |  |
| P-13 | P0 | 手机号绑定 | 首版是否强制绑定手机号？绑定时机在哪里？拒绝授权是否可继续？后台手机号为空如何展示？ |  |
| P-13A | P0 | 登录方式/SMS | 登录注册是否只保留微信授权，还是保留手机号+密码/验证码？如保留，SMS 网关资质和供应商是否已确认？ |  |
| P-14 | P0 | 后台用户管理形态 | 后台用户管理是否按 UI 做卡片宫格+右侧画像抽屉，而不是传统表格？页面名叫客户管理、App 用户管理还是用户准入？ |  |
| P-15 | P0 | 红娘/牵线 | 红娘相关字段是否全部删除？牵线/快速牵线入口是否保留？ |  |

| T-10 | P0 | 敏感字段存储 | 身份证号、证书编号、在线验证码、手机号是否加密存储？日志是否禁止敏感明文？ |  |
| T-11 | P0 | 认证事件 | 实名/头像/学历通过后是否发事件？推荐曝光池、邀请奖励、准入状态、后台统计如何刷新？ |  |

| T-15 | P0 | 导出审计 | 导出是否异步？下载链接是否短期有效？是否记录导出人、条件、字段范围、下载日志？ |  |
