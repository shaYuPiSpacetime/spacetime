# 04-商业化（VIP、千寻币、解锁与资产中心）静态 HTML 实现方案

## 1. Demo 目标

本 Demo 用可直接打开的静态 HTML 还原 PRD-04 的移动端会员/千寻币/资产中心/支付结果/付费弹窗，以及管理后台商业化配置、财务查询、订单内退款、退款记录和轻量对账页面。用户商业化详情已由用户管理详情弹窗/详情页承载，本 Demo 不再单独绘制。Demo 不接真实接口、不写数据库、不依赖登录态或本地 dev server，全部数据来自本地 mock。

页面总数为 13 个，超过 5 页，按工作流拆分为：

| 文件 | 作用 |
|------|------|
| `html/index.html` | 总导航入口，展示模块目标、端入口、页面范围、关键业务闭环。 |
| `html/miniapp.html` | 小程序端全量页面，使用手机外框、锚点导航和本地 JS 模拟交易/资产状态。 |
| `html/admin.html` | 管理后台全量页面，使用后台骨架、左侧菜单、筛选、表格、分页、抽屉和弹窗。 |
| `html/assets/demo.css` | 04 模块专属样式：商业化移动端卡片、付费弹窗、后台配置 Tab、财务表格。 |
| `html/assets/demo.js` | 04 模块专属交互：套餐切换、支付结果、付费弹窗、后台 Tab/筛选/抽屉/导出。 |
| `html/mock/demo-data.js` | mock 数据：会员、套餐、千寻币、流水、订单、配置、退款、对账。 |

复用公共资源：

| 公共文件 | 复用范围 |
|----------|----------|
| `shared/base.css` | 设计变量、顶部栏、按钮、标签、弹窗、抽屉、Toast。 |
| `shared/admin.css` | 管理后台骨架、侧边栏、表格、查询区、分页。 |
| `shared/admin-state.css` | 后台 hash 页面显示/隐藏状态层。 |
| `shared/demo-common.js` | 通用选择器、状态样式、Toast、弹窗和抽屉方法。 |
