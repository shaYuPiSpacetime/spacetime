## 5. 缺口

| 编号 | 缺口 | 等级 | 对 HTML 影响 | 处理方式 |
|------|------|------|-------------|----------|
| GAP-01 | 未提供 Figma/蓝湖设计稿链接。 | P1 | 不能声明像素级还原设计稿。 | 按 PRD 文字、现有静态 Demo 公共样式和移动端/后台通用模式生成高保真静态原型。 |
| GAP-02 | 套餐价格、币数、标签默认值不固定，全部由后台配置。 | P0 | HTML 不能把 mock 数字宣称为正式默认值。 | 在 mock 中展示“后台配置”标签，并在文档/页面标注为演示数据。 |
| GAP-03 | 前台无主动退款申请，后台首版退款页不开放人工改状态。 | P0 | 容易误做退款申请/审批按钮。 | HTML 仅展示退款状态、详情和导出，不提供申请退款或改状态按钮。 |
| GAP-04 | 真实微信支付、微信连续订阅、登录态和三重认证状态不可用。 | P0 | 不能做真实支付、真实取消续费或真实准入校验。 | 全部用本地 mock、弹窗和 toast 模拟成功/失败/降级态。 |
| GAP-05 | `ADM-GLB-PAGE-copy-message-center` 是全局后台页面，不在 PRD-04 页面规格目录下。 | P1 | 如单独生成容易越界。 | 在配置页展示商业化文案入口和引用说明，不单独实现全局页面。 |
