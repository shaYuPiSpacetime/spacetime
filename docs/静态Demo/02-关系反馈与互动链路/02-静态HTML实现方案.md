# 02-关系反馈与互动链路静态 Demo - 静态 HTML 实现方案

## 1. 技术方案

| 项 | 说明 |
|----|------|
| 形式 | 纯静态 HTML/CSS/少量本地 JS |
| 依赖 | 复用 `docs/静态Demo/shared/base.css`、`admin.css`、`admin-state.css`、`demo-common.js` |
| 数据 | 直接写入静态页面，无后端依赖 |
| 路由 | `miniapp.html` 展示移动端页面，`admin.html` 展示后台合并说明 |
| 目标 | 补齐 PRD-02 移动端页面演示，并明确后台用户管理增强已合并至 01、关系反馈规则按代码固定实现、不提供后台配置页 |

## 2. 文件结构

```text
docs/静态Demo/02-关系反馈与互动链路/
├─ 00-文档读取与页面范围.md
├─ 01-页面元素清单.md
├─ 02-静态HTML实现方案.md
└─ html/
   ├─ index.html
   ├─ miniapp.html
   ├─ admin.html
   ├─ assets/
   │  ├─ demo.css
   │  └─ miniapp.js
```

## 3. 样式原则

| 原则 | 处理 |
|------|------|
| 与 04 保持一致 | 沿用后台暗色侧边栏、浅灰工作区、表格与抽屉样式 |
| 移动端真实可演示 | 沿用 04 手机壳、弹窗、toast 和按钮样式，补关系反馈独有卡片、模糊头像、状态切换 |
| 合并口径清晰 | 用户列表字段补充、用户详情关系记录区块在 02 中展示为承接说明，不重复做独立页面 |
| 规则不配置化 | 关系反馈规则不做后台配置页，按代码固定实现 |

## 4. 自测口径

| 检查项 | 方式 |
|--------|------|
| 页面可打开 | 浏览器加载 `html/index.html`、`html/miniapp.html` 与 `html/admin.html` |
| 移动端交互可用 | 状态切换、单条解锁弹窗、匹配成功弹窗、失效原因弹窗 |
| 导航可用 | 首页入口、移动端侧栏、后台合并说明页入口 |
| 合并入口可用 | 点击打开 01 App 用户管理入口 |
| 文档口径一致 | 搜索 PRD-02 关键 ID 与 Demo 页面元素 |
