# 01-用户准入与资料认证初始化 静态 HTML 实现方案

## 1. Demo 目标

本 Demo 用可直接打开的静态 HTML 还原 PRD-01 的移动端用户准入、资料认证初始化和管理后台审核/配置页面，用于产品、需求方和技术评审对齐。Demo 不接真实接口、不写数据库、不依赖登录态、不依赖本地 dev server，所有数据来自本地 mock。

页面总数为 23 个，超过 5 页，按工作流拆分为：

| 文件 | 作用 |
|------|------|
| `html/index.html` | 总导航入口，说明端和页面组，链接到移动端/后台演示。 |
| `html/miniapp.html` | 小程序端全量页面，使用手机外框和本地 JS 切换页面。 |
| `html/admin.html` | 管理后台全量页面，使用后台骨架、侧边菜单、筛选、列表、抽屉、弹窗和配置 Tab。 |
| `html/assets/demo.css` | 01 模块专属样式：手机外框、准入配置、用户画像、审核队列和页面细节覆盖。 |
| `html/assets/demo.js` | 01 模块专属交互：mock 渲染、认证流程、审核详情、配置 Tab 和状态切换。 |
| `html/mock/demo-data.js` | mock 数据：用户、认证、配置、审核队列、状态样例。 |

公共资源已抽取到 `docs/静态Demo/shared/`：

| 公共文件 | 复用范围 |
|----------|----------|
| `shared/base.css` | 设计变量、顶部栏、按钮、标签、弹窗、抽屉、Toast。 |
| `shared/admin.css` | 管理后台骨架、侧边栏、表格、查询区、分页、后台弹窗画板。 |
| `shared/admin-state.css` | 后台 hash 页面显示/隐藏状态层，加载在模块样式之后。 |
| `shared/demo-common.js` | 通用选择器、HTML 转义、状态样式、Toast、弹窗和抽屉方法。 |

## 3. HTML 结构

| 区块 | 类型 | 用途 | 覆盖页面 |
|------|------|------|----------|
| `demo-shell` | 首页导航 | 展示模块目标、交付物、端入口、页面组 | `index.html` |
| `mini-layout` | 手机外框 | 模拟小程序导航、页面、底部操作栏、toast | 全部 APP 页面 |
| `mini-page-nav` | 移动端页面索引 | 快速切换 APP-01 到 APP-16 | 全部 APP 页面 |
| `stepper` | 四步进度 | 强引导基本资料/头像/自我介绍/三重认证 | APP-07 到 APP-10 |
| `modal` | 通用弹窗 | 协议、学历前置、核心拦截、年龄阻断、审核确认 | 多页面 |
| `toast` | 反馈提示 | 保存成功、失败、定位失败、上传失败、导入结果 | 全部交互页 |
| `admin-layout` | 后台骨架 | 左侧菜单、顶部栏、面包屑、内容区 | 全部 ADM 页面 |
| `filter-bar` | 筛选区 | 输入框、下拉、日期范围、重置 | 后台列表页 |
| `pagination` | 分页区 | 默认每页 10 条、页码、总数 | 后台列表页 |
| `drawer` | 右侧抽屉 | 用户画像、配置日志、审核详情 | 用户管理、配置、审核详情 |
| `audit-table` | 审核列表 | 表格、状态标签、审核来源、操作 | 五类审核页 |
